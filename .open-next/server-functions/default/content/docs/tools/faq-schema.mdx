## 你可以用它做什麼

FAQ Schema 工具幫助你產生結構化資料，讓頁面在搜尋結果中顯示 FAQ 區塊：

- **AI 生成問答**：輸入主題，自動產生相關 FAQ
- **手動編輯**：自己撰寫或批量貼上問答
- **即時預覽**：看到 JSON-LD 程式碼的格式
- **一鍵匯出**：複製程式碼貼到你的網站

<Callout type="info">
  FAQ Schema 正確實作後，你的頁面可能在搜尋結果中顯示「常見問題」區塊，增加曝光面積和點擊率。這對 SEO 有正面影響。
</Callout>

## 快速開始

1. **進入 FAQ Schema 工具**：從側邊欄「工具 > FAQ Schema」進入
2. **選擇模式**：AI 生成（推薦新手）或手動撰寫
3. **輸入主題**：例如「如何選擇 SEO 工具」
4. **生成/編輯 FAQ**：調整問答內容
5. **複製 Schema**：點擊「複製 Schema」取得 JSON-LD
6. **貼到網站**：放入頁面的 `<head>` 或透過 GTM

## 前置條件

| 條件 | 必要性 | 說明 |
|------|--------|------|
| 有目標頁面 | **必要** | 需要知道 FAQ 要放在哪個頁面 |
| AI 額度（如用 AI 生成） | 建議 | AI 生成會消耗額度 |
| 可編輯網站 HTML | **必要** | 需要能把 Schema 貼到網站 |

## 什麼是 FAQ Schema？

FAQ Schema（FAQPage）是一種結構化資料格式，用 JSON-LD 語法告訴搜尋引擎這個頁面包含問答內容。

### 效果展示

實作 FAQ Schema 後，搜尋結果可能變成：

```
你的網頁標題
你的網頁描述...
▼ 問題 1：如何使用這個功能？
▼ 問題 2：需要付費嗎？
▼ 問題 3：有試用期嗎？
```

這樣的展開區塊會：
- 佔據更大的搜尋結果版面
- 吸引更多點擊
- 直接回答使用者問題

## 三種操作模式

### 1. AI 生成 FAQ（推薦新手）

讓 AI 根據主題自動產生問答：

| 輸入 | 說明 | 範例 |
|------|------|------|
| **FAQ 主題** | AI 會根據這個主題生成問答 | 「如何選擇適合的 SEO 工具」 |
| **期望數量** | 要生成幾組問答（1-20） | 5 |
| **額外提示**（選填） | 給 AI 的額外指示 | 「針對中小企業、用口語化語氣」 |
| **參考頁面**（選填） | 讓 AI 參考你網站的內容 | 選擇相關頁面 |

**操作步驟**：

1. 切換到「AI 生成」分頁
2. 輸入主題和設定
3. 點擊「生成」
4. 等待 AI 產生結果（約 10-30 秒）
5. 在編輯器中審核和調整

<Callout type="info">
  AI 生成會消耗 AI 額度。額度狀態顯示在頁面右上角。
</Callout>

### 2. 手動撰寫 FAQ

自己撰寫或批量貼上問答：

**單一新增**：
1. 點擊「新增問答」
2. 輸入問題和答案
3. 重複直到完成

**批量貼上**：
1. 準備問答文字（支援常見格式）
2. 貼到編輯區
3. 系統自動解析成問答對

支援的格式：
```
Q: 問題 1？
A: 答案 1。

Q: 問題 2？
A: 答案 2。
```

或

```
問：問題 1？
答：答案 1。
```

### 3. 已儲存的 FAQ

管理之前儲存的 FAQ 集合：

| 功能 | 說明 |
|------|------|
| **列表檢視** | 看到所有已保存的 FAQ |
| **搜尋** | 快速找到特定 FAQ |
| **編輯** | 點擊進入編輯 |
| **刪除** | 移除不需要的 FAQ |
| **匯出** | 批量匯出 Schema |

## 編輯器功能

### 問答編輯

| 功能 | 說明 |
|------|------|
| **編輯問題** | 修改問題文字 |
| **編輯答案** | 修改答案內容，支援基本格式 |
| **調整順序** | 拖放改變問答順序 |
| **刪除項目** | 移除不需要的問答 |
| **AI 優化** | 讓 AI 改寫現有問答 |

### 即時 Schema 預覽

編輯器右側即時顯示 JSON-LD 格式：

```json
{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "問題 1？",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "答案 1。"
      }
    }
  ]
}
```

### 來源標籤

每個問答會標示來源：

| 標籤 | 說明 |
|------|------|
| **AI 生成** | 由 AI 自動產生 |
| **手動** | 手動撰寫 |
| **匯入** | 從外部匯入 |

## 匯出與實作

### 複製 Schema

1. 確認預覽的 JSON-LD 正確
2. 點擊「複製 Schema」按鈕
3. Schema 程式碼已在剪貼簿

### 貼到網站的方法

**方法 1：直接編輯 HTML**

把 Schema 貼到頁面的 `<head>` 區塊：

```html
<head>
  <!-- 其他 meta tags -->
  <script type="application/ld+json">
  {
    "@context": "https://schema.org",
    "@type": "FAQPage",
    ...
  }
  </script>
</head>
```

**方法 2：透過 Google Tag Manager (GTM)**

1. 在 GTM 建立新的「自訂 HTML」標籤
2. 貼上 `<script type="application/ld+json">...</script>`
3. 設定觸發條件（指定頁面）
4. 發佈 GTM 容器

**方法 3：CMS 外掛**

部分 CMS（如 WordPress）有 Schema 外掛，可以貼 JSON-LD 到特定欄位。

### 驗證 Schema

實作後建議驗證：

1. 前往 [Google Rich Results Test](https://search.google.com/test/rich-results)
2. 輸入頁面 URL
3. 確認偵測到 FAQ Schema
4. 檢查是否有錯誤或警告

## 常見問題

### AI 生成的內容不太對

**處理方式**：

- AI 生成只是初稿，需要人工審核
- 在編輯器中直接修改
- 使用「額外提示」給 AI 更明確的指示
- 選擇「參考頁面」讓 AI 參考你的內容

### 加了 Schema 但搜尋結果沒顯示 FAQ

**可能原因**：

| 原因 | 解法 |
|------|------|
| Google 尚未重新爬取 | 等待 1-2 週，或在 GSC 要求索引 |
| Schema 格式有錯 | 用 Rich Results Test 檢查 |
| 頁面品質不足 | Google 不一定會顯示所有 FAQ Schema |
| FAQ 內容與頁面不符 | 確保 FAQ 與頁面主題一致 |
| 頁面上沒有對應內容 | Google 要求 FAQ 內容要在頁面可見 |

### 需要幾個問答才有效？

建議數量：

| 數量 | 建議 |
|------|------|
| 1-2 個 | 太少，效果有限 |
| **3-7 個** | 最適合，不會太多也不會太少 |
| 8-10 個 | 可以，但注意品質 |
| > 10 個 | 通常太多，考慮精簡 |

### 可以在每個頁面都加 FAQ Schema 嗎？

可以，但要注意：

- 每個頁面的 FAQ 必須與該頁面內容相關
- 不要在所有頁面放相同的 FAQ
- 只在真正適合的頁面加（如產品頁、服務頁、教學頁）

## 最佳實踐

### FAQ 內容原則

| 做 | 不做 |
|-----|------|
| 問題具體明確 | 問題太籠統（「這是什麼？」） |
| 答案完整有價值 | 答案只有一句話 |
| 與頁面主題相關 | FAQ 與頁面無關 |
| 回答使用者真正會問的問題 | 自問自答行銷語言 |

### 問題撰寫技巧

好的問題：
- 「如何開始使用 Otlex 的 SEO 稽核功能？」
- 「SEO 稽核報告多久會產生？」
- 「免費方案包含哪些功能？」

不好的問題：
- 「Otlex 好用嗎？」（太主觀）
- 「SEO」（不是問題）
- 「為什麼要選擇我們？」（太行銷）

### 答案撰寫技巧

- 直接回答問題，不要繞圈子
- 提供具體資訊（數字、步驟、清單）
- 長度適中（50-200 字）
- 避免純粹的推銷語言

### 技術實作建議

| 項目 | 建議 |
|------|------|
| **放置位置** | `<head>` 區塊（推薦）或 `<body>` 結尾 |
| **一頁一份** | 每個頁面只放一個 FAQPage Schema |
| **內容可見** | FAQ 內容必須在頁面上實際可見 |
| **定期更新** | FAQ 內容過時就更新 Schema |

## 與其他功能整合

| 功能 | 如何搭配 |
|------|----------|
| **SEO 稽核** | 稽核發現缺少 Schema → 用此工具補上 |
| **AI 內容生成** | 文章生成後可加 FAQ 增強 |
| **內容地圖** | 找出重要頁面優先加 FAQ |
