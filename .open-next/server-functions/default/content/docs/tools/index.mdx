## 你可以用它做什麼

專用工具是針對特定 SEO 任務設計的獨立功能模組：

- **解決特定問題**：每個工具專注處理一種 SEO 挑戰
- **產出可行動的結果**：不只是分析，而是直接產出可使用的內容或建議
- **與其他功能整合**：工具的產出可以應用到網站、內容地圖、稽核流程中

## 工具總覽

| 工具 | 解決什麼問題 | 主要產出 | 適用時機 |
|------|-------------|----------|----------|
| [FAQ Schema](/docs/tools/faq-schema) | 頁面缺乏結構化資料 | JSON-LD 程式碼 | 想獲得搜尋結果的 FAQ 展開效果 |
| [內部連結](/docs/tools/internal-links) | 內連不足或分布不均 | 連結建議清單 | 想提升頁面權重傳遞 |
| [關鍵字研究](/docs/tools/keyword-research) | 不知道該鎖定哪些關鍵字 | 關鍵字機會分析 | 規劃新內容或優化方向 |
| [重複內容偵測](/docs/tools/content-duplication) | 內容重複影響排名 | 重複頁面配對報告 | 網站內容量大、擔心重複 |

## 我該用哪個工具？

根據你目前面對的問題選擇工具：

```
你的問題是什麼？
│
├── 想讓頁面在搜尋結果更突出？
│   └── FAQ Schema → 產生結構化資料讓 Google 顯示 FAQ
│
├── 網站內連太少 / 新頁面沒人連？
│   └── 內部連結工具 → 找出適合加連結的位置
│
├── 不確定該寫什麼內容？
│   └── 關鍵字研究 → 分析 GSC 資料找出機會關鍵字
│
├── 懷疑網站有重複內容？
│   └── 重複內容偵測 → 掃描並配對相似頁面
│
└── 剛完成 SEO 稽核，不知從何下手？
    ├── 有「缺少 FAQ Schema」建議 → FAQ Schema 工具
    ├── 有「內連不足」建議 → 內部連結工具
    └── 有「內容重複」建議 → 重複內容偵測
```

## 共同前置條件

所有工具都需要：

| 條件 | 說明 | 如何確認 |
|------|------|----------|
| 已建立品牌 | 工具需要知道要分析哪個網站 | 「設定 > 品牌」有資料 |
| 已完成內容爬取 | 大部分工具需要網站內容資料 | 「內容地圖」有頁面資料 |

部分工具的額外條件：

| 工具 | 額外條件 | 原因 |
|------|----------|------|
| 關鍵字研究 | GSC 已連接 | 需要真實搜尋資料 |
| 內部連結 | 內容地圖已爬取 | 需要頁面連結關係 |
| 重複內容偵測 | 內容地圖已爬取 | 需要頁面內容 |

## 各工具輸入/輸出說明

### FAQ Schema 工具

| 項目 | 說明 |
|------|------|
| **輸入** | 頁面 URL 或主題 |
| **處理** | AI 分析頁面內容，產生相關 FAQ |
| **輸出** | FAQ 問答 + JSON-LD 結構化資料程式碼 |
| **完成定義** | 取得 JSON-LD 並貼到網頁的 `<head>` 或 `<body>` |

### 內部連結工具

| 項目 | 說明 |
|------|------|
| **輸入** | 選擇來源頁面（要被加連結的頁面） |
| **處理** | 分析全站內容，找出語意相關的連結機會 |
| **輸出** | 建議清單：「在 A 頁面的 X 段落加上連到 B 頁面的連結」 |
| **完成定義** | 依建議在 CMS 中加入內部連結 |

### 關鍵字研究工具

| 項目 | 說明 |
|------|------|
| **輸入** | GSC 資料（自動讀取） |
| **處理** | 分析曝光量、排名、點擊率，找出機會關鍵字 |
| **輸出** | 關鍵字機會清單（含分類：快速致勝、潛力成長、需要新內容） |
| **完成定義** | 選定要優化的關鍵字，規劃內容行動 |

### 重複內容偵測工具

| 項目 | 說明 |
|------|------|
| **輸入** | 內容地圖資料（自動讀取） |
| **處理** | 計算頁面間的內容相似度 |
| **輸出** | 重複配對清單（相似度百分比、建議處理方式） |
| **完成定義** | 處理重複內容（合併、canonical、刪除、改寫） |

## 工具使用建議

### 建議的使用順序

如果你是第一次使用這些工具，建議按以下順序：

1. **先做重複內容偵測**：確保基礎乾淨
2. **再做關鍵字研究**：找出優化方向
3. **接著用內部連結**：強化重要頁面
4. **最後加 FAQ Schema**：提升搜尋呈現

### 搭配 SEO 稽核使用

最有效的使用方式是搭配 SEO 稽核：

1. 執行 SEO 稽核，取得問題清單
2. 根據稽核建議，選用對應工具
3. 用工具產出結果來修復問題
4. 再次稽核確認改善

### 定期維護建議

| 工具 | 建議頻率 | 原因 |
|------|----------|------|
| 重複內容偵測 | 每月 | 新內容可能產生新重複 |
| 關鍵字研究 | 每週/每兩週 | GSC 資料持續更新 |
| 內部連結 | 發布新內容後 | 新頁面需要內連 |
| FAQ Schema | 按需 | 重要頁面才需要 |

## 常見問題

### 工具跑很久是正常的嗎？

部分工具需要處理大量資料：

| 工具 | 預期時間 | 影響因素 |
|------|----------|----------|
| FAQ Schema | 10-30 秒 | AI 生成速度 |
| 內部連結 | 30 秒 - 2 分鐘 | 網站頁面數量 |
| 關鍵字研究 | 10-30 秒 | GSC 資料量 |
| 重複內容偵測 | 1-5 分鐘 | 網站頁面數量 |

如果超過預期時間太久，可以重新整理頁面再試。

### 工具結果不準確怎麼辦？

- **FAQ Schema**：AI 生成的內容需要人工審核，可能需要編輯
- **內部連結**：建議是參考，需判斷是否符合內容脈絡
- **關鍵字研究**：機會分類是演算法判斷，需結合業務判斷
- **重複內容**：相似度是數學計算，低相似度的配對可忽略

### 可以匯出工具結果嗎？

| 工具 | 匯出格式 | 如何匯出 |
|------|----------|----------|
| FAQ Schema | JSON-LD / 複製 | 頁面內直接複製 |
| 內部連結 | CSV | 匯出按鈕 |
| 關鍵字研究 | CSV | 匯出按鈕 |
| 重複內容偵測 | CSV | 匯出按鈕 |
