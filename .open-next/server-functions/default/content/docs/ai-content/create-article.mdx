## 開始之前

在生成第一篇文章前，建議先完成以下準備（越完整越好用）：

| 項目 | 必要性 | 說明 |
|-----|-------|------|
| 品牌基本資訊 | ✅ 必要 | 先建立品牌並確認網站網域 |
| Google Search Console | 推薦 | 可在生成時選取 GSC 查詢/關鍵字當作參考 |
| 已爬取的網站頁面 | 推薦 | 用於品牌寫作風格分析、內部連結/內容參考等功能 |
| CMS 發佈整合 | 選用 | 若你想一鍵發佈/匯出到 WordPress/Shopify/Webflow/Notion/Google Docs，請先在「設定 > 整合」完成設定 |

## 文章生成流程

### 步驟 1：建立或挑選一個主題（Topics）

1. 進入左側選單「主題」
2. 在上方分頁中，你可以：
   - **主題生成設定**：調整生成條件與權重後，送出主題生成任務
   - **已生成主題**：挑選一個主題作為文章生成的起點
   - **內容差距分析**：輸入競品網址，探索缺口主題

> 建議：主題越具體（要解決的問題、受眾、關鍵字/語氣限制），後續文章生成越穩定。

### 步驟 2：設定文章生成（Generation）

1. 進入左側選單「內容生成」
2. 切到「文章生成設定」
3. 選擇要生成的主題，並調整：
   - 生成語氣（tone）
   - 長度（short/medium/long）
   - 自訂提示詞（custom prompt）
   - 參考檔案（可選）
   - GSC 查詢/關鍵字（可選；需先完成 GSC 整合）

### 步驟 3：送出生成並等待背景任務完成

點擊開始生成後，系統會建立背景任務處理文案。你可以先離開頁面，完成後再回到：

- 「內容生成 > 已生成文章」查看文章清單與狀態
- 或直接開啟文章進入編輯

### 步驟 4：在文章編輯器編輯/協作

文章頁提供幾個常用能力（依權限/環境可能略有差異，以畫面為準）：

- **富文字 / HTML 模式**：可在可視化編輯與 HTML 原始碼間切換
- **插入圖片**：透過工具列選擇圖片上傳
- **選取改寫**：選取文字後請 AI 改寫（會消耗 AI 額度）
- **內部連結工具**：在編輯器內插入內部連結（獨立管理分頁仍為即將推出）
- **註解/協作**：可在文章上建立註解並 @mention
- **歷史版本**：重新生成/覆蓋內容前，系統會保留版本方便回復

### 步驟 5：發佈／匯出／產生社群貼文

在文章頁右上角的「發佈／匯出」可：

- 複製 **HTML / Markdown**
- 依整合狀態發佈到 **WordPress / Shopify / Webflow**
- 匯出到 **Notion / Google Docs**（依整合模式可能是臨時匯出或純文字寫入）

你也可以開啟文章頁的「貼文助手」，快速產生多平台的社群貼文草稿（含 UTM）。

## 常見問題

### 生成的內容品質不滿意？

1. 先確認 **主題內容夠具體**（受眾、要解的問題、關鍵字/語氣限制）
2. 設定並啟用 **品牌寫作風格**（可提高一致性）
3. 在編輯器內使用 **選取改寫** 或 **重新生成**（重新生成會先建立歷史版本）

### 生成失敗怎麼辦？

可能原因：
- AI 配額/點數不足（或自備 API Key 額度不足）
- 整合/權限設定不完整
- 內容限制或服務暫時性錯誤

解決方式：
-- 檢查 AI 額度與整合設定
-- 稍後重試
-- 若持續失敗，建議截圖錯誤訊息聯絡管理員

### 可以重新生成嗎？

可以。重新生成會呼叫 AI 服務並消耗額度；為避免遺失，系統會先將目前內容保存成「歷史版本」。

### 生成的文章版權歸誰？

你擁有所有使用 Otlex 生成內容的完整版權，可自由使用、修改和發布。
