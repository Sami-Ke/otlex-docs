<Callout type="info">
  以目前版本為準：WordPress / Shopify / Webflow 支援「基本文章發佈」；Notion / Google Docs 預設為「臨時匯出」（複製/下載），需啟用對應 API 模式後才會變成可直接發佈。實際可用項目請以 App 的「發佈」面板與「設定 > 整合 > 內容發佈」頁面顯示為準。
</Callout>

## 概覽

CMS 整合讓你可以將 Otlex 中生成或編輯的內容，直接發布到你使用的內容管理系統，省去複製貼上的步驟。

### 支援的平台

| 平台 | 狀態 | 主要功能 |
|-----|------|---------|
| **WordPress** | ✅ 支援 | 發布文章（title / content / excerpt / slug / status） |
| **Shopify** | ✅ 支援 | 發布部落格文章（title / body_html / tags / draft or published） |
| **Webflow** | 🚧 Beta | 發布 CMS Item（需 token + collectionId + 欄位對應） |
| **Notion** | 🚧 預設臨時匯出 | 臨時匯出 Markdown/HTML；啟用 API 模式後可建立 Notion Page/Database item（目前以純文字段落為主） |
| **Google Docs** | 🚧 預設臨時匯出 | 臨時匯出 Markdown/HTML；啟用 API 模式後可建立 Google 文件（目前以純文字寫入為主） |

### 為什麼需要 CMS 整合？

| 好處 | 說明 |
|-----|------|
| **節省時間** | 省去複製貼上、格式調整的步驟 |
| **保持格式** | 標題層級、列表、連結等格式完整保留 |
| **減少錯誤** | 避免手動複製時遺漏或格式錯誤 |
| **流程自動化** | 可搭配 CMS Publishing API（Webhook）由外部工具（Zapier/n8n 等）觸發發佈 |

> 自動排程發佈的完整流程仍在調整中；目前以「手動發佈/匯出」為主。

## Notion 整合

### 連接 Notion

1. 進入「設定」>「整合」>「內容發佈」
2. 在 Notion 卡片輸入：
   - Integration Token
   - Parent Page ID（或 Database ID）
3. 點擊「儲存 Notion 憑證」

### 授權設定

Notion 使用 **Internal Integration Token**。請先在 Notion 建立 Integration，並把目標 Page/Database 分享給該 Integration（否則會發佈失敗）。

<Callout type="info">
  安全建議：請建立專用的 Notion Integration，並只授權必要的 Page/Database。
</Callout>

### 發布到 Notion

1. 在文章編輯器中完成文章
2. 點擊「發佈」
3. 選擇 Notion

> 若目前 Notion 顯示為「Notion（臨時匯出）」：代表尚未啟用 Notion API 模式，會提供複製 Markdown / 下載檔案的匯出方式。

4.（API 模式）系統會以你設定的 Parent Page/Database 建立一個新頁面

### Notion 資料庫欄位對應

目前以「建立新頁面」為主，欄位對應/同步到資料庫欄位的完整 mapping 仍在調整中。

### 更新已發布內容

目前不支援「同步更新既有 Notion 頁面」；如需更新請重新發佈（建立新頁面），或在 Notion 端手動調整。

## Google Docs 整合

### 連接 Google Docs

1. 進入「設定」>「整合」
2. 找到「Google Docs」卡片
3. 點擊「連接」
4. 選擇 Google 帳號並授權

### 權限說明

| 權限 | 用途 |
|-----|------|
| 查看和管理 Google Drive 中的文件 | 建立和編輯 Google 文件 |
| 查看 Google Drive 資料夾 | 選擇發布位置 |

### 發布到 Google Docs

1. 在 Otlex 編輯器中完成文章
2. 點擊「發布」>「Google Docs」
3. 選擇目標資料夾
4. 輸入文件名稱（選用）
5. 點擊「發布」

### 格式保留

目前的 Google Docs 發佈以「純文字寫入」為主（會把 HTML/Markdown 轉成文字內容寫入文件），不保證保留標題、列表、圖片等富文本格式。

## WordPress 整合

### 連接 WordPress

WordPress 發佈目前以「文章發佈面板」為主：

1. 進入文章編輯器，點擊「發佈」
2. 選擇 WordPress
3. 輸入：
   - 站點網址（`https://your-site.com`）
   - 使用者名稱
   - Application Password
4. 選擇發佈狀態並送出

### 建立應用程式密碼

在 WordPress 後台建立應用程式密碼：

1. 登入 WordPress 後台
2. 進入「使用者」>「個人資料」
3. 滾動到「應用程式密碼」區塊
4. 輸入名稱（如「Otlex」）
5. 點擊「新增應用程式密碼」
6. 複製生成的密碼（只會顯示一次！）

<Callout type="warning">
  應用程式密碼只會顯示一次，請立即複製。如果忘記，需要重新建立一個。
</Callout>

### 發布到 WordPress

1. 在 Otlex 編輯器中完成文章
2. 點擊「發布」>「WordPress」
3. 設定發布選項：

| 選項 | 說明 |
|-----|------|
| **發布狀態** | draft / publish / pending |

4. 點擊「發布」

### 發布狀態選項

| 狀態 | 說明 |
|-----|------|
| **草稿** | 儲存為草稿，不公開 |
| **待審閱** | 提交審閱，等待核准 |
| **立即發布** | 直接發布為公開文章 |

> 目前不支援：排程發布、分類/標籤、特色圖片、SEO 外掛欄位（Yoast/Rank Math 等）。

## Shopify 整合

### 連接 Shopify

Shopify 發佈目前以「文章發佈面板」為主：

1. 進入文章編輯器，點擊「發佈」
2. 選擇 Shopify
3. 輸入：
   - Store Domain（例如 `your-store.myshopify.com`）
   - Admin API Access Token
   - Blog ID
4. 選擇狀態（draft / published）並送出

### 發布到 Shopify

1. 在 Otlex 編輯器中完成文章
2. 點擊「發布」>「Shopify」
3. 設定發布選項（狀態、tags 等）
4. 點擊「發布」

### Shopify 限制

| 功能 | 支援狀態 |
|-----|---------|
| 部落格文章（Blog Article） | ✅ 支援 |
| 產品描述 / 一般 Pages | ❌（目前不在發佈範圍） |
| 複雜格式 | ⚠️ 依 Shopify editor 處理結果而定 |

## 媒體處理

### 圖片上傳

目前 CMS 發佈以「文字/HTML 內容」為主，不會自動把圖片上傳到目標 CMS，也不會自動壓縮圖片。若你的內容包含圖片，建議：

- 在目標 CMS 內手動上傳圖片並替換連結
- 或改用可公開存取的圖片 URL

## 發布記錄

### 查看發布記錄

目前尚未提供獨立的「發布記錄」頁面。你可以在文章的「發佈」面板看到最近一次發佈結果（若有）。

### 記錄內容

（若已顯示）通常會包含：目標平台、發佈結果連結、成功/失敗訊息。

### 重新發布

若發佈失敗，請先修正憑證/權限/格式後再重新嘗試發佈。

## 疑難排解

### 發布失敗怎麼辦？

常見原因和解決方式：

| 原因 | 解決方式 |
|-----|---------|
| 憑證錯誤/過期 | 重新設定憑證或重新授權（Google Docs） |
| 權限不足 | 確認 CMS 帳號/Integration 有對應權限 |
| 網路問題 | 稍後重試 |
| 內容太長/格式不相容 | 先用臨時匯出（Markdown/HTML）再貼到 CMS 端調整 |

### WordPress 連接失敗？

檢查項目：

1. 網站網址是否正確（含 https://）
2. 應用程式密碼是否正確
3. 網站是否允許 REST API
4. 是否有安全外掛阻擋

### Notion 找不到頁面？

請確認目標 Page/Database 已「分享」給你使用的 Notion Integration（Internal Integration）。如果沒有分享，Notion API 會拒絕存取或建立內容。

## 常見問題

### 可以同時連接多個 CMS 嗎？

可以。每種 CMS 都可以獨立連接，發布時選擇目標平台。

### 發布後在 Otlex 編輯會同步嗎？

目前不支援自動同步或「更新已發布內容」。若內容有更新，請重新發佈（建立新內容），或在 CMS 端手動修改。

### 可以排程發布嗎？

目前不支援從 Otlex 端直接排程發布。若需要排程，請使用各 CMS 平台自身的排程功能，或透過外部工具（Zapier/n8n）搭配 CMS Publishing API 建立自動化流程。

### CMS 上刪除會影響 Otlex 嗎？

不會。Otlex 的內容和 CMS 是獨立的，在 CMS 上刪除不會影響 Otlex 中的內容。
