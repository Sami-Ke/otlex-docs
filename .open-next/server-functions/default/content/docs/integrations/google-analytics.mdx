## 概覽

Google Analytics 4 (GA4) 整合讓你在 Otlex 中查看網站的流量和用戶行為數據，與 GSC 的搜尋數據形成互補。

### 連接後可獲得的數據

| 數據類型 | 說明 | 用途 |
|---------|------|------|
| **網站流量** | 訪客數、工作階段數 | 了解整體流量規模 |
| **流量來源** | 自然搜尋、直接、社群等 | 分析流量組成 |
| **頁面表現** | 瀏覽量、停留時間、跳出率 | 評估內容品質 |
| **用戶特徵** | 地區、裝置、瀏覽器 | 了解目標受眾 |
| **轉換數據** | 目標完成數、轉換率 | 衡量商業價值 |

### GA4 與 GSC 的區別

| 維度 | Google Search Console | Google Analytics 4 |
|-----|----------------------|-------------------|
| 數據來源 | Google 搜尋結果 | 網站追蹤碼 |
| 涵蓋範圍 | 僅 Google 搜尋流量 | 所有流量來源 |
| 主要指標 | 曝光、點擊、排名、CTR | 使用者、工作階段、跳出率 |
| 數據延遲 | 2-3 天 | 24-48 小時 |
| 用途 | SEO 表現分析 | 整體流量與用戶行為 |

兩者結合使用可以獲得最完整的網站分析視角。

## 連接前準備

### 確認你有 GA4 資源

1. 前往 [Google Analytics](https://analytics.google.com)
2. 確認你有 GA4 資源（非舊版 Universal Analytics）
3. 確認網站已正確安裝追蹤碼

<Callout type="warning">
  Otlex 只支援 **GA4**，不支援舊版 Universal Analytics (UA)。如果你還在使用 UA，請先升級到 GA4。
</Callout>

### 權限要求

你的 Google 帳號需要有該 GA4 資源的：
- **檢視者（Viewer）**權限以上 ✅
- **無權限** ❌ 無法連接

## 連接步驟

### 步驟 1：開始連接

1. 登入 Otlex
2. 進入「設定」>「整合」
3. 找到「Google Analytics」卡片
4. 點擊「連接」

### 步驟 2：選擇 Google 帳號

1. 彈出 Google 登入視窗
2. 選擇擁有 GA4 權限的 Google 帳號
3. 確認選擇正確的帳號

### 步驟 3：授權權限

1. 查看 Otlex 要求的權限說明
2. 點擊「允許」授予權限

**權限說明**：

| 權限 | 用途 |
|-----|------|
| 查看 Analytics 數據 | 獲取流量和用戶行為數據 |
| 查看帳戶和資源列表 | 顯示可連接的資源 |

<Callout type="info">
  Otlex 只需要**唯讀**權限，不會修改任何 GA4 設定或追蹤碼。
</Callout>

### 步驟 4：選擇 GA4 資源

授權後會顯示你有權限的 GA4 帳戶和資源：

1. 選擇帳戶
2. 選擇資源
3. 選擇資料串流（通常是網站串流）

確保選擇的資源對應你的品牌網站。

### 步驟 5：確認連接

1. 點擊「確認」完成連接
2. 系統開始同步歷史數據
3. 顯示「連接成功」訊息

## 數據同步

### 首次同步

連接成功後，系統會同步歷史數據：

| 數據類型 | 同步範圍 | 預計時間 |
|---------|---------|---------|
| 流量概覽 | 近期區間（預設最近 14 天） | 依站點大小而定（通常數分鐘內可看到資料） |
| 流量來源 | 近期區間（預設最近 14 天） | 依站點大小而定（通常數分鐘內可看到資料） |
| 頁面數據 | 近期區間（預設最近 14 天） | 依站點大小而定（通常數分鐘內可看到資料） |
| 用戶特徵 | 近期區間（預設最近 14 天） | 依站點大小而定（通常數分鐘內可看到資料） |

### 持續同步

你可以在「設定 > 整合」點擊同步按鈕，Otlex 會將同步工作排程到背景任務中執行（通常幾分鐘內完成）。

<Callout type="info">
  部分環境/權限可能會啟用「補齊歷史（Backfill）」機制：在背景逐步往更早的日期補資料。實際是否啟用與補齊範圍，以 App 顯示為準。
</Callout>

## 查看數據

### Dashboard 流量卡片

連接後，Dashboard 會新增流量區塊：

| 指標 | 說明 |
|-----|------|
| **使用者** | 不重複訪客數 |
| **工作階段** | 總瀏覽工作階段數 |
| **跳出率** | 只瀏覽一頁就離開的比例 |
| **平均停留時間** | 每次工作階段的平均時長 |

### 流量來源分析

查看流量從哪裡來：

| 來源類型 | 說明 |
|---------|------|
| **Organic Search** | 自然搜尋流量（來自 Google、Bing 等） |
| **Direct** | 直接輸入網址或書籤 |
| **Referral** | 從其他網站連結點入 |
| **Social** | 來自社群媒體 |
| **Paid Search** | 付費搜尋廣告 |
| **Email** | 電子郵件中的連結 |

### 頁面表現數據

在頁面分析中，結合 GA4 數據顯示：

| 指標 | 來源 | 說明 |
|-----|------|------|
| 瀏覽量 | GA4 | 頁面被瀏覽的次數 |
| 不重複瀏覽量 | GA4 | 不重複的頁面瀏覽數 |
| 平均停留時間 | GA4 | 訪客在該頁面停留的時間 |
| 跳出率 | GA4 | 該頁面的跳出比例 |
| 點擊數 | GSC | 從搜尋結果點入的次數 |
| 排名 | GSC | 在搜尋結果中的平均排名 |

### SEO 與流量關聯分析

Otlex 會自動分析 SEO 與流量的關聯：

| 分析類型 | 說明 |
|---------|------|
| **排名 vs 流量** | 排名變化對流量的影響 |
| **高排名低流量** | 排名好但流量不如預期的頁面 |
| **流量來源占比** | 自然搜尋佔總流量的比例 |
| **內容價值評估** | 根據停留時間和跳出率評估內容品質 |

## GA4 數據延遲說明

GA4 數據通常有 **24-48 小時延遲**：

- 標準報表：約 24 小時
- 部分指標：可能需要 48 小時
- 即時報表：不在同步範圍內

這是 GA4 的正常處理時間。

## 與 GSC 數據比較

### 為什麼數據不一致？

GSC 和 GA4 的數據**不會完全相符**，這是正常的：

| 差異原因 | 說明 |
|---------|------|
| **計算方式不同** | GSC 計算點擊；GA4 計算工作階段 |
| **追蹤機制不同** | GSC 是伺服器端；GA4 需要 JavaScript |
| **涵蓋範圍不同** | GSC 只有 Google 搜尋；GA4 包含所有來源 |
| **AdBlock 影響** | GA4 可能被 AdBlock 阻擋 |
| **時區不同** | 報表時區設定可能不同 |

### 如何解讀

- **趨勢比對**：兩者趨勢應該一致（都上升或下降）
- **絕對數字**：不要比較絕對數字，看比例和趨勢
- **GSC 更準確**：對於自然搜尋流量，GSC 數據更可靠

## 疑難排解

### 看不到我的 GA4 資源？

可能原因：

| 原因 | 解決方式 |
|-----|---------|
| 帳號權限不足 | 確認有該 GA4 資源的存取權限 |
| 是 UA 而非 GA4 | Otlex 不支援 Universal Analytics |
| 選錯 Google 帳號 | 確認使用正確的 Google 帳號 |

### 數據與 GA4 後台不同？

可能原因：

- **時區設定不同**：確認 Otlex 和 GA4 時區設定
- **採樣差異**：大型網站可能有採樣
- **同步時間差**：數據可能還在更新中

### 如何切換到不同的 GA4 資源？

1. 進入「設定」>「整合」
2. 找到 Google Analytics 整合
3. 點擊「中斷連接」
4. 重新連接並選擇新資源

## 常見問題

### GA4 整合是必要的嗎？

不是必要的，但強烈建議。GA4 整合可以讓你：
- 了解完整的流量圖像
- 分析用戶行為和內容品質
- 結合 SEO 和業務數據

### 沒有 GA4 怎麼辦？

如果你的網站還沒有設定 GA4：
1. 前往 Google Analytics 建立 GA4 資源
2. 安裝追蹤碼到網站
3. 等待數據開始收集（約 24-48 小時）
4. 回到 Otlex 連接 GA4

### 可以只連接 GA4 不連接 GSC 嗎？

可以，但不建議。GSC 提供的搜尋數據是 SEO 的核心，沒有 GSC 會失去大部分 SEO 分析功能。

### 連接會影響我的 GA4 數據嗎？

不會。Otlex 只有唯讀權限，不會修改任何 GA4 設定或影響數據收集。
