## 帳號架構說明

Otlex 使用三層組織架構：

```
帳號（User Account）
  └── 組織（Organization）
        ├── 品牌 A（Brand）
        ├── 品牌 B（Brand）
        └── 團隊成員（Members）
```

### 各層級說明

| 層級 | 說明 | 管理內容 |
|-----|------|---------|
| **帳號** | 你的個人登入帳號 | 個人資料、密碼、登入方式 |
| **組織** | 公司或團隊的最高層級 | 訂閱方案、計費、成員管理 |
| **品牌** | 代表一個網站/品牌 | SEO 數據、內容、整合設定 |

## 帳號管理

你可以在 App 的「帳號設定」頁管理個人資料（例如顯示名稱、頭像等）。可用項目會依部署環境與權限而不同，以實際畫面為準。

<Callout type="info">
  若你找不到某些設定（例如通知/安全性相關），代表目前環境尚未開放或由管理員集中管理。
</Callout>

## 組織管理

### 成員管理

管理組織內的團隊成員：

| 功能 | 說明 |
|-----|------|
| 邀請成員 | 透過 Email 邀請新成員 |
| 設定角色 | 指派 Owner/Admin/Member/Viewer 角色 |
| 調整權限 | 變更現有成員的角色 |
| 移除成員 | 從組織移除成員 |

詳細說明請參考 [邀請團隊成員](/docs/account-billing/invite-members)。

## 訂閱與計費

### 查看目前方案

1. 進入「設定」>「方案與計費」
2. 查看目前訂閱狀態：

| 資訊 | 說明 |
|-----|------|
| 目前方案 | Starter / Pro / Business / Enterprise |
| 計費週期 | 依 App 顯示為準 |
| 下次扣款日 | 下一次自動扣款的日期 |
| 用量統計 | SEO 稽核次數、AI 生成點數使用情況 |

### 用量監控

即時追蹤各項功能的使用量：

| 項目 | 顯示內容 |
|-----|---------|
| SEO 稽核 | 已使用 / 總配額 |
| AI 生成點數 | 已使用 / 總配額 |
| 品牌數 | 已建立 / 上限 |
| 成員數 | 已加入 / 上限 |
| 資料保留 | 歷史數據保留天數 |

用量接近上限時是否會出現提醒，會依你的環境與權限設定而定；以 App 顯示為準。

### 升級與降級

- **升級/降級**：目前透過 Stripe 結帳頁進行（計費週期與生效時間以 Stripe 顯示為準）。
- **取消訂閱**：目前尚未在 App 內完整開放，請聯絡管理員協助處理。

詳細說明請參考 [管理訂閱](/docs/account-billing/manage-subscription)。

## 付款方式

付款方式與可用卡別以 Stripe 結帳頁顯示為準。若你需要發票/請款/合約協助，請聯絡管理員。

## 資料與隱私

### 資料保留

資料保留規則會依你的方案、合約與權限而不同；以 App 顯示與管理員提供的資訊為準。

### 資料匯出

你隨時可以匯出你的資料：

- SEO 稽核報告
- 關鍵字追蹤數據
- AI 生成的內容
- 帳號設定

### 帳號刪除

目前尚未提供自助刪除帳號的入口。如需刪除帳號或資料，請聯絡管理員協助處理（會依公司/合約政策確認資料保留與刪除範圍）。

## 常見問題

### 忘記密碼怎麼辦？

1. 在登入頁點擊「忘記密碼」
2. 輸入註冊 Email
3. 收取重設密碼信件
4. 設定新密碼

### 需要協助怎麼辦？

若你遇到「沒有看到某個按鈕/頁面」或需要進行計費、發票、成員權限調整等操作，請聯絡管理員協助（不同環境的開放範圍可能不同）。
