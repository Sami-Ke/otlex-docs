## 什麼是內容地圖？

內容地圖（Content Map）是 Otlex 的核心分析工具，提供多個視角深入理解網站的內容架構：

- **看清全局**：網站所有頁面的關係和結構
- **發現問題**：找出孤島頁面、內連不足、流量機會
- **整合 GSC 數據**：在地圖中直接查看點擊、曝光、排名
- **產生行動建議**：直接得到可執行的內連建議

### 為什麼內容架構對 SEO 很重要？

| 架構健康 | 影響 |
|----------|------|
| 清晰的內部連結 | 搜尋引擎更容易發現並理解所有頁面 |
| 主題群集（Topic Cluster）| 強化特定領域的搜尋引擎權威性 |
| 合理的連結深度 | 確保深層頁面不被搜尋引擎忽略 |
| 無孤島頁面 | 所有重要頁面都有入口，不浪費爬取預算 |

## 使用前準備

內容地圖需要以下資料來源：

### 1. 網站爬取（必要）

爬取資料用於建立：
- 所有頁面的清單
- 頁面之間的內部連結關係
- 各頁面的內容摘要

**如何啟動爬取？**

若尚未有爬取資料，內容地圖頂部會顯示提示。點擊「啟動爬取」選擇爬取模式：

| 模式 | 說明 | 適合情況 |
|------|------|----------|
| **首頁快速分析** | 從首頁出發，爬取主要鏈接頁面 | 快速了解主要結構 |
| **sitemap 智慧發現** | 讀取 sitemap.xml 中列出的頁面 | 有完整 sitemap 的網站 |
| **整站深度爬取** | 遞迴爬取所有可達頁面 | 需要完整頁面庫 |

### 2. Google Search Console（建議）

GSC 資料用於：
- 各頁面的點擊數、曝光數、CTR、排名
- 著陸關鍵字分析（哪些搜尋詞帶流量到哪個頁面）

**如何同步 GSC？**

若已完成 GSC 授權，內容地圖會提供「同步 GSC」按鈕。點擊後任務排程到背景執行，通常幾分鐘內完成。

<Callout type="info">
  未連接 GSC 時，仍可使用內容地圖查看頁面結構與連結關係，但無法看到搜尋流量和排名相關數據。
</Callout>

## 六大視圖詳解

### 視圖 1：健檢（Health）

快速找出需要優先處理的「問題頁面」，以機會清單形式呈現。

#### 常見健檢問題類型

| 問題類型 | 判斷條件 | 建議行動 |
|----------|----------|----------|
| **孤島頁面** | 沒有任何內部連結指向的頁面 | 從相關頁面加入指向它的連結 |
| **標題優化機會** | 高曝光但 CTR 低於預期 | 改善 Title Tag 和 Meta Description |
| **首頁機會** | 排名在 11-20 名的關鍵字/頁面 | 優化頁面內容和內連，衝上首頁 |
| **內連不足** | 有流量但入站內連數量不足 | 增加指向這些頁面的內部連結 |

#### 自訂門檻

點擊「調整門檻」可修改各問題的判斷條件：
- 孤島頁面：最低入站連結數
- CTR 門檻：判斷「低 CTR」的百分比
- 排名門檻：什麼排名算「接近首頁」

這讓你可以根據網站規模和自身標準來調整什麼叫做「需要優化」。

---

### 視圖 2：頁面（Focus Page）

聚焦分析**單一頁面**的完整狀況。

#### 頁面詳情包含

**指標卡片（需 GSC 資料）**

| 指標 | 說明 |
|------|------|
| 點擊數 | 從 Google 搜尋點入的次數（最近 N 天） |
| 曝光數 | 在搜尋結果出現的次數 |
| CTR | 點擊 / 曝光的比率 |
| 品牌相關度 | 頁面與品牌核心主題的相關程度 |

**連結關係圖**

以互動式圖形顯示：
- **入站連結（Inbound）**：哪些頁面連到這裡
- **出站連結（Outbound）**：這個頁面連到哪裡

滑過節點可看到頁面標題，點擊可跳轉到該頁面的詳情。

**關鍵字資訊**

- AI 分析的頁面主題關鍵字
- GSC 帶來流量的搜尋詞
- 快速跳轉到「著陸關鍵字」視圖

---

### 視圖 3：著陸關鍵字（Page Landing）

以「**單一頁面**」為中心，深入分析哪些搜尋詞帶來流量。

#### 使用場景

- 確認某頁面的主要搜尋意圖是什麼
- 找出頁面排名表現好的意外關鍵字
- 發現可以進一步強化的「接近首頁」關鍵字

#### 資料維度

| 維度 | 說明 |
|------|------|
| 搜尋詞 | 使用者實際輸入的查詢 |
| 點擊數 | 這個搜尋詞帶來多少點擊 |
| 曝光數 | 這個搜尋詞有多少曝光 |
| CTR | 點擊率 |
| 平均排名 | 在搜尋結果的平均位置 |

<Callout type="info">
  著陸關鍵字視圖需要有 GSC 資料。建議先完成 GSC 連接並同步至少 14 天的資料，才能看到有意義的分析結果。
</Callout>

---

### 視圖 4：關鍵字（Keyword Hub）

以「**關鍵字**」為中心，了解同一個關鍵字由哪些頁面承接流量。

#### 解決什麼問題？

常見問題：**關鍵字蠶食（Keyword Cannibalization）** — 多個頁面同時競爭同一個關鍵字，導致互相干擾、沒有一個能排到最好。

Keyword Hub 幫助你：

| 需求 | 如何使用 |
|------|----------|
| 找出哪些頁面在搶相同關鍵字 | 搜尋該關鍵字，查看對應頁面清單 |
| 決定哪個頁面應該「主導」這個關鍵字 | 比較各頁面的排名和流量，選出最強的 |
| 找出內容缺口 | 看哪些重要關鍵字完全沒有對應頁面 |

---

### 視圖 5：地圖（Graph）

用互動式**網路圖**視覺化整個網站的連結結構。

#### 三種顯示模式

**頁面模式（Pages）**

以頁面為節點，連線代表內部連結：
- 節點大小 = 被連結次數（越大越重要）
- 可識別「中心頁面」與孤立的邊緣頁面
- 幫助看出 Hub and Spoke 架構是否清晰

**問題模式（Questions）**

以「問題/問答」為節點：
- 顯示網站能回答哪些問題
- 問答之間的關聯性
- 特別適合評估 AEO（AI Engine Optimization）的覆蓋度

**關鍵字模式（Keywords）**

以關鍵字為節點：
- 顯示網站涵蓋的關鍵字生態
- 關鍵字群集之間的距離
- 找出未被充分連結的關鍵字群

#### 地圖操作

| 操作 | 說明 |
|------|------|
| 縮放 | 滾輪或觸控板雙指縮放 |
| 拖移 | 點擊空白處拖動視角 |
| 搜尋 | 輸入頁面標題/URL 快速定位 |
| 篩選 | 連線類型、品牌相關度、孤島節點 |
| 點擊節點 | 顯示該節點的詳細資訊 |

#### 匯出地圖

| 格式 | 用途 |
|------|------|
| **JSON** | 程式化分析、存入其他系統 |
| **CSV** | 試算表分析 |
| **SVG** | 高品質向量圖，用於報告或簡報 |

---

### 視圖 6：列表（List）

以清單形式整理兩種重要資料。

#### 問答列表（Q&A List）

從已爬取的頁面中萃取的問題與答案：

| 欄位 | 說明 |
|------|------|
| 問題 | 頁面中識別到的問題 |
| 答案摘要 | 頁面給出的答案概要 |
| 答案品質評分 | AI 評估答案的完整性和清晰度 |
| 品牌相關度 | 這個問答與品牌核心主題的關聯程度 |
| 來源頁面 | 問答出自哪個 URL |

**使用情境：**
- 評估現有內容的問答結構質量
- 找出答案品質差、需要改善的頁面
- 規劃 FAQ Schema 的優先頁面

#### 內連建議（Internal Link Suggestions）

系統根據頁面主題相關性生成的內部連結建議清單：

| 欄位 | 說明 |
|------|------|
| 來源頁面 | 需要在這個頁面加入連結 |
| 目標頁面 | 連結指向這個頁面 |
| 信心分數 | AI 對此建議的確信程度（0-100%）|
| SEO 價值 | 預估的排名影響（高/中/低）|

<Callout type="info">
  內連建議清單與「工具 > 內部連結」是同步的，在任一頁面標記的處理狀態會互相反映。
</Callout>

## 常見使用場景

### 場景一：新網站的內容架構規劃

1. 完成首次爬取，查看地圖視圖（頁面模式）
2. 確認主要頁面是否形成清晰的主題群集
3. 找出孤島頁面，加入首頁或主要分類的連結
4. 使用內連建議，強化重要頁面的內連入站

### 場景二：已有流量的網站優化

1. 連接 GSC 同步搜尋數據
2. 進入「健檢」視圖，找出首頁機會（排名 11-20）
3. 對這些頁面：
   - 從關鍵字視圖確認沒有關鍵字蠶食問題
   - 用內連建議增加更多高流量頁面的連結指向它
   - 回到 SEO 審查確認技術和內容面無問題

### 場景三：發佈新文章後的收尾

1. 等爬蟲更新後（或手動觸發爬取），在地圖中找到新文章
2. 確認新文章不是孤島頁面（至少有 2-3 個入站連結）
3. 使用內連建議找出適合加連結的舊文章
4. 確認新文章有連結到相關的舊文章（出站連結）

| 資料類型 | 何時更新 |
|----------|----------|
| 頁面清單與連結關係 | 每次爬蟲完成後 |

| AI 問答分析 | 爬蟲後重新分析 |

## 常見問題

### 內容地圖是空的？

最可能的原因是尚未完成爬取。請點擊頁面頂部的「啟動爬取」按鈕，選擇合適的爬取模式並等待完成。

### 頁面數量不對/有缺漏？

可能原因：
- robots.txt 封鎖了某些頁面（爬蟲無法存取）
- 使用 sitemap 模式但 sitemap 不完整
- 頁面需要登入才能存取

解決方式：切換爬取模式（例如改用「整站深度爬取」），或確認 robots.txt 設定。

### GSC 數據顯示「無資料」？

確認以下事項：
1. 品牌的「設定 > 整合」中 GSC 已連接
2. 已點擊「同步 GSC」且同步任務已完成
3. GSC 本身有至少幾天的資料（新網站需要時間累積）
