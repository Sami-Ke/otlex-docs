## 什麼是內容地圖？

內容地圖（Content Map）是 Otlex 的視覺化工具，幫助你理解網站的內容架構和頁面之間的關係。一個健康的內容架構是 SEO 成功的基礎。

### 為什麼內容架構很重要？

| 重要性 | 說明 |
|--------|------|
| **搜尋引擎爬取** | 清晰的結構讓 Googlebot 更容易發現和理解所有頁面 |
| **權重傳遞** | 內部連結傳遞 PageRank，合理的連結結構提升重要頁面權重 |
| **使用者體驗** | 良好的導覽結構幫助訪客找到相關內容 |
| **主題權威** | 主題群集（Topic Cluster）策略強化你在特定領域的權威性 |

### 內容地圖能幫助你

- 🔍 **視覺化連結關係**：一眼看出頁面之間的連結網絡
- 🏝️ **找出孤島頁面**：發現沒有被連結到的頁面
- 🎯 **優化內部連結**：識別連結不足或過度連結的頁面
- 📊 **分析主題群集**：檢視主題之間的關聯強度
- 📈 **追蹤結構變化**：隨時間監控內容架構的演變

## 三種視圖模式

### Page View（頁面視圖）

以個別頁面為節點，展示頁面之間的內部連結關係。

#### 節點說明

| 元素 | 意義 |
|-----|------|
| **節點大小** | 頁面重要性（被連結次數） |
| **節點顏色** | 頁面類型（文章/產品/分類等） |
| **連線粗細** | 連結強度（單向/雙向） |
| **連線方向** | 連結流向（箭頭指向被連結頁面） |

#### 互動操作

- **拖曳**：移動畫布探索不同區域
- **縮放**：滾輪放大縮小檢視層級
- **點擊節點**：查看頁面詳細資訊
- **懸停**：顯示該頁面的連結摘要

#### 頁面詳情面板

點擊任一頁面節點會顯示：

| 資訊 | 說明 |
|-----|------|
| 頁面標題 | 頁面的 Title 標籤 |
| URL | 頁面網址 |
| 連入連結數 | 有多少頁面連到此頁 |
| 連出連結數 | 此頁連到多少其他頁面 |
| 頁面類型 | Article/Product/Listing 等 |
| SEO 分數 | 最近一次稽核的分數 |
| 快速操作 | 執行稽核、編輯、查看 |

### Topic View（主題視圖）

以主題群集為中心，展示相關頁面如何圍繞核心主題組織。

#### 什麼是主題群集？

主題群集（Topic Cluster）是一種 SEO 內容策略：

```
                    支柱頁面
                   (Pillar Page)
                        │
        ┌───────────────┼───────────────┐
        │               │               │
    子主題 A         子主題 B         子主題 C
    (Cluster)       (Cluster)       (Cluster)
        │               │               │
    ┌───┼───┐       ┌───┼───┐       ┌───┼───┐
    │   │   │       │   │   │       │   │   │
   文章 文章 文章   文章 文章 文章   文章 文章 文章
```

#### 視圖說明

| 元素 | 意義 |
|-----|------|
| **中心大圓** | 支柱頁面（Pillar Page） |
| **周圍節點** | 相關的群集內容（Cluster Content） |
| **群集顏色** | 不同主題用不同顏色區分 |
| **連線** | 頁面之間的實際連結 |

#### 群集健康度指標

每個主題群集會顯示健康度：

| 指標 | 說明 |
|-----|------|
| 🟢 **健康** | 支柱頁面與群集內容雙向連結完整 |
| 🟡 **需改善** | 部分頁面缺少連結 |
| 🔴 **有問題** | 群集結構不完整或孤立 |

### Keyword View（關鍵字視圖）

以目標關鍵字為核心，展示著陸頁面（Landing Page）與關鍵字的對應關係。

#### 視圖說明

| 元素 | 意義 |
|-----|------|
| **關鍵字節點** | 你設定的目標關鍵字 |
| **頁面節點** | 目標排名的著陸頁面 |
| **連線** | 頁面對關鍵字的優化狀態 |
| **節點大小** | 關鍵字的搜尋量 |

#### 關鍵字-頁面對應狀態

| 狀態 | 說明 |
|-----|------|
| ✅ **已對應** | 有明確的著陸頁面 |
| ⚠️ **競爭中** | 多個頁面爭奪同一關鍵字（Keyword Cannibalization） |
| ❌ **未對應** | 關鍵字沒有對應頁面 |

## 如何使用內容地圖

### 基本操作

1. 進入左側選單「內容地圖」
2. 系統會自動載入你的網站結構
3. 預設顯示 Page View
4. 使用上方切換按鈕切換視圖

### 篩選與搜尋

| 功能 | 說明 |
|-----|------|
| **搜尋頁面** | 輸入關鍵字快速定位頁面 |
| **篩選類型** | 只顯示特定類型頁面 |
| **篩選連結數** | 顯示連結數在特定範圍的頁面 |
| **隱藏孤島** | 暫時隱藏無連結頁面 |

### 匯出與分享

- **匯出圖片**：下載 PNG 格式的地圖截圖
- **匯出數據**：下載 CSV 格式的連結數據
- **分享連結**：生成可分享的唯讀連結

## 重要功能

### 孤島頁面偵測

孤島頁面（Orphan Pages）是沒有被任何其他頁面連結到的頁面。這些頁面：

- 搜尋引擎難以發現
- 不會獲得內部連結權重
- 可能被認為不重要

#### 處理孤島頁面

| 處理方式 | 適用情況 |
|---------|---------|
| **新增內部連結** | 頁面有價值，需要被連結 |
| **加入導航** | 重要頁面應出現在選單或頁尾 |
| **合併內容** | 內容過少，可合併到其他頁面 |
| **noindex** | 不需要被索引的頁面 |
| **刪除** | 真的不需要的頁面 |

### 連結缺口分析

系統會自動分析並標示：

| 問題類型 | 說明 | 建議行動 |
|---------|------|---------|
| **連入過少** | 重要頁面但很少被連結 | 從相關頁面新增連結 |
| **連出過少** | 頁面沒有連到其他頁面 | 新增相關內容連結 |
| **單向連結** | 只有單向連結，沒有回連 | 建立雙向連結 |
| **過度連結** | 同一頁面過多連結 | 精簡連結數量 |

### 連結建議

基於內容分析，AI 會建議：

- 哪些頁面之間應該建立連結
- 連結錨點文字建議
- 連結位置建議（內文/sidebar/footer）

## 主題群集建立

### 自動偵測

系統會根據以下因素自動識別主題群集：

- URL 結構
- 頁面內容相似度
- 現有內部連結
- 共同關鍵字

### 手動調整

你可以：

1. 新增主題群集
2. 指定支柱頁面
3. 拖曳頁面到群集
4. 調整群集名稱

### 建立新群集

1. 點擊「新增主題」
2. 輸入主題名稱
3. 選擇或建立支柱頁面
4. 選取相關的群集內容
5. 系統會提示缺少的連結

## 與其他功能整合

### 與 SEO 稽核整合

- 稽核結果中的內部連結建議會反映在地圖上
- 點擊地圖節點可直接執行稽核

### 與 AI 內容生成整合

- 生成文章時自動建議內部連結
- 連結建議基於內容地圖的分析結果

### 與 GSC 數據整合

如果已連接 Google Search Console：

- 節點會顯示實際流量
- 可篩選高/低流量頁面
- 識別高價值但連結不足的頁面

## 資料更新

### 自動掃描

| 方案 | 掃描頻率 |
|-----|---------|
| Starter | 每週一次 |
| Pro | 每三天一次 |
| Business | 每日一次 |
| Enterprise | 即時更新 |

### 手動觸發

點擊「重新掃描」可立即更新網站結構。

掃描時間視網站大小而定：
- 100 頁以下：約 1-2 分鐘
- 100-500 頁：約 5-10 分鐘
- 500+ 頁：約 15-30 分鐘

## 最佳實踐

### 理想的內部連結結構

| 原則 | 說明 |
|-----|------|
| **淺層架構** | 任何頁面都應在 3 次點擊內可達 |
| **相關性連結** | 連結應基於內容相關性，非隨機 |
| **錨點文字多樣** | 避免重複使用相同錨點文字 |
| **雙向連結** | 相關頁面之間建立雙向連結 |
| **定期審查** | 每月檢視並更新連結策略 |

### 每週檢查清單

- [ ] 檢查新增的孤島頁面
- [ ] 處理連結缺口警告
- [ ] 確認主題群集完整性
- [ ] 審查 AI 建議的連結
- [ ] 監控重要頁面的連入連結數

## 常見問題

### 掃描沒有找到所有頁面？

可能原因：
- 頁面需要登入才能訪問
- robots.txt 封鎖了爬蟲
- 頁面沒有在 sitemap 中
- JavaScript 渲染的內容

解決方式：
- 檢查 robots.txt 設定
- 更新 sitemap.xml
- 確認頁面可公開訪問

### 內容地圖和 SEO 稽核的連結建議不同？

兩者分析角度不同：
- **SEO 稽核**：針對單一頁面的最佳化
- **內容地圖**：從整站架構角度分析

建議優先處理兩者都提出的建議。

### 可以匯入現有的 sitemap 嗎？

可以。在設定中選擇「匯入 sitemap」，系統會根據 sitemap 補充掃描範圍。
