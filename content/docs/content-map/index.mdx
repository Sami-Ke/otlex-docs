## 什麼是內容地圖？

內容地圖（Content Map）是 Otlex 的視覺化工具，幫助你理解網站的內容架構和頁面之間的關係。一個健康的內容架構是 SEO 成功的基礎。

### 為什麼內容架構很重要？

| 重要性 | 說明 |
|--------|------|
| **搜尋引擎爬取** | 清晰的結構讓 Googlebot 更容易發現和理解所有頁面 |
| **權重傳遞** | 內部連結傳遞 PageRank，合理的連結結構提升重要頁面權重 |
| **使用者體驗** | 良好的導覽結構幫助訪客找到相關內容 |
| **主題權威** | 主題群集（Topic Cluster）策略強化你在特定領域的權威性 |

### 內容地圖能幫助你

- 🔍 **視覺化連結關係**：一眼看出頁面之間的連結網絡
- 🏝️ **找出孤島頁面**：發現沒有被連結到的頁面
- 🎯 **優化內部連結**：找出內連不足的高價值頁面，並查看內連建議
- 📊 **用多視角看內容**：用頁面 / 問題（問答）/ 關鍵字的方式理解內容結構
- 📈 **搭配 GSC 指標**：在內容地圖內查看點擊/曝光/CTR/排名與著陸關鍵字分佈

## 使用前準備（很重要）

內容地圖的資料主要來自兩個來源：

1. **網站爬取資料**：用來建立頁面清單、內部連結關係與內容片段（你可以在內容地圖內直接啟動爬取）。
2. **Google Search Console（GSC）**：用來提供點擊/曝光/CTR/排名等指標與「著陸關鍵字」分析。

如果你尚未完成爬取或尚未連結 GSC，系統會在內容地圖上方顯示提示，並提供快速前往整合設定或啟動爬取的入口。

## 六大視圖（App 內可切換）

### 健檢（Health）

用「機會清單」快速找出需要優先處理的項目，常見包含：

- **孤島頁面**：沒有內連指向的頁面（Orphan Pages）
- **標題優化**：高曝光、低 CTR 的頁面（多半是標題/描述可改善）
- **首頁機會**：排名接近首頁（例如 11–20）的關鍵字/頁面
- **內連不足**：有流量但內連數不足的頁面

你可以在健檢頁面切換篩選類型，並用「調整門檻」修改判斷條件。

### 頁面（Focus Page）

聚焦單一頁面，提供：

- 該頁面的 **點擊/曝光/CTR/品牌相關度** 指標卡
- 以圖形方式呈現 **內連關係（入站/出站）**
- 顯示該頁的 **AI 關鍵字** 與 **GSC 搜尋詞**，可快速跳到「關鍵字」或「著陸關鍵字」視圖

### 著陸關鍵字（Page Landing）

以「單一頁面」為中心，查看該頁面在搜尋中帶來流量/曝光的關鍵字分佈與排名機會（需要有 GSC 資料）。

### 關鍵字（Keyword Hub）

以「關鍵字」為中心，查看對應的頁面與關聯內容，用於：

- 檢視某個關鍵字目前主要由哪些頁面承接
- 找到內容缺口與內容佈局的調整方向

### 地圖（Graph）

用網路圖視覺化呈現內容結構，你可以在地圖內切換三種模式：

- **頁面模式**：以頁面為節點看內部連結結構
- **問題模式**：以「問答/問題」為節點，看內容能回答哪些問題
- **關鍵字模式**：以關鍵字為節點，看關鍵字之間的關聯與結構

地圖也支援搜尋與篩選（例如：連線類型、品牌相關度範圍、是否只看孤島節點），並提供 **匯出**（JSON / CSV / SVG）方便你做後續分析或留存。

### 列表（List）

提供兩個清單，適合快速瀏覽：

- **問答列表**：從已分析的頁面中整理的問答/內容片段（含品牌相關度、答案品質）
- **內連建議**：列出推薦的內連方向與信心/SEO 價值分數

## 資料更新與同步

- **啟動網站爬取**：若尚未有爬取資料，內容地圖會提示你啟動爬取；可選擇「首頁快速分析 / sitemap 智慧發現 / 整站爬取」等模式。
- **同步 GSC**：若已完成 GSC 授權並選擇站點，內容地圖會提供「同步」入口，將同步任務排程到背景執行。
