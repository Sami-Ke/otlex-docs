## 你可以用它做什麼

內部連結工具自動分析你的網站內容，找出適合互相連結的頁面：

- **發現連結機會**：找出主題相關但尚未互連的頁面
- **評估 SEO 價值**：了解哪些連結對排名幫助最大
- **取得具體建議**：知道該在哪裡加什麼連結
- **追蹤執行進度**：標記已處理的建議

<Callout type="info">
  內部連結對 SEO 很重要：它幫助搜尋引擎理解網站結構、傳遞頁面權重、並提升使用者體驗。
</Callout>

## 快速開始

1. **確認內容地圖已爬取**：進入「內容地圖」確認有頁面資料
2. **進入內部連結工具**：從側邊欄「工具 > 內部連結」進入
3. **檢視建議列表**：從高 SEO 價值的建議開始
4. **實作連結**：到 CMS 編輯頁面，加入建議的連結
5. **標記完成**：處理後標記為已處理

## 前置條件

| 條件 | 必要性 | 說明 |
|------|--------|------|
| 已完成內容爬取 | **必要** | 需要頁面內容才能分析主題關聯 |
| 頁面數量足夠 | 建議 | 至少 10+ 頁面才有連結機會 |
| 主題分析完成 | **必要** | 系統需時間分析頁面主題 |

## 運作原理

```
頁面內容 → 主題分析 → 相關性計算 → SEO 價值評估 → 建議排序
```

系統會：

1. 分析每個頁面的主題和關鍵字
2. 計算頁面間的主題相關性
3. 評估加入連結的 SEO 價值
4. 產生優先順序排列的建議

## 建議列表解讀

### 建議卡片

每個建議包含：

| 欄位 | 說明 | 如何解讀 |
|------|------|----------|
| **來源頁面** | 應該加入連結的頁面 | 你要去編輯這個頁面 |
| **目標頁面** | 連結指向的頁面 | 連結的 href 目標 |
| **信心分數** | AI 對建議的確信程度 | 越高越可靠 |
| **SEO 價值** | 預估對排名的幫助 | 高價值優先處理 |
| **主題標籤** | 相關的內容主題 | 確認主題相關性 |

### 信心分數

| 分數範圍 | 意義 | 建議 |
|----------|------|------|
| 80-100% | 非常相關，強烈建議 | 優先執行 |
| 60-79% | 相關性不錯 | 值得考慮 |
| 40-59% | 有些相關 | 視內容判斷 |
| < 40% | 相關性較低 | 可以跳過 |

### SEO 價值

| 等級 | 說明 | 影響因素 |
|------|------|----------|
| **高** | 對排名幫助大 | 目標頁面重要、來源頁面有流量 |
| **中** | 有正面影響 | 頁面有一定相關性和價值 |
| **低** | 影響較小 | 頁面流量或相關性較低 |

## 處理建議的流程

### 1. 評估建議

在執行前先評估：

- 信心分數是否夠高？（建議 > 60%）
- SEO 價值是否值得投入時間？
- 點擊預覽來源和目標頁面，確認確實相關

### 2. 決定錨點文字

錨點文字（Anchor Text）是連結的顯示文字：

| 類型 | 範例 | SEO 效果 |
|------|------|----------|
| **關鍵字型** | 「SEO 審查工具」 | 最佳 - 告訴搜尋引擎目標頁面主題 |
| **描述型** | 「學習如何做 SEO 審查」 | 良好 - 自然且有資訊 |
| **通用型** | 「點這裡」 | 差 - 沒有 SEO 價值 |
| **URL 型** | 「example.com/audit」 | 差 - 不自然 |

### 3. 找到適合的插入位置

在來源頁面中找到自然的位置：

- **段落中間**：融入已有的句子
- **相關段落結尾**：「想了解更多，請參考...」
- **延伸閱讀區塊**：統一放在文章底部

### 4. 實作連結

到 CMS 編輯來源頁面：

1. 找到適合的插入位置
2. 加入連結，使用適當的錨點文字
3. 確認連結可正常點擊
4. 儲存/發佈頁面

### 5. 標記完成

回到內部連結工具，將建議標記為「已處理」。

## 篩選與排序

| 功能 | 選項 | 使用時機 |
|------|------|----------|
| **按 SEO 價值** | 高→低 | 優先處理重要的 |
| **按信心分數** | 高→低 | 優先處理確定的 |
| **按主題篩選** | 選擇特定主題 | 一次處理同主題 |
| **顯示狀態** | 待處理/已處理 | 只看未完成的 |

## 與其他功能整合

### 內容地圖整合

建議也會出現在「內容地圖 > 列表 > 內連建議」：

- 可以在頁面詳情中看到該頁的建議
- 兩邊的標記狀態是同步的

### 關鍵字研究整合

結合關鍵字研究的發現：

1. 在關鍵字研究找出要優化的關鍵字
2. 用內部連結工具找出可以連到該頁面的機會
3. 用關鍵字作為錨點文字

## 常見問題

### 建議列表是空的

**可能原因**：

| 原因 | 解法 |
|------|------|
| 內容爬取未完成 | 等待爬取完成，或手動觸發 |
| 頁面數量太少 | 需要更多頁面才有連結機會 |
| 主題分析未完成 | 等待系統分析，通常需要幾分鐘 |
| 頁面主題太分散 | 內容沒有明顯的主題群聚 |

### 建議的相關性不太對

AI 判斷有時會有誤差：

- 低信心分數的建議可以跳過
- 人工判斷永遠優先於 AI 建議
- 不確定的建議就不要執行

### 加了連結後多久會有效果？

內部連結的效果：

- **爬取發現**：Google 下次爬取來源頁面時會發現新連結
- **權重傳遞**：需要數週到數月累積
- **排名變化**：視競爭程度，可能需要 1-3 個月

建議加入排名追蹤觀察效果。

### 一個頁面該加多少內部連結？

| 頁面類型 | 建議數量 | 說明 |
|----------|----------|------|
| 長文章（2000+ 字） | 5-15 個 | 可以較多 |
| 一般文章（500-2000 字） | 3-8 個 | 適中 |
| 短頁面（< 500 字） | 1-3 個 | 不宜太多 |
| 產品頁 | 2-5 個 | 連到相關產品 |

## 最佳實踐

### 錨點文字原則

| 做 | 不做 |
|-----|------|
| 使用描述性關鍵字 | 「點這裡」、「了解更多」 |
| 自然融入句子 | 生硬地插入 |
| 多樣化錨點文字 | 每次都用相同文字 |
| 簡潔明確 | 太長的錨點文字 |

### 連結策略

**重點頁面優先**：

1. 確認哪些是你的「重點頁面」（轉換頁、重要關鍵字頁）
2. 優先處理連到這些頁面的建議
3. 從高流量頁面連出去效果更好

**新內容整合**：

每次發佈新內容後：

1. 執行內部連結工具，查看新建議
2. 為新頁面找出可以連過來的舊頁面
3. 也找出新頁面可以連出去的目標

### 定期維護 SOP

建議每月執行：

1. **檢視新建議**：查看是否有新的連結機會
2. **處理高價值建議**：至少處理 5-10 個
3. **檢查已有連結**：確認沒有壞掉的連結
4. **標記處理狀態**：保持列表整潔

### 避免的錯誤

| 錯誤 | 問題 | 正確做法 |
|------|------|----------|
| 過度連結 | 體驗差、可能被視為垃圾 | 只加有意義的連結 |
| 相同錨點文字 | 不自然、可能觸發演算法 | 多樣化錨點文字 |
| 只連首頁 | 沒有 SEO 價值 | 連到具體相關頁面 |
| 忽略深層頁面 | 深層頁面難被爬取 | 確保所有頁面都有入口 |

## 進階：連結架構概念

### Hub and Spoke 模式

```
        [主題中心頁]
       /    |    \
      /     |     \
[子頁A] [子頁B] [子頁C]
```

- 建立一個主題的「中心頁」
- 所有相關子頁面都連到中心頁
- 中心頁連到所有子頁面

### 鏈狀連結

```
[文章1] → [文章2] → [文章3] → [文章4]
```

- 系列文章可以用「上一篇/下一篇」串連
- 適合教學系列或分集內容
