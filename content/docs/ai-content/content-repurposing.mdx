## 什麼是內容再利用？

你花心思寫了一篇 SEO 文章，但它的價值不應該只停留在搜尋流量。透過內容再利用，你可以：

- 把文章的核心洞察改寫成 **LinkedIn 商業觀點**
- 把步驟教學提煉成 **Twitter 系列推文**
- 把案例研究轉化成 **Facebook 長貼文**
- 把數據結論調整為 **Instagram 視覺貼文**

每篇文章可以延伸為 4+ 篇社群內容，最大化你的內容投資報酬率。

### 目前支援的社群平台

| 平台 | 說明 |
|------|------|
| **LinkedIn** | 個人頁或公司頁發文，適合 B2B 和專業內容 |
| **Facebook** | 粉絲專頁發文，適合社群互動和連結分享 |
| **X（Twitter）** | 推文，適合即時資訊和短型觀點 |
| **Instagram** | 貼文（需商業帳號），適合視覺化內容配文字 |

<Callout type="info">
  Threads、TikTok/Reels、Email Newsletter 等格式目前不在一般使用者的 UI 支援範圍。Help Center 不以「已支援」描述未上線能力。
</Callout>

## 方式一：從文章使用「貼文助手」

這是最常用的內容再利用方式：直接在文章編輯頁啟動社群貼文生成。

### 步驟詳解

**1. 開啟文章**
進入「內容生成 > 已生成文章」，找到要再利用的文章，點擊開啟。

**2. 打開貼文助手**
在文章頁右側（或工具列），找到「貼文助手」按鈕並點擊開啟面板。

**3. 選擇目標平台**

可以一次選擇多個平台，AI 會針對每個平台的特性生成對應格式的貼文。

| 平台特性 | AI 的調整方式 |
|----------|--------------|
| Twitter/X | 自動控制在 280 字元內，句子精簡有力 |
| LinkedIn | 較長格式，加入段落結構和 Call to Action |
| Facebook | 中等長度，加入對話感和互動引導 |
| Instagram | 配上 Hashtag 建議，強調視覺描述 |

**4. 設定追蹤連結（選用但強烈建議）**

在「連結設定」區塊可以設定 UTM 參數：

| UTM 欄位 | 說明 | 範例 |
|----------|------|------|
| utm_source | 流量來源平台 | `linkedin`、`facebook` |
| utm_medium | 媒介類型 | `social` |
| utm_campaign | 活動名稱 | `q1-content-push` |
| utm_content | 區分不同貼文 | `article-title-short` |

系統會自動組合成完整的追蹤網址，並嵌入貼文草稿中。

**5. 點擊「產生」**

AI 根據文章內容、平台特性和品牌風格（若已設定）生成貼文草稿，通常需要 10-30 秒。

**6. 審核並調整**

AI 生成只是初稿，建議：
- 確認語氣符合品牌調性
- 調整對特定平台受眾的說法
- 加入個人觀點或補充時事背景
- 確認 Hashtag 是否適合（特別是 Instagram）

**7. 後續處理**

| 選項 | 說明 |
|------|------|
| **複製貼文** | 直接複製到剪貼簿，手動貼到社群平台發布 |
| **儲存為草稿** | 儲存到「社群運營 > 貼文列表」統一管理 |
| **直接排程** | 若已連接社群帳號，可設定自動發布時間（目前僅 LinkedIn 支援自動發布，其他平台請用複製貼文） |

## 方式二：從「社群運營」集中管理

若你需要統一管理多篇貼文、規劃一週發布行程，或追蹤已發布貼文的狀態，請進入「**社群運營**」功能。

詳細功能請參考[社群運營文件](/docs/social-operations)。這裡簡要說明各分頁：

| 分頁 | 說明 |
|------|------|
| **產生貼文** | 不需要有文章，直接輸入主題讓 AI 生成 |
| **貼文列表** | 管理所有草稿、已排程、已發布、失敗的貼文 |
| **排程管理** | 月/週/日行事曆，拖放安排發布時間 |
| **平台帳號** | 連接和管理各社群平台的 OAuth 授權 |

---

## 平台帳號連接（啟用自動發布）

若要使用**自動排程發布**功能，需要先在「社群運營 > 平台帳號」連接對應的社群帳號。

<Callout type="info">
  自動發布目前僅 **LinkedIn** 完整支援。Facebook、Instagram、Twitter/X 的自動發布尚在開發中，這些平台請使用「複製貼文」方式手動發布。
</Callout>

### 連接流程

1. 進入「社群運營 > 平台帳號（Platforms）」
2. 點擊「連接帳號」
3. 選擇要連接的平台
4. 完成 OAuth 授權流程
5. 若有多個帳號/頁面，選擇要管理的那個

### 各平台注意事項

| 平台 | 特別注意 |
|------|----------|
| **Instagram** | 必須是**商業帳號**，且透過 Facebook 粉專連接 |
| **Facebook** | 發佈到**粉絲專頁**，非個人帳號 |
| **LinkedIn** | 可選擇發布到個人頁或公司頁 |
| **Twitter/X** | 標準 OAuth 授權即可 |

<Callout type="warning">
  平台 OAuth 授權可能會過期（通常是幾個月後，或你在平台端撤銷授權時）。若貼文發布失敗，請先到「平台帳號」確認授權狀態，並重新授權。
</Callout>

### 未連接帳號時的替代方案

即使沒有連接社群帳號，你仍可以：
1. 在「貼文助手」生成貼文草稿
2. 複製內容
3. 手動登入社群平台貼上發布

## UTM 追蹤詳解

UTM 參數是追蹤社群貼文帶來流量的關鍵工具。設定好後，在 GA4 中可以看到：
- 哪篇貼文帶來最多網站訪客
- 哪個平台的社群流量轉換率最高
- 特定活動的整體成效

### 建議的 UTM 命名規則

```
utm_source  = 平台名稱（linkedin / facebook / twitter / instagram）
utm_medium  = social
utm_campaign = 活動名稱（用連字號分隔，例如 2026-q1-seo-guide）
utm_content = 區分同一活動的不同貼文（例如 post-1 / post-2）
```

**範例完整網址：**
```
https://yoursite.com/blog/seo-guide?utm_source=linkedin&utm_medium=social&utm_campaign=2026-q1-seo-guide&utm_content=linkedin-post
```

## 品牌風格套用

若你已在「設定 > 品牌設定 > 寫作風格」建立了品牌寫作風格（Style Profile），AI 在生成社群貼文時可以自動套用：

- 使用你的品牌慣用詞彙
- 維持品牌語調（專業/親切/幽默等）
- 避免使用你設定的禁用語

關於品牌風格設定的詳細說明，請參考[品牌風格設定文件](/docs/ai-content/brand-style)。

## 貼文品質提升技巧

### 給 AI 更好的指示

在「貼文助手」的「額外指示」欄位，可以告訴 AI 更具體的要求：

| 指示類型 | 範例 |
|----------|------|
| 語氣調整 | 「用輕鬆口語，不要太正式」 |
| 重點指定 | 「重點放在文章的第三個建議」 |
| 格式要求 | 「用條列式分點，每點不超過一行」 |
| 受眾設定 | 「針對剛入行的 SEO 新手」 |
| 號召行動 | 「結尾加上邀請讀者留言分享的句子」 |

### 不同平台的最佳策略

| 平台 | 效果最好的內容類型 |
|------|------------------|
| LinkedIn | 個人觀點＋行業洞察，問「你怎麼看？」 |
| Facebook | 故事性內容，互動引導（留言、分享） |
| Twitter/X | 數據統計、反直覺觀點、簡短有力的主張 |
| Instagram | 結構清晰的清單（適合圖文卡片設計） |

## 常見問題

### AI 生成的貼文語氣不符合品牌？

1. 先確認是否已設定並啟用「品牌寫作風格」
2. 在「額外指示」欄位補充具體的語氣要求
3. 生成後直接在編輯器修改
4. 如果整體效果不理想，嘗試多生成幾次

### 排程的貼文沒有自動發布？

常見原因：
- 平台帳號授權已過期（最常見）→ 到「平台帳號」重新授權
- 帳號/頁面的發布權限不足
- 貼文內容違反平台規定被擋

排查步驟：進入「社群運營 > 貼文列表」查看失敗貼文的錯誤訊息。

### 為什麼生成的推文超過 280 字元？

AI 會嘗試控制長度，但有時估算不準確。請在預覽中確認字數，並手動縮短超出的部分。

### 可以一次生成多個平台的貼文嗎？

可以！在「目標平台」勾選多個平台，AI 會分別為每個平台生成對應的草稿。
