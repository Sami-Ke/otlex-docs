## 什麼是內容再利用？

一篇優質文章的價值不應只限於部落格。內容再利用（Content Repurposing）是將原始內容轉換為不同格式，在多個平台觸及更多受眾的策略。

### 為什麼要再利用內容？

| 好處 | 說明 |
|-----|------|
| **擴大觸及** | 同一內容觸及不同平台的使用者 |
| **節省時間** | 不需從零開始為每個平台創作 |
| **強化訊息** | 重複曝光加深受眾印象 |
| **提高 ROI** | 一份投入，多份產出 |
| **測試效果** | 找出哪個平台最適合你的內容 |

### Otlex 支援的轉換格式

| 格式 | 適用平台 |
|-----|---------|
| 社群貼文 | Facebook、LinkedIn、Twitter/X、Instagram、Threads |
| 影片腳本 | YouTube、TikTok、Instagram Reels |
| Podcast 大綱 | Apple Podcasts、Spotify、各 Podcast 平台 |
| Infographic 內容 | 設計師製作圖表的文案參考 |
| Email Newsletter | 電子報內容 |
| 簡報大綱 | PowerPoint、Keynote、Google Slides |

## 使用方式

### 方式一：從文章詳情頁轉換

1. 開啟已完成的文章
2. 點擊右上角「內容再利用」按鈕
3. 選擇要轉換的格式
4. 設定轉換參數（見下方詳解）
5. 點擊「開始轉換」
6. 等待生成完成（約 30 秒 - 2 分鐘）
7. 編輯、調整、匯出

### 方式二：從文章列表批次轉換

處理多篇文章時：

1. 進入「內容管理」>「文章」
2. 勾選要轉換的文章（可多選）
3. 點擊上方「批次再利用」
4. 選擇統一的轉換格式
5. 系統會逐一處理並通知完成

### 方式三：從主題直接轉換

如果文章還沒生成，但想直接產出社群內容：

1. 在主題詳情頁點擊「快速產出」
2. 選擇目標格式（跳過完整文章）
3. AI 會根據主題直接生成內容

<Callout type="info">
  快速產出適合時效性內容，但建議重要主題還是先生成完整文章再轉換，品質會更好。
</Callout>

## 轉換格式詳解

### 社群貼文

針對每個平台的特性優化內容：

#### Facebook 貼文

| 設定 | 說明 |
|-----|------|
| 長度 | 中長篇（300-500 字） |
| 風格 | 故事性、情感連結 |
| Emoji | 適度使用 |
| CTA | 鼓勵留言互動 |
| 生成數量 | 3-5 則不同角度 |

#### LinkedIn 貼文

| 設定 | 說明 |
|-----|------|
| 長度 | 中長篇（200-400 字） |
| 風格 | 專業、洞見分享 |
| 格式 | 善用換行、列點 |
| CTA | 邀請討論、分享經驗 |
| 生成數量 | 3 則不同切入點 |

#### Twitter/X 貼文

| 設定 | 說明 |
|-----|------|
| 長度 | 精簡（280 字以內） |
| 風格 | 重點突出、金句式 |
| Thread | 可選擇生成串文（5-10 則） |
| Hashtag | 自動建議相關標籤 |
| 生成數量 | 5 則獨立推文 |

#### Instagram 貼文

| 設定 | 說明 |
|-----|------|
| 長度 | 中等（150-300 字） |
| 風格 | 視覺導向、故事性 |
| Hashtag | 自動生成 20-30 個相關標籤 |
| 輪播建議 | 提供分頁內容建議 |
| 生成數量 | 3 則含 hashtag |

### 影片腳本

生成結構化的影片製作指南：

#### 腳本結構

```
1. Hook（前 3 秒）
   - 吸引注意力的開場
   - 問題或驚人事實

2. 內容主體
   - 分段腳本（每段 30-60 秒）
   - 視覺提示（建議畫面/B-roll）
   - 關鍵字幕文字

3. 結尾
   - 總結重點
   - CTA（訂閱、留言、分享）
```

#### 影片類型選項

| 類型 | 長度 | 適用平台 |
|-----|------|---------|
| **短影片** | 30-60 秒 | TikTok、Reels、Shorts |
| **中型影片** | 3-5 分鐘 | YouTube、LinkedIn Video |
| **長影片** | 10-15 分鐘 | YouTube 教學、Webinar |

### Podcast 大綱

生成完整的 Podcast 錄製指南：

#### 大綱內容

| 區塊 | 內容 |
|-----|------|
| **開場** | 節目介紹、本集主題預告（1-2 分鐘） |
| **主要內容** | 分段討論點、每段預估時間、轉場提示 |
| **互動環節** | 聽眾問題、討論引導 |
| **結尾** | 重點總結、下集預告、CTA |

#### 額外提供

- 節目標題建議（3 個選項）
- 節目描述文案
- 社群宣傳文案
- 時間戳記模板

### Email Newsletter

將文章轉換為電子報格式：

| 元素 | 說明 |
|-----|------|
| **主旨** | 3 個吸引開信的主旨選項 |
| **預覽文字** | 搭配主旨的預覽文字 |
| **內文** | 重新組織為信件格式 |
| **CTA** | 明確的行動呼籲按鈕文字 |

### Infographic 內容

為設計師準備的圖表製作文案：

#### 提供內容

- 主標題和副標題
- 區塊標題（3-5 個區塊）
- 每區塊的關鍵數據或要點
- 圖示建議（icon 關鍵字）
- 整體視覺風格建議

<Callout type="info">
  Infographic 內容不包含實際設計，而是提供給設計師的文案架構和內容指引。
</Callout>

## 轉換設定選項

### 通用設定

| 設定 | 說明 |
|-----|------|
| **使用品牌風格** | 轉換時套用品牌寫作風格 |
| **語調調整** | 比原文更正式/更輕鬆 |
| **目標受眾** | 指定受眾特性（影響用詞和角度） |
| **加入 UTM** | 自動為連結加上追蹤參數 |

### UTM 追蹤設定

系統會自動為轉換內容中的連結加入 UTM：

```
https://yoursite.com/article
  ?utm_source=facebook       // 發布平台
  &utm_medium=social         // 媒體類型
  &utm_campaign=repurpose    // 活動名稱
  &utm_content=post1         // 內容識別
```

你可以在「設定」>「UTM 規則」中自訂預設值。

### 自訂 CTA

每種格式都可以設定預設的 CTA：

| 格式 | 預設 CTA 範例 |
|-----|-------------|
| 社群貼文 | 「看完整文章連結在留言」 |
| 影片腳本 | 「喜歡的話記得訂閱！」 |
| Email | 「閱讀完整內容 →」 |

## 編輯與發布

### 轉換後編輯

生成的內容會進入編輯器，你可以：

- **直接編輯**：修改任何文字
- **AI 調整**：選取文字請 AI 重寫/精簡/擴充
- **格式調整**：調整段落、列表、emoji
- **預覽**：查看在該平台上的呈現效果

### 匯出選項

| 匯出方式 | 說明 |
|---------|------|
| **複製文字** | 複製到剪貼簿直接貼上 |
| **下載檔案** | 下載為 .txt 或 .md |
| **發布到 CMS** | 直接發到 WordPress/Notion |
| **加入排程** | 加入社群排程器 |

## 社群排程器

轉換後的社群貼文可以加入排程器統一管理：

### 功能

| 功能 | 說明 |
|-----|------|
| **日曆檢視** | 視覺化看所有排程內容 |
| **最佳時間建議** | AI 建議各平台發文最佳時間 |
| **預覽** | 預覽在各平台的呈現 |
| **批次排程** | 一次排程多則貼文 |

### 設定排程

1. 在轉換完成後點擊「加入排程」
2. 選擇目標平台
3. 設定發布時間或讓 AI 建議
4. 確認並儲存

<Callout type="info">
  自動發布功能需要先連接社群帳號（目前支援 Facebook Pages、LinkedIn Pages）。其他平台會在排程時間發送提醒。
</Callout>

## 成效追蹤

在 Google Analytics 4 中，你可以：

1. 進入「流量獲取」報表
2. 選擇「來源/媒介」維度
3. 篩選 utm_campaign = "repurpose"
4. 查看各平台的流量和轉換

### Otlex 內建追蹤

在「內容再利用」報表中可以看到：

| 指標 | 說明 |
|-----|------|
| 轉換次數 | 每篇文章被轉換的次數 |
| 平台分布 | 轉換到各平台的比例 |
| 點擊追蹤 | 帶有 UTM 連結的點擊數（需 GA4 連接） |

## 最佳實踐

### 選擇適合再利用的內容

| 適合 | 不適合 |
|-----|-------|
| ✅ 深度教學文章 | ❌ 簡短新聞快訊 |
| ✅ 列表型內容（X 個方法） | ❌ 時效性過強的內容 |
| ✅ 有數據/案例的文章 | ❌ 過於技術性的內容 |
| ✅ 觀點清晰的評論 | ❌ 公司內部公告 |
| ✅ 長青內容（Evergreen） | ❌ 過於小眾的主題 |

### 各平台調整重點

| 平台 | 調整重點 |
|-----|---------|
| **Facebook** | 加入情感元素、故事性開頭 |
| **LinkedIn** | 強調專業價值、加入個人見解 |
| **Twitter** | 找出金句、製造討論點 |
| **Instagram** | 視覺化描述、生活化語言 |
| **YouTube** | 增加視覺提示、互動引導 |

### 避免的錯誤

- ❌ 直接複製貼上不做調整
- ❌ 所有平台用完全相同內容
- ❌ 忘記檢查連結是否正確
- ❌ 忽略各平台的格式限制
- ❌ 沒有加入追蹤參數

## 常見問題

### 轉換會完全自動嗎？

AI 會完成大部分工作，但建議每則內容都檢查調整。自動生成是起點，人工潤飾是必要的。

### 同一篇文章可以轉換多次嗎？

可以。每次轉換會產生不同的內容版本，適合需要多則貼文的情況。

### 轉換後的內容版權歸誰？

你擁有完整版權，可自由使用、修改、發布。

### 可以指定轉換的角度嗎？

可以。在轉換設定中選擇「進階選項」，輸入希望的切入角度或強調重點。
