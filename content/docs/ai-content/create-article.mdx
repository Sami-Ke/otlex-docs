## 開始之前

在生成第一篇文章前，準備越完整，生成品質越好：

| 項目 | 必要性 | 說明 |
|------|--------|------|
| 品牌基本資訊 | ✅ 必要 | 先建立品牌並確認網站網域 |
| 品牌寫作風格 | 強烈建議 | 讓 AI 學習你的語調，產出更符合品牌的內容 |
| Google Search Console | 推薦 | 生成時可選取真實的 GSC 關鍵字作為優化方向 |
| 已爬取的網站頁面 | 推薦 | 用於品牌風格分析、內部連結建議、內容參考 |
| CMS 發佈整合 | 選用 | 若想一鍵發佈到 WordPress/Webflow，請先在「設定 > 整合」設定（Shopify UI 即將開放） |

## 文章生成完整流程

### 步驟 1：從「主題」出發

Otlex 的文章生成以「**主題（Topics）**」為起點。主題代表你想寫的方向或目標關鍵字，品質好的主題能讓 AI 生成更聚焦的文章。

**進入主題頁面：** 從左側選單點擊「主題」

**主題來源有三種方式：**

| 方式 | 說明 | 適合情況 |
|------|------|----------|
| **主題生成** | 設定條件讓 AI 建議主題清單 | 不知道要寫什麼時 |
| **手動建立** | 自己輸入主題和目標關鍵字 | 已有明確想法 |
| **內容差距分析** | 輸入競品網址，找出你沒有的主題 | 想追競品的內容布局 |

**什麼是好的主題？**

好的主題應該包含：
- **目標受眾**：這篇文章是寫給誰的？
- **核心問題**：讀者想解決什麼問題或學到什麼？
- **目標關鍵字**：要針對哪個搜尋詞優化？
- **內容角度**：「怎麼做」教學、「是什麼」說明、「比較型」還是「案例型」？

**範例：**
```
❌ 差的主題：「SEO 文章」
✅ 好的主題：「給 B2B SaaS 新手的 SEO 入門指南 — 如何在 6 個月內建立自然流量」
```

---

### 步驟 2：設定文章生成參數

挑選好主題後，進入「**內容生成 > 文章生成設定**」配置文章參數。

#### 核心生成設定

**文章類型（Article Type）**

不同文章類型對應不同的結構與格式：

| 類型 | 說明 | 適合的搜尋意圖 |
|------|------|----------------|
| **部落格文章** | 資訊性長文，多段落結構 | informational 查詢 |
| **教學指南（How-to）** | 步驟清晰的操作說明 | 「如何…」查詢 |
| **條列式清單** | N 個技巧/方法/工具 | 「最佳…清單」查詢 |
| **比較型文章** | A vs B 的比較分析 | 「X 和 Y 哪個好」查詢 |
| **案例研究** | 真實案例的深度分析 | 研究型查詢 |
| **落地頁** | 轉換導向的產品/服務說明 | commercial/transactional 查詢 |
| **FAQ 頁面** | 問答格式的資訊彙整 | question-type 查詢 |

**語氣（Tone）**

| 語氣選項 | 說明 | 適合情境 |
|----------|------|----------|
| 專業 | 正式、有論據 | B2B、技術文章 |
| 親切 | 友善、易讀 | B2C、入門教學 |
| 權威 | 自信、有說服力 | 意見領袖文章 |
| 幽默 | 輕鬆、有趣 | 生活風格、娛樂 |
| 中立 | 客觀、不帶立場 | 比較分析、新聞 |

**創意度（Creativity）**

控制 AI 在生成內容時引入多少變化，以滑桿調整：

| 設定 | 說明 | 適合場景 |
|------|------|----------|
| 保守 | 穩定輸出，結構清晰 | 正式文章、法律/金融/醫療等需要精確措辭的內容 |
| 平衡（預設） | 兼顧穩定性與自然變化 | 一般部落格、教學指南 |
| 創意 | 較多變化，更接近自然的人類寫作風格 | 品牌故事、吸睛內容、生活風格類文章 |

**文章長度**

| 長度 | 字數範圍（參考） | 適合場景 |
|------|-----------------|----------|
| 短 | ~800-1,200 字 | 簡單概念說明、新聞型文章 |
| 中 | ~1,500-2,500 字 | 一般部落格文章 |
| 長 | ~3,000-5,000 字 | 深度指南、完整教學 |
| 超長 | ~5,000-8,000 字 | 終極指南、完整手冊 |

<Callout type="info">
  重新生成（Regenerate）時，系統會自動從文章中讀取原始設定（文章類型、語氣、長度等），確保重新生成後維持原有風格，不會縮短到預設的中等長度。
</Callout>

#### 進階設定

**自訂提示詞（Custom Prompt）**

直接告訴 AI 你對這篇文章的特殊要求：

```
範例：
- 「開頭要有一個吸引人的問題，引發讀者共鳴」
- 「每個 H2 段落後要有一個小結摘要」
- 「不要使用「革命性」、「顛覆性」等過度行銷語言」
- 「包含至少 3 個真實案例，若無法確認就用假設性案例」
```

**參考檔案（Reference Documents）**

上傳 PDF、Word 或文字檔作為 AI 的參考資料：
- 內部研究報告
- 白皮書或產業調查
- 你自己寫的草稿或大綱

**GSC 關鍵字（需已連接 GSC）**

從 Google Search Console 的查詢資料中選取目標關鍵字：
- AI 會將選取的關鍵字融入文章內容
- 提升文章對目標搜尋詞的相關性
- 特別適合「排名 11-20 需要優化」的關鍵字

---

### 步驟 3：送出生成任務

設定完成後，點擊「**開始生成**」，系統會：

1. 建立背景任務（避免等待的頁面阻塞）
2. 呼叫 AI 生成服務
3. 在背景完成整篇文章的生成

**等待期間你可以做什麼？**
- 離開頁面繼續其他工作
- 進入「內容生成 > 已生成文章」查看進度
- 進度狀態：`排隊中 → 處理中 → 完成`

**預計等待時間：**

| 文章長度 | 大約時間 |
|----------|----------|
| 短 | 1-2 分鐘 |
| 中 | 2-4 分鐘 |
| 長 | 4-8 分鐘 |
| 超長 | 8-15 分鐘 |

---

### 步驟 4：在編輯器審閱與編輯

文章生成完成後，進入文章編輯頁進行審閱和修改。

#### 編輯器功能一覽

| 功能 | 說明 |
|------|------|
| **富文字編輯** | 所見即所得的視覺化編輯，支援標題、粗體、清單、表格等 |
| **HTML 模式** | 切換到原始 HTML 編輯，適合需要精確控制格式的場景 |
| **圖片插入** | 從工具列上傳圖片，嵌入文章中 |
| **選取改寫** | 選取任意段落，請 AI 重新改寫（消耗 AI 額度） |
| **內部連結** | 在編輯器內直接插入到其他品牌頁面的內連建議 |
| **歷史版本** | 重新生成或大幅修改前，系統自動保存版本供回復 |
| **協作註解** | 在文章特定位置建立評論，可 @mention 團隊成員 |

#### 選取改寫（AI Rewrite）

這是最常用的 AI 輔助編輯功能：

1. 選取要改寫的文字段落
2. 點擊工具列「AI 改寫」按鈕
3. 可選擇改寫方向：精簡、擴充、換語氣、改為條列式等
4. AI 產生替代版本
5. 接受或繼續修改

<Callout type="info">
  每次選取改寫會消耗少量 AI 額度。額度狀態顯示在頁面右上角。
</Callout>

---

### 步驟 5：發佈或匯出

文章完成後，點擊右上角「**發佈 / 匯出**」進入發布流程。

#### 匯出格式

| 格式 | 說明 | 適合情況 |
|------|------|----------|
| **複製 HTML** | 複製完整 HTML 程式碼 | 貼入任何支援 HTML 的 CMS |
| **複製 Markdown** | 複製 Markdown 格式文字 | 技術部落格、GitHub Pages 等 |

#### CMS 直接發佈

若已在「設定 > 整合」完成 CMS 整合，可以直接發佈到：

| 目的地 | 整合類型 | 說明 |
|--------|----------|------|
| **WordPress** | 完整發佈整合 | 建立或更新 WordPress 文章 |
| **Webflow** | 完整發佈整合 | 發佈到 Webflow CMS Collection |
| **Shopify** | 🚧 準備中 | 後端已就緒，UI 設定入口即將開放 |
| **HTTP API 端點** | 自訂整合 | 提供 API 讓外部工具（Zapier / n8n）呼叫發布 |
| **Notion** | 匯出整合 | 建立 Notion 頁面（純文字寫入） |
| **Google Docs** | 匯出整合 | 建立 Google Docs 文件 |

#### 自動發布排程

如果你想在特定時間自動發佈，可設定排程：

1. 點擊「排程發佈」
2. 選擇目標 CMS 目的地
3. 設定發佈日期和時間
4. 確認排程

排程到時，系統會自動呼叫對應的 CMS 整合發佈文章。

#### 社群貼文（貼文助手）

在文章頁同時可開啟「**貼文助手**」，快速產生對應的社群媒體貼文：
- 選擇平台（LinkedIn / Facebook / Twitter / Instagram）
- AI 自動把文章重點改寫成符合該平台的格式
- 可設定 UTM 追蹤參數
- 直接儲存到社群運營，或複製手動發布

詳細說明請參考[內容再利用文件](/docs/ai-content/content-repurposing)。

## 文章管理功能

### 已生成文章清單

在「內容生成 > 已生成文章」可以：

| 功能 | 說明 |
|------|------|
| 查看所有文章 | 包含狀態（生成中/完成/失敗）、字數、建立時間 |
| 搜尋/篩選 | 按主題、狀態、日期篩選 |
| 批量操作 | 批量刪除 |
| 發佈狀態追蹤 | 查看已發佈到哪些 CMS 目的地 |

### 文章版本歷史

每篇文章在以下情況會自動建立新版本：

- 點擊「重新生成」整篇文章
- 在編輯器做大量修改後手動建立快照

查看版本歷史：
1. 在文章頁點擊右上角「...」選單
2. 選擇「版本歷史」
3. 選擇要回復的版本

## 常見問題

### 生成的內容品質不滿意？

1. **主題更具體**：加入受眾描述、要解的問題、語氣和關鍵字限制
2. **啟用品牌風格**：確認「設定 > 品牌設定 > 寫作風格」已完成設定
3. **加入參考資料**：提供你的研究或草稿作為參考
4. **使用自訂提示詞**：告訴 AI 具體的結構要求
5. **編輯器選取改寫**：對不滿意的段落局部改寫

### 重新生成時可以更換參考資料嗎？

可以。點擊「重新生成」會開啟 dialog，讓你：

- **上傳新的參考檔案**：替換原始生成時使用的 PDF/Word 文件
- **查看原始參考檔案**：若第一次生成時有上傳檔案，系統會顯示原始檔案名稱供參考
- **從 Research Vault 選取文件**：若你的品牌有 Research Vault，可直接選取已儲存的研究文件
- **自訂提示詞**：輸入對這次重新生成的特別要求

此外，文章類型、語氣、長度、創意度等設定也會自動從原始設定讀取保留。

### 為什麼重新生成後文章變短了？

如果文章原本是「長」或「超長」設定，但重新生成後變成「中等」長度，可能是：

- 舊版文章沒有儲存原始設定，系統使用了預設長度
- 解法：在「生成設定」重新選擇正確的長度再生成

新版的重新生成邏輯已改進，會自動讀取文章原始的 recipe 設定（文章類型、語氣、長度等），確保維持一致。

### 生成失敗怎麼辦？

| 可能原因 | 解決方式 |
|----------|----------|
| AI 配額用完 | 查看「方案與計費」確認 AI 點數餘額 |
| 自備 API Key 額度不足 | 到 AI Provider 補充額度 |
| 服務暫時性錯誤 | 等待幾分鐘後重試 |
| 主題內容有限制問題 | 調整主題描述，避免觸發內容限制 |

若多次重試仍失敗，請截圖錯誤訊息來信 [support@ethorx.com](mailto:support@ethorx.com)。

### 可以批量生成多篇文章嗎？

可以。在「主題」頁面選取多個主題，然後批量送出生成任務。各篇文章會排隊依序處理。

### 生成的文章版權歸誰？

使用 Otlex 生成的內容通常可依你的商業需求使用、修改和發佈。實際版權與使用規定以 Otlex 服務條款及所使用的 AI 模型供應商條款為準。
