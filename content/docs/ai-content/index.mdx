## 這裡能做什麼（以目前 App 介面為準）

Otlex 的「AI 內容生成」是一套從主題到文章、再到發佈/匯出的工作流程。文件內容會盡量跟著實際功能走，避免宣稱尚未上線的能力。

| 模組 | 你可以做什麼 |
|-----|-------------|
| **主題（Topics）** | 調整主題生成設定、檢視已生成主題、做競品內容差距分析（Gap analysis） |
| **文章生成（Generation）** | 以主題為中心建立文章生成條件，並透過背景任務產生文章草稿 |
| **文章編輯器** | 富文字/HTML 編輯、插入圖片、選取文字請 AI 改寫、內部連結工具、註解與協作、歷史版本與重新生成 |
| **發佈／匯出** | 複製 HTML/Markdown；或依「整合設定」發佈到 WordPress/Shopify/Webflow（Notion/Google Docs 依模式可能是臨時匯出或純文字寫入） |
| **社群貼文** | 在文章頁用「貼文助手」產生多平台貼文草稿（含 UTM）；在「社群運營」集中管理貼文、排程與平台帳號 |

## 典型流程

```
主題（Topics）→ 文章生成（Generation）→ 編輯/協作 → 發佈／匯出 → 社群貼文（可選）
```

<Callout type="info">
  內部連結相關功能可在「工具 > 內部連結」使用，文章編輯器內也可使用內連工具協助插入連結。
</Callout>

## AI 額度（重要）

產生文章、重新生成、選取改寫、產生社群貼文等操作都會呼叫 AI 服務：

- 若你使用 **自備 API Key**：會消耗你的 API Key 額度
- 若你使用 **系統方案**：會消耗方案內建的額度/點數

（不同環境/權限的可用範圍與顯示文案，以 App 畫面為準。）
