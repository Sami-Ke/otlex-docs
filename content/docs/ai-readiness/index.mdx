## 你可以用它做什麼

AI Readiness 分析讓你了解 AI 搜尋引擎如何「看待」你的網頁內容：

- **評估 AI 可讀性**：從 AI 的視角檢視頁面結構、內容品質與技術標記
- **找出優化缺口**：具體指出哪些問題可能讓 AI 忽略或誤解你的內容
- **跟蹤改善進度**：每次優化後重新分析，觀察分數趨勢
- **競品語意對比**（輸入目標關鍵字後啟用）：找出你的頁面相比 SERP 前幾名競品缺少了哪些語意覆蓋

<Callout type="info">
  **AI Readiness 消耗 AI 額度。** 每次分析都會呼叫 AI 服務：若使用自備 API Key，消耗你的 API 額度；若使用平台方案，消耗方案內建額度。分析前會顯示確認視窗。
</Callout>

## 快速開始

1. 進入左側導覽的「**AI Readiness**」
2. 在輸入框貼上要分析的頁面 URL
3. （選填）輸入「目標關鍵字」以啟用語意深度分析
4. 點擊「**開始分析**」
5. 等待分析完成（通常約 30-60 秒）
6. 查看結果並依建議逐步優化

## 三步驟流程

### 第一步：輸入

- **頁面 URL**：要分析的目標頁面
- **目標關鍵字**（選填）：此頁面想要排名的關鍵字。輸入後會額外解鎖：
  - 關鍵字叢集覆蓋率分析
  - 競品語意差距分析

### 第二步：分析中

系統依序執行四個階段：

| 階段 | 說明 |
|------|------|
| 擷取頁面 | 取得頁面的 HTML 與結構 |
| 解析內容 | 提取文字、標記、Schema 等資訊 |
| AI 分析 | 呼叫 AI 服務評估內容品質與語意 |
| 計算評分 | 依四大維度加權計算最終分數 |

分析過程中可選擇取消；若意外中斷，可從記錄中重試。

### 第三步：查看結果

分析完成後，你會看到整體評分、四大維度評分、可執行建議，以及（若有輸入關鍵字）語意深度分析。

## 評分系統

### 整體評級（A–F）

| 等級 | 說明 |
|------|------|
| **A** | 優秀！AI 搜尋引擎能很好地理解此頁面 |
| **B** | 良好，但仍有改進空間 |
| **C** | 一般，建議參考下方建議進行優化 |
| **D** | 待改進，有多個重要問題需要處理 |
| **F** | 需要大幅改進，AI 可能難以理解此頁面 |

<Callout type="info">
  評分約以 85 分以上為最佳實踐門檻，70 分以下為需優先處理區間。實際分數顯示於結果頁。
</Callout>

### 四大維度評分

每個維度各自計分，最後加權合算為整體分數：

| 維度 | 評估重點 |
|------|---------|
| **結構化數據** | Schema.org 標記、JSON-LD 完整性、Article/FAQ/Product 等結構化標記的正確實作 |
| **內容品質** | 內容深度、可信度、事實準確性、是否適合被 AI 摘要引用 |
| **語意 SEO** | 主題覆蓋廣度、語意密度、實體識別、與目標關鍵字的語意連結 |
| **技術優化** | Meta 標記、標題層級、頁面結構、載入效能等技術面 |

## 語意深度分析（需輸入目標關鍵字）

分析目標關鍵字後，結果頁的「語意 SEO 深度分析」區塊會顯示：

### 關鍵字叢集分析

檢視頁面是否覆蓋了目標關鍵字周圍的語意叢集（相關詞彙、概念、子主題），幫助你找出遺漏的內容方向。

### 競品語意差距

將你的頁面與 SERP 前幾名競品進行語意對比，顯示競品提到而你沒有提到的核心概念和詞彙。

### 實體識別分析

辨識頁面中出現的具名實體（人物、地點、品牌、概念），評估實體豐富度是否足以讓 AI 正確理解頁面主題。

## 結果頁功能

### 報告模式切換

| 模式 | 適合對象 | 說明 |
|------|---------|------|
| **商務版** | PM、行銷、業務 | 以業務影響為重點，降低技術術語 |
| **技術版** | 工程師、SEO 技師 | 顯示完整技術細節與具體修改建議 |

### 提案模板

選擇不同模板，報告內容會自動調整敘事角度：

- **標準報告（無模板）**：通用格式
- **電商成長**：強調轉換率、產品頁最佳化
- **SaaS 需求開發**：強調潛在客戶教育與解決方案內容
- **在地服務**：強調本地 SEO 與地理相關標記

### AI 視角

切換到「AI 視角」，查看 AI 系統如何「閱讀」你的頁面——這是 AI 在理解你的內容時所看到的簡化視圖。

### 分享連結

點擊「分享連結」生成可公開存取的報告 URL，方便與客戶、同事分享分析結果。點擊「撤銷分享」可停用該連結。

### 重新分析

優化頁面後，點擊「**重新分析（跳過快取）**」立即重跑分析，確認改善是否有效。

<Callout type="info">
  系統會快取最近一次的分析結果。若頁面已更新但想立即看到新結果，請務必使用「跳過快取」選項。
</Callout>

## 歷史趨勢

每次分析都會記錄評分，結果頁會顯示歷史趨勢圖，讓你觀察：

- 整體分數的長期走向（改善中 / 下降中 / 穩定）
- 四大維度各自的趨勢
- 分析次數與總變化

## 比較功能

可以並排比較同品牌下兩個頁面的 AI Readiness 分數，幫助你決定資源優先分配給哪個頁面。

## 統計總覽

在 AI Readiness 主頁面可以看到品牌的整體數據：

| 數據 | 說明 |
|------|------|
| 已分析頁面數 | 品牌內累積分析過的頁面數量 |
| 平均評分 | 所有頁面的加權平均 |
| 需要改進 | 評分低於 70 分的頁面數 |
| 最佳實踐 | 評分高於 85 分的頁面數 |

## 常見問題

### 分析結果準確嗎？

分析基於頁面的實際 HTML 結構、結構化標記與 AI 語意評估。結果反映「AI 在分析時刻的頁面狀態」，若頁面有 JavaScript 動態注入的內容，可能無法完整被評估（側邊欄會顯示抓取方式說明）。

### 為什麼我的分數比預期低？

最常見的原因：
- 缺少 Schema.org 結構化標記（例如 Article、FAQ、BreadcrumbList）
- 內容深度不足（字數太少、缺少具體數據或範例）
- Meta description 過短或不夠描述性
- 頁面主題模糊、語意密度偏低

### 每次分析都消耗額度嗎？

是的，每次觸發分析都會消耗 AI 額度。若在快取有效期內重新分析，系統會提示使用快取結果；若要強制重跑，需使用「跳過快取」，此時才消耗額度。

### 分析多久可以完成？

通常 30-60 秒。若頁面較複雜或 AI 服務回應較慢，可能稍微更長。
