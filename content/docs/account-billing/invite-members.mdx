## 組織與團隊架構

Otlex 以「組織（Organization / Tenant）」管理團隊成員與計費。一個組織可以包含多個品牌，所有成員共享組織的方案配額。

```
組織（Organization / Tenant）
  ├── 品牌 A（Brand）—— 所有 SEO、內容、排名資料
  ├── 品牌 B（Brand）—— 各品牌資料獨立
  └── 成員（Members）—— 組織層級統一管理
```

### 成員與品牌的關係

| 概念 | 說明 |
|------|------|
| 組織成員 | 受邀後加入組織，可存取組織底下的品牌 |
| 品牌存取 | 通常組織成員可存取所有品牌（以角色和 App 設定為準） |
| 配額共享 | SEO 審查次數、AI 點數等配額由組織所有成員共用 |

<Callout type="info">
  不同部署環境與方案下，成員管理頁面與角色名稱可能略有差異；以你在 App 看到的畫面為準。
</Callout>

## 誰可以邀請成員？

邀請成員需要具備「組織管理」相關權限，通常是以下角色：

| 角色 | 是否可邀請 |
|------|-----------|
| 擁有者（Owner） | ✅ 可以 |
| 管理員（Admin） | ✅ 可以（依設定） |
| 編輯者（Editor） | ❌ 通常不行 |
| 檢視者（Viewer） | ❌ 不行 |

若你沒有看到「邀請成員」或「成員管理」的入口，通常是因為目前帳號權限不足。請聯絡你的組織擁有者或管理員協助處理。

## 邀請成員步驟

### 步驟 1：進入成員管理

有幾種方式可以找到成員管理頁面：

**方式一：透過帳號設定**
1. 點擊右上角的帳號頭像或名稱
2. 選擇「設定」或「帳號」
3. 找到「成員」或「Members」分頁

**方式二：透過方案與計費**
1. 進入「設定」>「方案與計費」
2. 找到「成員管理」區塊

**方式三：透過組織設定**
1. 進入「設定」>「組織設定」
2. 找到「成員」分頁

### 步驟 2：點擊「邀請成員」

找到「邀請成員」按鈕，點擊進入邀請流程。

### 步驟 3：填寫邀請資訊

| 欄位 | 必填 | 說明 |
|------|------|------|
| Email | ✅ | 受邀者的電子郵件地址 |
| 角色 | ✅ | 選擇賦予的權限等級 |

<Callout type="warning">
  確認 Email 輸入正確再送出，部分系統不支援邀請後修改 Email。
</Callout>

### 步驟 4：送出邀請

點擊「送出邀請」後，系統會：

1. 發送邀請電子郵件給受邀者
2. 在「成員管理」頁面顯示「待接受」狀態
3. 等待受邀者接受邀請

### 步驟 5：受邀者接受邀請

受邀者收到邀請後的操作：

**已有 Otlex 帳號：**
1. 點擊信中的邀請連結
2. 登入 Otlex
3. 確認加入組織
4. 即可存取組織的品牌和功能

**尚未有帳號：**
1. 點擊信中的邀請連結
2. 完成 Otlex 帳號註冊
3. 帳號建立後自動加入組織

## 角色與權限說明

Otlex 的角色設計基於最小權限原則，確保成員只能存取工作所需的功能。

<Callout type="info">
  以下角色說明為一般性描述，實際權限範圍以 App 中的設定顯示為準。部分功能的存取控制可能依方案和部署環境而異。
</Callout>

### 常見角色概覽

| 角色 | 適用對象 | 典型能力 |
|------|----------|----------|
| **擁有者（Owner）** | 帳號創建者或最高管理者 | 完整控制：計費、成員、設定、內容 |
| **管理員（Admin）** | 主要管理者 | 大部分功能，可管理成員 |
| **編輯者（Editor）** | 內容和 SEO 工作者 | 審查、內容生成、發佈、工具使用 |
| **檢視者（Viewer）** | 需要查看報表的人 | 只讀，查看報表但不能修改 |

### 代理商使用情境

若你是代理商，管理多個客戶的品牌，建議：

- 為每個客戶建立獨立**組織（或品牌）**
- 內部成員設為 Admin 或 Editor
- 客戶代表可以設為 Viewer，讓他們查看報表

## 管理現有成員

### 查看成員清單

在成員管理頁面可以看到：

| 資訊 | 說明 |
|------|------|
| 成員名稱/Email | 識別身份 |
| 角色 | 目前的權限等級 |
| 狀態 | 已接受 / 待接受（邀請中）/ 停用 |
| 加入時間 | 何時加入組織 |

### 修改成員角色

1. 在成員清單找到目標成員
2. 點擊角色旁的編輯按鈕（或下拉選單）
3. 選擇新的角色
4. 確認變更

角色變更**立即生效**，成員下次操作時就會套用新權限。

### 移除成員

1. 在成員清單找到目標成員
2. 點擊「移除」或「刪除」
3. 確認移除

<Callout type="warning">
  移除成員後，該成員立即失去組織存取權限，無法再進入任何品牌。移除前請確認成員的工作已交接完畢。
</Callout>

## 邀請上限與方案配額

可邀請的成員數量取決於你的訂閱方案：

| 方案層級 | 成員上限（參考） |
|----------|-----------------|
| Free | 1-3 人 |
| Starter | 5-10 人 |
| Pro | 15-20 人以上 |
| Enterprise | 無限制（依合約） |

<Callout type="info">
  實際上限以「方案與計費」頁面顯示的數字為準。若需要更多成員，可升級方案或聯絡客服詢問。
</Callout>

## 配額共享機制

組織內所有成員共享以下配額：

| 配額類型 | 說明 |
|----------|------|
| SEO 審查次數 | 成員一起使用的總次數 |
| AI 生成點數 | 所有 AI 功能共同消耗 |
| 品牌數量 | 組織可建立的品牌上限 |

**建議做法：**
- 在組織設定中設定 AI 使用配額上限（若功能開放）
- 定期查看「方案與計費」確認配額使用情況
- 即將用完時提前升級方案

## 常見問題

### 對方沒有收到邀請信？

請依序確認：

1. **Email 是否正確**：檢查輸入的 Email 有無拼寫錯誤
2. **垃圾信件夾**：邀請信可能被過濾到垃圾郵件或「促銷」分頁
3. **公司信箱限制**：企業信箱可能封鎖外部寄件者
4. **重新發送**：在成員清單找到「待接受」狀態的邀請，點擊「重新發送邀請」

若以上都試過仍無法解決，請截圖錯誤訊息聯絡客服。

### 邀請有效期多久？

邀請連結的有效期以收到的邀請信內說明為準。若連結已過期，請管理員重新發送邀請。

### 我找不到「成員管理」入口？

最常見原因是帳號權限不足（非 Admin/Owner）。請聯絡組織的管理員或擁有者協助：
- 確認你的角色是否有成員管理權限
- 或請管理員直接代為邀請

### 可以轉移「擁有者」角色給其他人嗎？

可以，但通常需要聯絡客服協助處理，不支援自助操作。這是為了避免意外轉移導致失去帳號控制。

### 受邀成員可以存取所有品牌嗎？

通常是的，組織成員可以存取組織底下的所有品牌。若需要限制成員只能存取特定品牌，請聯絡客服確認目前版本的支援狀況。

## 最佳實踐

### 代理商管理建議

- 建立統一的命名規則，例如「[客戶名] - [環境]」
- 定期審視成員清單，移除已離職或換職的舊帳號
- 對客戶端的成員使用較低權限（如 Viewer），避免誤操作

### 安全建議

| 建議 | 原因 |
|------|------|
| 離職同仁立即移除 | 防止前員工存取品牌資料 |
| 避免共用帳號 | 無法追蹤誰做了什麼操作 |
| 最小權限原則 | 給予成員完成工作所需的最低權限 |
