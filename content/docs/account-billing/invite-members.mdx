## 組織與團隊架構

Otlex 使用三層架構來組織團隊：

```
組織（Organization）
  ├── 品牌 A（Brand）
  │     └── SEO 數據、內容、整合
  ├── 品牌 B（Brand）
  │     └── SEO 數據、內容、整合
  └── 團隊成員（Members）
        ├── Owner（擁有者）
        ├── Admin（管理員）
        ├── Member（成員）
        └── Viewer（檢視者）
```

### 架構說明

| 層級 | 說明 |
|-----|------|
| **組織** | 最高層級，管理計費、成員和所有品牌 |
| **品牌** | 代表一個網站，擁有獨立的 SEO 數據和內容 |
| **成員** | 可存取組織下所有品牌（依角色權限） |

## 角色與權限

### 角色說明

| 角色 | 說明 | 適合對象 |
|-----|------|---------|
| **Owner** | 組織擁有者，完整權限 | 公司負責人、帳號建立者 |
| **Admin** | 管理員，可管理成員和設定 | 部門主管、IT 管理員 |
| **Member** | 成員，可使用所有功能 | 行銷人員、SEO 專員 |
| **Viewer** | 檢視者，僅能查看數據 | 主管、外部顧問 |

## 邀請成員

### 邀請步驟

1. 進入「設定」>「成員管理」
2. 點擊「邀請成員」
3. 輸入對方的 Email
4. 選擇角色
5. （選用）輸入歡迎訊息
6. 點擊「送出邀請」

### 邀請流程

```
1. 送出邀請
      ↓
2. 對方收到 Email 邀請
      ↓
3. 對方點擊連結
      ↓
4. 已有帳號 → 直接加入組織
   沒有帳號 → 註冊後加入組織
      ↓
5. 完成加入
```

### 邀請限制

| 限制項目 | 說明 |
|---------|------|
| 邀請連結有效期 | 7 天 |
| 同一 Email | 不能重複邀請（需取消原邀請） |
| 成員上限 | 依方案限制 |

### 方案成員上限

| 方案 | 成員上限 |
|-----|---------|
| Starter | 2 人 |
| Pro | 5 人 |
| Business | 20 人 |
| Enterprise | 無限 |

## 管理待處理邀請

### 查看邀請狀態

1. 進入「設定」>「成員管理」
2. 切換到「待處理邀請」分頁
3. 查看所有待接受的邀請

### 邀請狀態說明

| 狀態 | 說明 |
|-----|------|
| ⏳ **待接受** | 邀請已發送，等待對方接受 |
| ✅ **已接受** | 對方已加入組織 |
| ⏰ **已過期** | 超過 7 天未接受 |
| ❌ **已取消** | 邀請被取消 |

### 重新發送邀請

如果對方沒收到或邀請過期：

1. 找到該邀請
2. 點擊「重新發送」
3. 系統會發送新的邀請 Email
4. 新邀請有效期重新計算 7 天

### 取消邀請

1. 找到要取消的邀請
2. 點擊「取消」
3. 確認取消

取消後可以重新邀請同一 Email。

## 管理現有成員

### 查看成員列表

1. 進入「設定」>「成員管理」
2. 查看「成員」分頁
3. 顯示所有組織成員

### 成員列表顯示

| 欄位 | 說明 |
|-----|------|
| 頭像 | 成員頭像 |
| 名稱 | 顯示名稱 |
| Email | 登入 Email |
| 角色 | Owner / Admin / Member / Viewer |
| 加入時間 | 加入組織的日期 |
| 最後活動 | 最後一次使用 Otlex 的時間 |

### 變更成員角色

1. 找到要變更的成員
2. 點擊角色欄位的下拉選單
3. 選擇新角色
4. 確認變更

**限制**：
- Owner 無法被降級（除非先轉移所有權）
- 無法將自己降級
- Admin 無法將其他 Admin 改為 Owner

### 移除成員

1. 找到要移除的成員
2. 點擊成員右側的選單
3. 選擇「移除成員」
4. 確認操作

**移除後**：
- 該成員立即失去所有存取權限
- 其建立的內容和稽核記錄會保留
- 可以重新邀請同一 Email

**無法移除**：
- 自己
- Owner（需先轉移所有權）

## 轉移組織所有權

### 轉移步驟

只有 Owner 可以轉移所有權：

1. 進入「設定」>「成員管理」
2. 找到要轉移的對象（必須是 Admin）
3. 點擊成員右側選單
4. 選擇「轉移所有權」
5. 輸入你的密碼確認
6. 完成轉移

### 轉移後的變化

| 項目 | 原 Owner | 新 Owner |
|-----|---------|---------|
| 角色 | 變為 Admin | 變為 Owner |
| 組織管理 | 保留管理權限 | 獲得完整權限 |
| 計費通知 | 不再收到 | 開始收到 |
| 刪除組織 | 無法 | 可以 |

## Email 網域自動加入

### 功能說明

Enterprise 方案支援設定 Email 網域白名單，讓公司員工可以：
- 自動申請加入組織
- 或自動加入（依設定）

### 設定步驟

1. 進入「設定」>「組織設定」
2. 找到「自動加入網域」
3. 新增公司 Email 網域（如 `yourcompany.com`）
4. 選擇模式：
   - **需要審核**：員工申請後，Admin 審核
   - **自動加入**：直接加入為指定角色
5. 如選自動加入，設定預設角色
6. 儲存設定

### 設定範例

```
網域：yourcompany.com
模式：自動加入
預設角色：Member
```

設定後，任何使用 `@yourcompany.com` Email 註冊的用戶會自動加入組織並成為 Member。

## 活動記錄

### 查看成員活動

Admin 和 Owner 可以查看成員活動：

1. 進入「設定」>「活動記錄」
2. 篩選成員或時間範圍
3. 查看活動詳情

### 記錄的活動類型

| 類型 | 範例 |
|-----|------|
| 登入 | 成員登入系統 |
| 稽核 | 執行 SEO 稽核 |
| 內容 | 生成或編輯內容 |
| 設定 | 變更品牌或整合設定 |
| 成員 | 邀請或移除成員 |

## 常見問題

### 邀請沒有收到？

1. 請對方檢查垃圾郵件資料夾
2. 確認 Email 拼寫正確
3. 確認沒有被公司郵件系統擋掉
4. 嘗試重新發送邀請

### 成員達到上限怎麼辦？

1. 升級到更高方案
2. 或移除不需要的成員
3. 或將部分成員改為 Viewer（仍佔用名額）

### 可以限制成員只能存取特定品牌嗎？

目前不支援。所有成員可以存取組織下的所有品牌（依角色權限操作）。Enterprise 方案可以客製這項功能。

### 成員離職後如何處理？

1. 立即移除該成員
2. 如果他是整合連接者，重新連接整合
3. 如果他有重要的待辦稽核，指派給其他成員

### 可以批次邀請多人嗎？

目前不支援批次邀請，需逐一邀請。Enterprise 方案可以透過 CSV 匯入批次邀請。
