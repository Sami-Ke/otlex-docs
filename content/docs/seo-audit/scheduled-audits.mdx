## 什麼是排程管理？

**排程管理（Scheduler）** 讓你建立週期性的自動化任務，無需手動操作就能定期執行各種 SEO 和內容維護工作。系統會在背景自動執行，完成後可回到對應頁面查看結果。

### 為什麼需要定期排程？

| 痛點 | 排程管理的解決方式 |
|------|------------------|
| 忘記定期做 SEO 審查 | 排程自動執行，不依賴人工提醒 |
| 每次手動爬蟲費時費力 | 設定固定爬蟲頻率，資料永遠最新 |
| 競品動態難以追蹤 | 定期自動分析競品，即時掌握變化 |
| GSC/GA4 資料同步不及時 | 定期自動同步，儀表板資料不落伍 |

## 八大排程任務類型

### 1. SEO 審查（SEO Audit）

自動對指定頁面或整站執行 SEO 審查，產出四維度分析報告。

**可設定的審查範圍：**

| 模式 | 說明 | 適用場景 |
|------|------|----------|
| 整站審查 | 掃描 robots.txt、sitemap、全站共用設定 | 定期網站健檢 |
| 指定 URL | 針對特定重要頁面的深度分析 | 核心落地頁監控 |
| 批次 URL | 一次審查多個指定頁面 | 系列文章定期追蹤 |

**審查結果查看：**
- 完成後進入「AI Search 審查」頁面查看最新報告
- 可對比不同時間的分數變化，追蹤優化成效

**建議排程頻率：**

| 頁面重要性 | 建議頻率 |
|------------|----------|
| 核心轉換頁（首頁、定價、服務頁） | 每週 |
| 重要部落格文章 | 每兩週 |
| 一般內容頁面 | 每月 |

---

### 2. 網站爬蟲（Crawl）

更新品牌的頁面快取，讓內容地圖、內部連結工具等功能獲得最新的頁面資料。

**可設定的爬取模式：**

| 模式 | 說明 | 爬取範圍 |
|------|------|----------|
| **首頁快速分析** | 從首頁出發，爬取連結到的主要頁面 | 適合中型網站快速更新 |
| **sitemap 智慧發現** | 讀取 sitemap.xml 爬取列出的所有頁面 | 適合有完整 sitemap 的網站 |
| **整站深度爬取** | 遞迴爬取所有可達頁面 | 適合需要完整頁面庫的場景 |
| **指定 URL** | 只爬取指定的特定頁面 | 更新指定頁面的快取 |

**何時需要更新爬取？**

- 發布了大量新文章後
- 網站改版或調整了導覽結構後
- 進行內部連結優化分析前
- 發現內容地圖資料落後實際網站時

**建議排程頻率：**

| 情況 | 建議頻率 |
|------|----------|
| 內容更新頻繁的部落格 | 每天或每週 |
| 穩定的企業官網 | 每兩週或每月 |
| 電商（商品頻繁更動） | 每天 |

---

### 3. 內容重複檢測（Duplicate Detection）

定期掃描整站，找出重複或高度相似的頁面，避免 SEO 排名被稀釋。

**掃描模式：**

| 模式 | 速度 | 準確度 | 說明 |
|------|------|--------|------|
| 快速掃描 | 最快 | 中等 | 標題與摘要比對，適合日常監控 |
| 精準 Hash | 中等 | 高 | 完整 HTML 比對，找完全相同的頁面 |
| 近似掃描 | 較慢 | 最高 | SimHash 演算法，偵測高度相似內容 |

**定期執行的價值：**
- 新發布的內容可能不知不覺與舊文章重複
- CMS 自動產生的分類頁、標籤頁容易造成重複
- 定期排程確保問題及早發現

**建議排程頻率：**每月一次（新增大量內容後可額外執行一次）

---

### 4. 競爭對手分析（Competitor Analysis）

定期分析指定競爭對手的 SEO 策略與關鍵字布局，讓你掌握市場動態。

**可分析的競品資訊：**

| 分析面向 | 說明 |
|----------|------|
| 關鍵字排名 | 競品在哪些關鍵字排名靠前 |
| 新增/消失的關鍵字 | 競品最近在布局哪些新詞 |
| 內容策略 | 競品發布了哪些新內容 |
| Schema 使用 | 競品採用了哪些結構化資料 |
| SEO 分數趨勢 | 競品整體 SEO 健康度變化 |

**設定競品清單：**
- 最多可追蹤數個主要競爭對手（以 App 方案為準）
- 輸入競品的完整網站 URL
- 系統會自動識別並定期分析

**建議排程頻率：**每週（競爭激烈行業）或每月（競爭穩定行業）

---

### 5. 分析同步（Analytics Sync）

自動同步 Google Search Console (GSC) 和 Google Analytics 4 (GA4) 的最新資料。

**同步的資料類型：**

| 資料來源 | 同步內容 |
|----------|----------|
| **Google Search Console** | 搜尋查詢、頁面曝光、點擊率、平均排名 |
| **Google Analytics 4** | 網站流量、使用者數、工作階段、跳出率 |

**為什麼需要定期同步？**

GSC 和 GA4 本身不會即時推送資料到第三方工具，需要主動拉取。設定定期同步排程後：
- 儀表板數據不落伍
- 排名追蹤有最新的搜尋表現數據
- 關鍵字研究有最新的曝光和點擊資料

**注意：GSC 本身有 2-3 天的資料延遲（Google 的限制），這是正常的。**

**建議排程頻率：**每天（讓數據盡量接近即時）

---

### 6. 商家資訊提要（Merchant Feed）

自動更新電商/商家的產品資訊提要（Product Feed），確保搜尋引擎和購物平台獲得最新的商品資訊。

**適用場景：**
- 電商網站有大量商品頁面
- 商品價格、庫存狀態頻繁變動
- 需要維護 Google Merchant Center 資料

<Callout type="info">
  Merchant Feed 功能需要品牌已完成商品目錄設定，且方案支援此功能。詳情以 App 顯示為準。
</Callout>

**建議排程頻率：**每天（商品資訊變動頻繁時）或每週

---

### 7. 主題發現（Topic Discovery）

自動分析品牌的內容缺口，發掘有潛力的新內容主題與關鍵字機會。

**分析內容：**

| 分析面向 | 說明 |
|----------|------|
| 內容缺口 | 競品有覆蓋但你尚未撰寫的主題 |
| 關鍵字機會 | 搜尋量穩定但競爭度較低的關鍵字 |
| 趨勢主題 | 近期搜尋量成長中的相關主題 |

**建議排程頻率：**每月（搭配內容日曆規劃使用）

---

### 8. 內容衰退檢測（Content Decay）

定期掃描既有內容的流量與排名趨勢，及早發現表現下滑的頁面，提供更新與修復建議。

**偵測指標：**

| 指標 | 說明 |
|------|------|
| 流量下降 | 頁面自然流量持續下滑 |
| 排名滑落 | 主要關鍵字排名退步 |
| CTR 降低 | 曝光量穩定但點擊率下降 |

**建議排程頻率：**每月（內容更新頻繁的網站可設為每兩週）

---

## 如何建立排程

### 步驟 1：進入排程管理

從左側選單進入「**排程管理（Scheduler）**」，你會看到目前品牌已設定的所有排程（或空清單）。

### 步驟 2：點擊「新增排程」

點擊右上角「**新增排程**」按鈕，開啟排程設定視窗。

### 步驟 3：選擇任務類型

從八種任務類型中選擇你要排程的任務：

```
SEO 審查 → 網站爬蟲 → 重複內容偵測 → 競爭對手分析 → 分析同步 → 商家資訊提要 → 主題發現 → 內容衰退檢測
```

### 步驟 4：設定任務參數

依任務類型設定對應的參數：

| 任務類型 | 需要設定的參數 |
|----------|----------------|
| SEO 審查 | 審查模式（整站 / 指定 URL）；若指定 URL 需貼入網址 |
| 網站爬蟲 | 爬取模式（首頁快速 / sitemap / 整站 / 指定 URL） |
| 重複內容偵測 | 掃描模式（快速 / 精準 Hash / 近似） |
| 競爭對手分析 | 競品網址（可設定一個或多個） |
| 分析同步 | 資料來源（GSC / GA4 / 兩者） |
| 商家資訊提要 | 提要類型與範圍 |
| 主題發現 | 分析範圍（品牌內容缺口 / 競品對比） |
| 內容衰退檢測 | 偵測範圍（全站 / 指定頁面群組） |

### 步驟 5：設定執行頻率

| 頻率 | 說明 |
|------|------|
| **每天** | 每天在設定的時間點自動執行 |
| **每週** | 選擇每週幾執行，設定時間 |
| **每月** | 選擇每月幾號執行 |

<Callout type="info">
  排程執行時間建議避開網站流量高峰期，以免影響使用者體驗。建議設定在深夜或清晨（例如 UTC 02:00）。
</Callout>

### 步驟 6：儲存排程

確認設定無誤後點擊「儲存」。排程立即生效，系統會在下一個排程時間點自動執行。

## 排程管理頁面功能

建立排程後，你可以在排程管理頁面看到：

| 欄位 | 說明 |
|------|------|
| 任務名稱 | 排程的描述性名稱 |
| 任務類型 | SEO 審查 / 爬蟲 / 重複偵測 / 競品分析 / 分析同步 / 商家提要 / 主題發現 / 內容衰退 |
| 頻率 | 每天 / 每週 / 每月 |
| 狀態 | 啟用中 / 暫停 |
| 上次執行 | 上次執行的時間點 |
| 下次執行 | 下次預計執行的時間點 |

### 管理操作

| 操作 | 說明 |
|------|------|
| **暫停** | 暫時停止執行，不刪除設定 |
| **啟用** | 重新啟動已暫停的排程 |
| **編輯** | 修改頻率或任務參數 |
| **刪除** | 永久移除排程 |
| **立即執行** | 不等下次排程時間，立即手動觸發一次 |

## 排程建議（Recommendations）

系統會根據品牌設定和歷史資料，自動產生一組**排程建議（Schedule Recommendations）**，推薦你設定的最佳排程組合。

建議依據：
- 品牌網站大小（影響爬蟲頻率建議）
- 是否已連接 GSC/GA4（影響分析同步建議）
- 是否有設定競品（影響競品分析建議）
- 方案等級（影響可用功能）

你可以直接採用建議，或根據自己的需求調整。

## 查看執行結果

排程執行完成後，需要到對應的功能頁面查看結果：

| 任務類型 | 在哪裡看結果 |
|----------|-------------|
| SEO 審查 | 「AI Search 審查」> 選擇對應日期的記錄 |
| 網站爬蟲 | 「內容地圖」> 頁面清單（資料會自動更新） |
| 重複內容偵測 | 「工具」>「重複內容偵測」 |
| 競爭對手分析 | 「工具」>「競爭對手分析」 |
| 分析同步 | 「儀表板」或「排名追蹤」（資料更新後） |
| 商家資訊提要 | 「工具」>「商品目錄最佳化」 |
| 主題發現 | 「主題」> 主題建議清單 |
| 內容衰退檢測 | 「工具」>「內容衰退檢測」 |

<Callout type="info">
  目前排程管理頁面以「建立/管理排程」為主，不包含個別任務的執行歷史記錄和通知功能。若任務執行失敗，請手動觸發並查看相應頁面。
</Callout>

## 建議的排程組合

以下是針對不同類型網站的推薦排程設定：

### 部落格 / 內容網站

| 排程 | 頻率 | 說明 |
|------|------|------|
| 分析同步（GSC）| 每天 | 保持搜尋數據新鮮 |
| 網站爬蟲（sitemap 模式）| 每週 | 定期更新頁面庫 |
| SEO 審查（核心頁面）| 每月 | 監控重要頁面 |
| 重複內容偵測 | 每月 | 確保內容不重複 |

### 電商網站

| 排程 | 頻率 | 說明 |
|------|------|------|
| 分析同步（GSC + GA4）| 每天 | 完整監控流量 |
| 網站爬蟲（sitemap 模式）| 每天 | 商品頁面頻繁更新 |
| 商家資訊提要 | 每天 | 保持商品資料最新 |
| 競爭對手分析 | 每週 | 追蹤競品動態 |
| 重複內容偵測 | 每月 | 偵測商品描述重複問題 |

### 企業官網

| 排程 | 頻率 | 說明 |
|------|------|------|
| 分析同步（GSC）| 每天 | 追蹤品牌搜尋 |
| 網站爬蟲 | 每月 | 官網內容更新不頻繁 |
| SEO 審查（重要服務頁）| 每月 | 確保核心頁面優化 |
| 競爭對手分析 | 每月 | 了解行業動態 |

## 常見問題

### 排程沒有在設定時間執行？

可能原因：
- 排程被暫停（檢查狀態是否為「啟用」）
- 系統維護或延遲（可在主頁查看系統狀態）
- 任務參數有誤（嘗試「立即執行」看是否有錯誤）

### 可以設定幾個排程？

排程數量限制依方案而異。一般建議優先設定最關鍵的任務，例如：分析同步、爬蟲、SEO 審查。

### 排程會消耗配額嗎？

是的，自動排程執行與手動執行一樣會消耗對應的功能配額（例如 SEO 審查次數）。請確保你的方案有足夠的配額支撐排程頻率。

### 如何知道排程是否成功執行？

- 在排程管理頁面檢查「上次執行時間」是否有更新
- 到對應的功能頁面確認是否有新的結果資料
