## 執行審查前的準備

在跑第一次審查之前，確認以下事項可以讓結果更完整：

| 準備事項 | 必要性 | 說明 |
|---------|--------|------|
| **網頁公開可訪問** | 必要 | 審查系統需要爬取你的頁面，確保頁面不需要登入 |
| **已連接 GSC** | 建議 | GSC 讓審查能看到真實的排名和關鍵字數據 |
| **已完成品牌爬取** | 建議 | 讓審查報告有更完整的內容地圖背景資訊 |
| **知道目標 URL** | 必要 | 準備好要審查的頁面完整網址（含 https://） |

---

## 執行單頁審查（標準流程）

<Steps>
  <Step>
    ### 進入 SEO Audit 頁面
    從左側選單點擊「SEO Audit」（或「AI Search 審查」），進入審查列表頁面。

    若有過去的審查記錄，這裡會顯示歷史清單；若是第一次，清單會是空的。
  </Step>

  <Step>
    ### 新增審查

    點擊右上角「新增審查」按鈕，進入審查設定頁面。
  </Step>

  <Step>
    ### 填入審查網址

    在網址欄位輸入你想審查的頁面完整 URL，例如：

    ```
    https://example.com/blog/seo-guide
    ```

    **小提示：**
    - 輸入完整網址，包含 `https://`
    - 建議從最重要的頁面開始（首頁、主要服務頁或流量最高的文章）
    - 一次審查一個 URL；若需批次審查多個頁面，可使用排程功能
  </Step>

  <Step>
    ### 送出審查

    確認網址無誤後，點擊「開始審查」。系統會：

    1. 爬取目標頁面的 HTML 內容
    2. 同時爬取競品頁面（用於 Competitor 維度分析）
    3. 使用 AI 分析四大維度
    4. 生成可執行的優化建議

    **預計等待時間：** 通常 30 秒到 3 分鐘，取決於頁面複雜度和競品數量。
  </Step>

  <Step>
    ### 查看審查報告

    審查完成後，點擊記錄進入詳細報告頁。你會看到：

    - **綜合分數**（0-100 分）
    - **四大維度分數**（AEO / Technical / Content / Competitor）
    - **分類建議清單**（依優先級排序）
    - **AI 輔助操作**（可直接生成 Meta Description、Alt 文字、FAQ Schema 等）
  </Step>
</Steps>

## 解讀你的第一份報告

### 了解綜合分數

| 分數範圍 | 等級 | 代表意義 |
|---------|------|---------|
| 90-100 | 🟢 優秀 | 頁面 SEO 優化非常完善，維持現狀並持續監控 |
| 70-89 | 🟡 良好 | 有小幅改進空間，處理高優先建議可進一步提升 |
| 50-69 | 🟠 中等 | 有明顯改進空間，建議優先處理技術和內容問題 |
| 0-49 | 🔴 待改善 | 有嚴重 SEO 問題，需要全面優化 |

<Callout type="info">
  **剛開始不要對分數太在意。** 更重要的是看「哪些具體問題需要解決」，分數只是概況指標。許多網站首次審查得到 40-60 分是很正常的情況。
</Callout>

### 四大維度說明

| 維度 | 權重 | 評估內容 |
|------|------|---------|
| **AEO（AI 搜尋優化）** | 25% | FAQ Schema、問答格式、結構化資料、AI 爬蟲友善度 |
| **Technical SEO** | 30% | Title/Description、Canonical、Open Graph、圖片 Alt 文字、H1-H6 結構 |
| **Content（內容品質）** | 30% | 字數深度、關鍵字使用、可讀性、媒體豐富度 |
| **Competitor（競品對比）** | 15% | 與搜尋前 10 競品的 Schema、字數、關鍵字差距 |

### 建議優先級解讀

建議清單中每項問題都有優先級標示：

| 優先級 | 說明 | 建議處理時機 |
|-------|------|------------|
| 🔴 **高** | 直接影響搜尋排名的關鍵問題 | 本週處理 |
| 🟡 **中** | 有改進空間但不緊急 | 本月處理 |
| 🟢 **低** | 錦上添花的優化項目 | 有空就做 |

## 第一次審查最常發現的問題

根據常見審查結果，以下是新手最常遇到的 SEO 問題：

### Technical SEO 常見問題

| 問題 | 解決方式 |
|------|---------|
| **Meta Description 缺失或太短** | 為每頁寫 140-160 字的吸引人摘要 |
| **H1 標題缺失或多個 H1** | 確保每頁只有一個 H1，且包含主要關鍵字 |
| **圖片缺少 Alt 文字** | 為所有圖片加上描述性的 Alt 屬性 |
| **Open Graph 標籤缺失** | 加入 og:title、og:description、og:image |
| **標題層級跳躍** | 確保 H2 → H3 → H4 的層級順序 |

### AEO 常見問題

| 問題 | 解決方式 |
|------|---------|
| **沒有 FAQ Schema** | 對有問答內容的頁面加入 FAQ Schema 結構化資料 |
| **沒有問答格式內容** | 在文章中加入「常見問題」區塊 |
| **結構化資料不完整** | 確保 Article、Product 或其他 Schema 的必要欄位都填齊 |

### Content 常見問題

| 問題 | 解決方式 |
|------|---------|
| **內容太短** | 增加頁面的內容深度和完整度 |
| **關鍵字密度過低或過高** | 自然地調整目標關鍵字的使用頻率 |
| **可讀性分數低** | 縮短長句、使用條列式清單、分段更短 |
| **沒有圖片或媒體** | 加入相關圖片、圖表或嵌入影片 |

## 使用 AI 輔助快速修復

審查報告中，部分建議提供一鍵 AI 生成草稿的功能，讓你不用從零開始手寫：

### Meta Description 一鍵生成

1. 在建議清單找到「Meta Description 需要優化」項目
2. 點擊「AI 生成 Meta Description」
3. AI 根據頁面內容自動撰寫 140-160 字的吸引人描述
4. 複製草稿，根據品牌語調微調後套用到你的 CMS

### FAQ Schema 一鍵生成

1. 找到「加入 FAQ Schema」建議
2. 點擊「AI 生成 FAQ Schema」
3. AI 從頁面內容提取問答，生成完整的 JSON-LD 程式碼
4. 複製程式碼，加入到頁面的 `<head>` 或指定 Schema 位置

詳細的 FAQ Schema 操作說明請參考 [FAQ Schema 工具](/docs/tools/faq-schema)。

## 整站審查（Site Audit）

除了單頁審查，你也可以執行整站審查，掃描全站的共用設定：

### 整站審查的範圍

| 檢查項目 | 說明 |
|---------|------|
| **robots.txt** | 是否存在、是否誤封鎖重要頁面 |
| **sitemap.xml** | 是否存在、格式是否正確、頁面是否完整 |
| **llms.txt** | AI 爬蟲指引是否存在（新興最佳實踐） |
| **favicon** | 瀏覽器標籤圖示設定 |
| **全站 Meta 設定** | 是否有重複的 Title 或 Meta Description |

### 如何執行整站審查

1. 在 SEO Audit 頁面點擊「新增審查」
2. 選擇「整站審查」模式（而非輸入單一 URL）
3. 系統會自動掃描整站共用設定

## Local SEO 審查流程

如果你的業務是在地服務（診所、律師、維修、連鎖門市等），建議先完成 Local SEO 設定再執行審查：

<Steps>
  <Step>
    ### 設定 Local SEO 資訊

    進入「設定 > 品牌設定 > Local SEO」，開啟功能後填入：

    - 服務城市（可設定多個）
    - 主要服務項目
    - 電話與聯絡資訊
    - 預約連結（如有）
  </Step>

  <Step>
    ### 補充每個城市的細節

    建議至少為每個服務城市補充：
    - 地標 / 地址
    - 當地備註（例如「台北市大安區服務」）

    這讓城市頁面建議更精準，也避免不同城市頁面內容過度相似。
  </Step>

  <Step>
    ### 執行 SEO Audit

    設定完成後，正常執行 SEO 審查。系統會在既有分析流程中自動加入城市覆蓋分析，**不需要額外按任何按鈕**。
  </Step>

  <Step>
    ### 查看 Local Coverage 分析

    進入審查詳情的「建議」分頁，找到「Local SEO 城市覆蓋率」區塊，可以看到：

    - ✅ 已有對應城市頁面的城市
    - ⚠️ 有城市頁面但內容過薄
    - ❌ 完全缺少對應頁面的城市
  </Step>

  <Step>
    ### 用 GEO 篩選優先修正

    在建議清單使用 GEO 篩選器，優先處理：
    - 缺少城市頁（影響覆蓋率最大）
    - LocalBusiness Schema 缺失
    - 城市修飾詞（「台北○○服務」）未出現在標題
    - 城市頁面之間缺少內部連結
  </Step>
</Steps>

## 常見審查失敗原因

| 原因 | 說明 | 解決方式 |
|-----|------|---------|
| **頁面需要登入** | 審查爬蟲無法存取需要認證的頁面 | 確保要審查的頁面對外公開 |
| **CAPTCHA 保護** | 網站有驗證碼擋住爬蟲 | 聯繫客服協助設定白名單 |
| **robots.txt 封鎖** | robots.txt 禁止爬蟲訪問 | 檢查並調整 robots.txt，確保允許 Otlex 爬蟲 |
| **頁面回應過慢** | 頁面加載時間超過 timeout | 檢查網站速度，或稍後重試 |
| **JavaScript 渲染問題** | 頁面內容完全依賴 JS 渲染 | 系統會嘗試 JS 渲染，但複雜的 SPA 可能有限制 |
| **URL 輸入有誤** | 例如缺少 https:// 或有多餘空格 | 重新確認 URL 格式 |

## 解讀分數後的行動建議

### 第一週優先做什麼？

取得第一份審查報告後，建議按照以下優先順序行動：

1. **先修復高優先（🔴）的 Technical SEO 問題**
   - 這些問題可能直接導致搜尋引擎無法正確索引你的頁面
   - 通常修復成本低但效果顯著

2. **補全 Meta Description 和 Alt 文字**
   - 使用 AI 輔助生成功能可以快速批量完成
   - 這些是最容易修復且效果穩定的項目

3. **加入基礎 Schema Markup**
   - 至少在文章頁加入 Article Schema
   - 若有問答內容，加入 FAQ Schema

4. **記錄基線分數**
   - 記下第一次審查的各維度分數
   - 作為之後追蹤優化效果的基準

### 建立持續優化的習慣

| 頻率 | 建議動作 |
|------|---------|
| **每次發新文章後** | 立即執行單頁審查，確認 SEO 基礎設定無誤 |
| **每月** | 對核心頁面執行審查，追蹤分數變化 |
| **每季** | 整站審查，確保技術設定正確 |

## 常見問題

### 審查會直接修改我的網站嗎？

不會。Otlex 的審查功能只會讀取你的頁面內容，生成報告和建議草稿，**不會自動修改你的網站**。所有修改都需要你手動在自己的 CMS 中執行。

### 為什麼審查花了很長時間？

常見原因：
- 目標頁面的加載速度慢
- 系統正在爬取多個競品頁面做對比
- 系統排隊（高峰期）

若超過 10 分鐘仍未完成，可以嘗試關閉頁面後重新進入，或重新執行審查。

### 競品是怎麼選出來的？

系統會根據你頁面的主要關鍵字，自動搜尋 Google 排名前 10 的頁面作為競品比較對象。你無法手動指定競品（但可以在競爭對手分析工具中針對特定網站分析）。

### 同一頁面可以執行多次審查嗎？

可以。每次執行都會產生新的獨立報告，方便你追蹤修改前後的分數變化。建議在修改後等待 Google 重新索引（通常 3-7 天）再執行審查，這樣分析到的頁面狀態才是最新的。

### 審查分數多久更新一次？

分數是即時計算的，每次執行審查都會得到當下頁面狀態的最新分數。系統不會自動追蹤排名變化，需要你定期手動執行或設定排程。
