## 開始審查

<Steps>
  <Step>
    ### 進入 SEO Audit 頁面
    從左側選單點擊「SEO Audit」進入審查列表頁面。
  </Step>
  
  <Step>
    ### 輸入要審查的網址
    點擊「新增審查」，輸入你想要審查的頁面完整網址（包含 https://）。
  </Step>
  
  <Step>
    ### 等待審查完成
    系統會開始爬取和分析頁面，通常需要 30 秒到 2 分鐘。
  </Step>
  
  <Step>
    ### 查看審查報告
    審查完成後，點擊查看詳細報告和建議。
  </Step>
</Steps>

## Local SEO 使用流程（設定 → 審查 → 執行）

如果你是做在地服務（例如診所、律師、維修、連鎖門市），建議先完成 Local SEO 設定再跑審查，才會看到城市覆蓋分析與 GEO 建議。

<Steps>
  <Step>
    ### 到「品牌設定」填 Local SEO
    進入「設定 → 品牌設定 → Local SEO」，開啟功能後填入服務城市、主要服務、電話與預約連結。
  </Step>

  <Step>
    ### 補齊每個城市的細節
    建議至少補「地標 / 地址 / 備註」，這會讓城市頁建議更精準，也能避免內容過度同質化。
  </Step>

  <Step>
    ### 儲存後執行 SEO Audit
    回到 SEO Audit 跑審查，系統會在既有流程中自動加跑城市覆蓋分析（不需要額外按鈕）。
  </Step>

  <Step>
    ### 在審查詳情查看 Local Coverage
    到「建議」分頁查看「Local SEO 城市覆蓋率」區塊，確認每個城市是已覆蓋、內容過薄、或缺少頁面。
  </Step>

  <Step>
    ### 用 GEO 篩選優先修正
    在建議列表用 GEO 篩選，先處理高優先的缺城市頁、LocalBusiness Schema、城市修飾詞與內部連結問題。
  </Step>
</Steps>

## 審查包含哪些項目？

### Technical SEO

- Title Tag 長度與格式
- Meta Description 存在與長度
- Canonical URL 設定
- Heading 結構 (H1-H6)
- 圖片 Alt 文字
- 內部連結數量

### Content 分析

- 內容長度與深度
- 關鍵字使用
- 可讀性評估
- 內容結構

### AEO (AI 搜尋優化)

- FAQ Schema 存在
- 問答格式內容
- 結構化數據完整性
- AI 爬蟲友善度

### 競品比較

- 同類頁面 Schema 對比
- 內容深度差異
- 結構差異分析

## 常見審查失敗原因

| 原因 | 說明 | 解決方式 |
|-----|------|---------|
| 登入牆 | 頁面需要登入才能存取 | 確保頁面對外公開 |
| CAPTCHA | 網站有驗證碼保護 | 將爬蟲 IP 加入白名單 |
| JavaScript 渲染 | 內容完全靠 JS 渲染 | 系統會嘗試 JS 渲染 |
| 網站阻擋 | 網站封鎖爬蟲 | 檢查 robots.txt 設定 |
