## Otlex 是什麼？

Otlex 是一個結合 SEO 分析、AI 搜尋優化（GEO）與內容生成的一站式平台，幫助行銷人員和 SEO 工作者在 AI 搜尋時代維持並提升曝光度。

**Otlex 能做的事：**

| 功能領域 | 說明 |
|---------|------|
| **SEO 審查與分析** | 四維度掃描網站健康度，給出可執行的優化建議 |
| **AI 搜尋追蹤** | 追蹤品牌在 Perplexity、ChatGPT、Gemini 等 AI 搜尋引擎的曝光狀況 |
| **內容地圖** | 視覺化網站的頁面結構和內部連結，整合 GSC 數據找優化機會 |
| **排名追蹤** | 監控 Google 傳統搜尋與 AI 搜尋引擎的排名變化 |
| **AI 文章生成** | 根據品牌風格和目標關鍵字，生成高品質 SEO 文章 |
| **社群內容再利用** | 把文章快速轉換成各平台格式的社群貼文 |
| **專業工具** | FAQ Schema 生成、內部連結建議、競爭對手分析、重複內容偵測 |

## 開始之前

### 必要條件

| 項目 | 說明 |
|------|------|
| **已上線的網站** | 你想要優化的網站（需公開可訪問） |
| **Otlex 帳號** | 透過 Email 或 Google OAuth 完成註冊/登入 |

### 建議準備（強烈推薦）

| 項目 | 好處 |
|------|------|
| **Google Search Console 存取權** | 獲取真實的搜尋排名、點擊率、曝光數據 |
| **Google Analytics 4 存取權** | 分析流量來源和使用者行為 |
| **品牌相關資料** | 品牌名稱、目標受眾描述、競爭對手清單 |

<Callout type="info">
  即使沒有 GSC，Otlex 的許多功能仍可使用（爬蟲、SEO 審查、AI 文章生成等）。但 GSC 整合後，分析結果的準確度和實用性會大幅提升，強烈建議盡早連接。
</Callout>

## Otlex 的核心概念

在開始操作前，了解以下兩個概念能讓你更有效率地使用平台。

### 組織（Organization / Tenant）

組織是 Otlex 的最高管理層級，通常對應一間公司或一個團隊。組織層級管理：

| 管理對象 | 說明 |
|---------|------|
| **成員與權限** | 邀請團隊成員、設定 Owner / Admin / Editor / Viewer 角色 |
| **訂閱與計費** | 管理訂閱方案、查看配額用量、下載帳單 |
| **AI 使用配額** | 可設定各成員/功能的使用上限 |
| **共用整合設定** | 跨品牌共用的 AI Provider 設定 |

### 品牌（Brand）

品牌代表一個獨立的網站或產品線。一個組織可以建立多個品牌，每個品牌有完全獨立的：

- 網站網址與爬取的頁面內容庫
- SEO 審查記錄與分數歷史趨勢
- 主題規劃與 AI 文章內容
- GSC / GA4 整合授權
- 品牌風格設定（影響 AI 內容生成語調）

<Callout type="info">
  **實際範例**：假設你的公司「ABC 科技」有一個官網和一個獨立部落格，可以建立一個組織「ABC 科技」，然後在下面建立兩個品牌：「ABC 官網」（example.com）和「ABC Blog」（blog.example.com）。兩個品牌的分析和內容完全獨立管理。
</Callout>

## 首次設定流程

完整的設定流程如下：

```
建立帳號 → 建立組織 → 建立品牌 → 連接 GSC → 執行爬蟲 → 開始分析
```

### 步驟 1：建立帳號與組織

首次登入 Otlex 時，系統會引導你完成：

1. 選擇登入方式（Email 或 Google）
2. 填寫基本資訊（姓名、公司）
3. 建立組織（輸入組織名稱）

**組織名稱建議**：使用公司或品牌的正式名稱，例如「ABC Technology」而非「abc」。

### 步驟 2：建立第一個品牌

建立品牌時需要填寫：

| 欄位 | 說明 |
|------|------|
| **品牌名稱** | 在 Otlex 內顯示的名稱，例如「官網」 |
| **網站網址** | 完整網址，例如 `https://example.com`（含 https，不加結尾斜線） |
| **品牌描述**（選用）| 簡短描述網站定位，有助 AI 理解品牌背景 |

詳細步驟請參考 [建立你的第一個品牌](/docs/getting-started/create-brand)。

### 步驟 3：連接 Google Search Console

這是 **最重要** 的一步。GSC 連接後，Otlex 可以取得：

- 哪些關鍵字帶來流量
- 各頁面的曝光、點擊與平均排名
- 哪些關鍵字排名在 11-20 名（最有機會衝首頁）

詳細步驟請參考 [連接 Google Search Console](/docs/getting-started/connect-gsc)。

### 步驟 4：執行首次爬蟲

連接 GSC 後，下一步是讓 Otlex 爬取你的網站：

1. 進入「設定」>「網站管理」或在「內容地圖」頁面點擊「啟動爬取」
2. 選擇爬取模式：

| 模式 | 說明 | 適合情況 |
|------|------|---------|
| **首頁快速分析** | 從首頁出發，爬取主要連結頁面 | 快速了解主要結構 |
| **sitemap 智慧發現** | 讀取 sitemap.xml 的頁面清單 | 有完整 sitemap 的網站 |
| **整站深度爬取** | 遞迴爬取所有可達頁面 | 需要完整頁面庫 |

爬蟲完成後，內容地圖、內部連結建議、AI 文章生成等功能才能充分運作。

### 步驟 5：設定品牌寫作風格（可選但建議）

如果你打算使用 AI 文章生成，建議在生成前完成品牌風格設定：

1. 進入「設定」>「品牌設定」>「寫作風格」
2. 從已爬取的頁面中選取 5-8 篇代表性文章
3. 點擊「開始分析」，系統會提取你的品牌語調、慣用詞彙和寫作結構

這讓 AI 生成的文章更接近你自己的寫作風格，減少事後編輯的時間。

## 新手建議的功能探索順序

如果你是第一次使用 Otlex，建議按照這個順序體驗各功能，從最能快速看到價值的地方開始：

### 第一週

| 優先順序 | 功能 | 目的 |
|---------|------|------|
| 1 | **執行 SEO 審查** | 了解網站目前的 SEO 健康度基線 |
| 2 | **查看儀表板** | 掌握流量趨勢與快速致勝機會 |
| 3 | **查看內容地圖（健檢視圖）** | 找出孤島頁面和近首頁關鍵字 |

### 第二週

| 優先順序 | 功能 | 目的 |
|---------|------|------|
| 4 | **排名追蹤** | 設定要追蹤的重要關鍵字，建立歷史基線 |
| 5 | **主題生成** | 讓 AI 建議下一篇文章要寫什麼 |
| 6 | **生成第一篇文章** | 體驗從主題到發布的完整 AI 內容流程 |

### 持續使用

| 功能 | 使用時機 |
|------|---------|
| **FAQ Schema 工具** | 針對已有問答結構的頁面強化 Schema |
| **內部連結建議** | 每次發新文章後，找適合加連結的舊文章 |
| **競爭對手分析** | 每月了解競品的關鍵字布局動態 |
| **AI 搜尋追蹤** | 定期確認品牌在 AI 搜尋引擎的曝光狀況 |

## 不同使用者的快速入門路徑

### 我是 SEO 工作者

**重點功能**：SEO 審查 → 內容地圖 → 排名追蹤 → 競爭對手分析

1. 先執行整站 SEO 審查，得到基線分數
2. 連接 GSC，查看哪些關鍵字排名在 11-20 名
3. 用內容地圖找出孤島頁面和內連不足的問題
4. 設定定期排程，讓系統自動追蹤變化

### 我是內容行銷人員

**重點功能**：主題生成 → AI 文章生成 → 社群貼文 → 品牌風格

1. 連接 GSC，看看哪些關鍵字有流量但沒有對應文章
2. 設定品牌寫作風格
3. 用主題生成建立一個月的內容日曆
4. 批量生成文章並用貼文助手同步產生社群內容

### 我是代理商/顧問

**重點功能**：多品牌管理 → 審查報告 → 競爭對手分析 → 排程

1. 為每個客戶建立獨立品牌
2. 設定各品牌的定期 SEO 審查排程
3. 使用「報告」功能匯出客戶可看的分析報告
4. 邀請客戶加入組織（設定為 Viewer 角色）

## 常見新手問題

### 要先爬蟲還是先 SEO 審查？

都可以，但功能效果有差別：
- **先爬蟲**：能讓內容地圖、內部連結建議、AI 文章生成（品牌風格分析）完整運作
- **先 SEO 審查**：不依賴爬蟲，可以直接針對網址進行技術 SEO 分析

**建議**：兩個同時進行。爬蟲在背景執行時，先開始執行 SEO 審查。

### 沒有 GSC 也能用嗎？

可以。以下功能不需要 GSC：
- SEO 審查
- 網站爬蟲與內容地圖（頁面模式）
- AI 文章生成
- FAQ Schema 工具
- 重複內容偵測

但這些功能需要 GSC 才能完整發揮：
- 內容地圖的關鍵字和排名數據
- 儀表板的搜尋流量圖表
- 排名追蹤
- AI 搜尋追蹤

### 爬蟲需要多久？

| 網站規模 | 預估時間 |
|---------|---------|
| 小型網站（< 50 頁） | 5-15 分鐘 |
| 中型網站（50-500 頁） | 15-60 分鐘 |
| 大型網站（500+ 頁） | 1-3 小時 |

爬蟲在背景執行，你可以繼續使用其他功能。

### 一個帳號可以管理幾個網站？

你可以建立多個「品牌」，每個品牌對應一個網站。品牌數量上限取決於你的訂閱方案，詳情請參考[方案比較](/docs/account-billing/plans)。
