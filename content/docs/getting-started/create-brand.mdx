## 什麼是品牌？

在 Otlex 中，**品牌（Brand）** 是核心的工作單位，代表一個獨立的網站或產品線。所有的 SEO 分析、內容生成、數據追蹤都以品牌為單位進行。

### 品牌包含的資源

每個品牌都有完全獨立的：

| 資源類型 | 說明 |
|---------|------|
| **網站網址** | 你要優化的網站首頁 URL |
| **內容庫** | 網站的所有已爬取頁面與內容摘要 |
| **審查記錄** | SEO 審查的歷史分數與建議 |
| **主題規劃** | AI 建議的內容主題與你的主題草稿 |
| **文章內容** | 使用 AI 生成或手動建立的文章 |
| **GSC / GA4 整合** | 獨立的搜尋分析與流量數據 |
| **品牌風格** | 用於 AI 內容生成的品牌語調設定 |

### 何時需要建立多個品牌？

| 情況 | 建議做法 |
|------|---------|
| 管理多個不同網域的網站 | 每個網站建立獨立品牌 |
| 官網 + 獨立部落格（不同子網域） | 分別建立兩個品牌 |
| 代理商管理多個客戶 | 每個客戶網站建立獨立品牌 |
| 同一域名的不同產品線 | 通常歸入同一品牌即可 |

<Callout type="info">
  **子目錄 vs 子網域**：如果你的部落格是 `example.com/blog`（子目錄），通常歸入同一品牌。如果是 `blog.example.com`（子網域），因為是不同的 GSC 資源，建議建立獨立品牌。
</Callout>

## 建立品牌的完整步驟

### 步驟 1：進入品牌管理

有兩種方式可以建立新品牌：

**方式一：從側邊欄品牌選擇器**
1. 點擊左上角的品牌選擇器（顯示目前品牌名稱的下拉區域）
2. 在下拉選單底部點擊「新增品牌」

**方式二：從設定頁面**
1. 進入左側選單「設定」>「品牌設定」
2. 點擊右上角「新增品牌」按鈕

### 步驟 2：填寫基本資訊

| 欄位 | 必填 | 說明 | 範例 |
|-----|------|------|------|
| **品牌名稱** | ✅ | 在 Otlex 中顯示的名稱 | `官網`、`產品部落格`、`ABC科技` |
| **網站網址** | ✅ | 要優化的網站首頁 URL | `https://example.com` |
| **品牌描述** | 選用 | 簡短描述網站定位與目標受眾 | 「面向中小企業的雲端會計軟體」 |
| **品牌 Slug** | ✅ | URL 識別碼，自動生成可手動修改 | `official-website` |

<Callout type="warning">
  **網址格式注意事項：**
  - 請輸入完整網址，必須包含 `https://`（或 `http://`）
  - 不要在末尾加斜線：用 `https://example.com` 而非 `https://example.com/`
  - 若網站同時有 www 和非 www 版本，輸入你在 Google Search Console 設定的主要版本
  - 若你的網站有語系前綴（如 `https://example.com/zh/`），通常輸入根網域即可
</Callout>

### 步驟 3：描述品牌背景（建議填寫）

品牌描述雖為選用欄位，但填寫後有幾個好處：

- AI 文章生成時能更了解你的產品/服務背景
- SEO 審查的「品牌相關度」分析更準確
- 主題生成時能根據你的行業提供更相關的建議

**好的品牌描述範例：**

```
✅ 「面向 B2B SaaS 企業的數位行銷工具，幫助行銷團隊提升 SEO 和內容產出效率，主要客群為 50 人以下的科技新創公司」

❌「行銷軟體公司」（太模糊）
```

### 步驟 4：上傳品牌視覺（選用）

| 項目 | 用途 |
|-----|------|
| **品牌 Logo** | 顯示在側邊欄品牌選擇器、匯出報告等位置 |
| **品牌主色** | 用於報告的圖表配色 |

支援格式：PNG、JPG（Logo 建議使用透明背景的 PNG）

### 步驟 5：確認並建立

確認資訊無誤後點擊「建立品牌」，系統會：

1. 建立品牌並產生專屬工作空間
2. 將你導向該品牌的 Dashboard 或設定頁
3. 提示你連接 Google Search Console（若尚未連接）

## 建立後的初始設定流程

品牌建立完成後，建議按照以下順序完成初始設定，才能充分發揮各功能：

### 1. 連接 Google Search Console（最重要）

GSC 連接後，Otlex 可以取得真實的搜尋排名和流量數據。沒有 GSC，許多分析功能無法顯示實際數據。

詳細步驟請參考 [連接 Google Search Console](/docs/getting-started/connect-gsc)。

### 2. 啟動網站爬蟲

爬蟲讓 Otlex 建立你網站的頁面庫，這是以下功能的基礎：
- 內容地圖：視覺化頁面結構與內部連結
- 品牌寫作風格分析：從你的頁面學習語調
- 內部連結建議：基於現有頁面給出建議

**啟動爬蟲的方式：**
1. 進入「設定」>「網站管理」，點擊「啟動爬取」
2. 或在「內容地圖」頁面點擊「啟動爬取」

**選擇爬取模式：**

| 模式 | 說明 | 建議場景 |
|------|------|---------|
| **首頁快速分析** | 爬取首頁及其直接連結的頁面 | 快速了解主要頁面結構 |
| **sitemap 智慧發現** | 讀取 sitemap.xml 的完整頁面清單 | 有完整 sitemap 的網站（推薦） |
| **整站深度爬取** | 遞迴跟隨所有內部連結爬取 | 需要完整頁面庫，適合中小型網站 |

### 3. 執行第一次 SEO 審查

爬蟲和 GSC 設定好之後，執行第一次 SEO 審查，得到目前網站健康度的基線分數。

詳細說明請參考 [執行第一次 SEO 審查](/docs/getting-started/first-audit)。

### 4. 設定品牌寫作風格（如需 AI 文章生成）

如果你計畫使用 AI 文章生成功能，建議在生成第一篇文章前完成品牌風格設定：

1. 確認網站爬蟲已完成（需要已爬取的頁面作為分析素材）
2. 進入「設定」>「品牌設定」>「寫作風格」
3. 選取 5-8 篇代表你品牌語調的文章
4. 點擊「開始分析」

詳細說明請參考 [品牌風格設定](/docs/ai-content/brand-style)。

## 管理現有品牌

### 在多個品牌之間切換

點擊左上角的品牌選擇器下拉選單，選擇要切換到的品牌。切換後，所有功能頁面（審查、內容、排名等）都會顯示該品牌的資料。

### 修改品牌設定

1. 進入「設定」>「品牌設定」
2. 可修改：品牌名稱、描述、網站網址、Logo、品牌色
3. 部分欄位修改後需重新觸發爬蟲才會生效

### 修改網站網址的注意事項

修改網站網址是**高影響操作**，請謹慎執行：

| 影響 | 說明 |
|------|------|
| **爬取內容清除** | 舊的頁面庫會被清空，需重新爬取 |
| **GSC 重新連接** | 原有的 GSC 授權可能需要重新連接 |
| **審查歷史保留** | 歷史審查記錄會保留，但分數基線可能不具可比性 |
| **內部連結建議重算** | 需重新爬取才能更新 |

### 刪除品牌

<Callout type="error">
  刪除品牌會**永久移除**該品牌的所有資料，包含審查記錄、內容、主題、GSC 連接等。此操作**無法復原**。
</Callout>

刪除品牌通常需來信 [support@ethorx.com](mailto:support@ethorx.com) 協助處理。若你在 App 內有刪除入口，請先確認你已備份重要資料。

## 代理商的多品牌管理建議

如果你是數位行銷代理商，管理多個客戶的網站：

### 建議的組織架構

**方案一：單一組織，多個品牌（推薦給小型代理商）**
```
代理商組織
├── 客戶 A 品牌
├── 客戶 B 品牌
└── 客戶 C 品牌
```

優點：集中管理、帳單統一、AI 點數共用。
注意：各客戶資料在同一組織內，需用角色權限控管。

**方案二：每個客戶建立獨立組織**
```
客戶 A 組織（單獨計費）
└── 客戶 A 品牌

客戶 B 組織（單獨計費）
└── 客戶 B 品牌
```

優點：客戶資料完全隔離，可讓客戶自己管理帳單。
注意：需要多個帳號或子帳號管理。

### 命名規則建議

| 類型 | 命名範例 |
|------|---------|
| 品牌名稱 | `[客戶名] 官網`、`[客戶名] Blog` |
| 品牌 Slug | `client-a-website`、`client-b-blog` |

統一的命名規則讓你在多個品牌間快速識別，避免混淆。

## 常見問題

### 可以建立幾個品牌？

品牌數量上限取決於你的訂閱方案。詳情請查看 [方案與定價](/docs/account-billing/plans)。

### 可以將品牌轉移到其他組織嗎？

目前不支援品牌跨組織轉移，如有需求請來信 [support@ethorx.com](mailto:support@ethorx.com) 協助。

### 同一個網站可以建立兩個品牌嗎？

技術上可以，但不建議。重複的品牌會讓 GSC 資料和審查數據分散，難以比較。如需追蹤不同子目錄，通常在同一個品牌內操作即可。

### 品牌的爬蟲多久更新一次？

爬蟲不會自動更新，需要手動觸發或設定排程。建議設定週期性的爬蟲排程（例如每週一次），確保內容地圖和內部連結建議反映最新的網站狀態。

詳情請參考 [排程管理](/docs/seo-audit/scheduled-audits)。
