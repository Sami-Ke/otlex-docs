## SEO 審查是什麼？

Otlex 的 SEO 審查功能會深度分析你的網頁，從多個維度評估 SEO 健康度，並提供具體可執行的優化建議。

### 審查涵蓋的檢測項目

| 檢測維度 | 檢測內容 |
|---------|---------|
| **AEO (AI Engine Optimization)** | 頁面對 AI 搜尋引擎的友善程度，包含結構化資料、問答格式、內容清晰度 |
| **Technical SEO** | 技術面問題：Meta Tags、Canonical、圖片 Alt、標題層級、頁面速度提示 |
| **Content** | 內容品質評估：內容深度、可讀性、關鍵字分布、內容獨特性 |
| **Competitor** | 競品分析：與排名靠前的競品頁面對比，找出差距 |

### 審查類型

Otlex 提供兩種審查模式：

| 類型 | 說明 | 適用場景 |
|-----|------|---------|
| **單頁審查 (Page Audit)** | 深度分析單一頁面 | 重要頁面優化、新文章發布前檢查 |
| **整站審查 (Site Audit)** | 批次檢查整個網站 | 定期健檢、全面了解網站 SEO 狀況 |

本文先介紹單頁審查，整站審查請參考 [整站審查](/docs/seo-audit/site-audit)。

## 開始第一次審查

### 方式一：從儀表板快速開始

1. 進入品牌的儀表板 (Dashboard)
2. 在頁面上方找到「快速操作」區塊
3. 點擊「執行 SEO 審查」
4. 在彈出視窗中輸入要審查的頁面網址
5. 點擊「開始審查」

### 方式二：從 SEO 審查頁面

1. 從左側選單進入「AI Search 審查」
2. 點擊右上角「新增審查」按鈕
3. 輸入頁面完整網址（包含 `https://`）
4. 點擊「開始審查」

### 方式三：從內容地圖

1. 進入「內容地圖」
2. 點擊任一頁面節點
3. 在側邊面板點擊「執行審查」

<Callout type="info">
  **網址格式**：請輸入完整的頁面網址，例如 `https://example.com/blog/my-article`。系統會自動檢測網址有效性。
</Callout>

## 審查流程

點擊「開始審查」後，Otlex 會執行以下步驟：

### 階段 1：頁面爬取 (5-15 秒)

- 發送請求取得頁面 HTML
- 等待 JavaScript 渲染（如有需要）
- 擷取完整的 DOM 結構

### 階段 2：結構分析 (10-20 秒)

- 解析 HTML 結構
- 提取 Meta Tags、Schema Markup
- 分析標題層級、內部/外部連結
- 檢測圖片與多媒體元素

### 階段 3：AI 分析 (20-40 秒)

- 評估內容品質與深度
- 分析 AEO 友善程度
- 與競品頁面進行對比
- 生成具體優化建議

### 階段 4：生成報告

- 計算各維度分數
- 整理優化建議清單
- 標註優先級
- 完成審查報告

<Callout type="warning">
  **審查可能失敗的情況**：
  - 頁面需要登入才能訪問
  - 頁面有 CAPTCHA 或 Bot 檢測
  - 伺服器回應過慢或超時
  - 頁面有嚴重的 JavaScript 錯誤

  系統會自動偵測這些情況並顯示對應的錯誤提示。
</Callout>

## 理解審查結果

審查完成後，你會看到一份詳細的報告。

### SEO 總分

頁面會獲得一個 0-100 分的綜合評分：

| 分數範圍 | 評級 | 建議行動 |
|---------|------|---------|
| 90-100 | 優秀 | 維持現狀，持續監控 |
| 70-89 | 良好 | 處理高優先建議 |
| 50-69 | 待改善 | 優先處理技術問題與內容問題 |
| 0-49 | 急需優化 | 全面檢視並執行重大改進 |

### 四大維度分數

除了總分，還會顯示四個維度的個別分數：

#### AEO 分數

評估頁面對 AI 搜尋引擎（如 ChatGPT、Perplexity、Google AI Overview）的友善程度：

- 是否有 FAQ Schema
- 內容是否有清晰的問答結構
- 資訊是否容易被 AI 引用
- 是否有明確的實體資訊

#### Technical SEO 分數

檢測技術層面的 SEO 問題：

- Title Tag 長度與品質
- Meta Description 是否存在與品質
- Canonical URL 設定
- 圖片是否有 Alt 文字
- 標題層級是否正確（H1 > H2 > H3）
- 行動裝置友善度提示

#### Content 分數

評估內容本身的品質：

- 內容深度與字數
- 可讀性分數
- 關鍵字使用與分布
- 內容結構完整性
- 原創性指標

#### Competitor 分數

與搜尋排名靠前的競品頁面對比：

- 競品有而你沒有的 Schema
- 內容長度對比
- 關鍵字涵蓋度對比
- 結構化程度對比

### 優化建議清單

報告最重要的部分是具體的優化建議，每個建議都包含：

| 欄位 | 說明 |
|-----|------|
| 建議標題 | 簡述要做什麼 |
| 問題說明 | 詳細解釋為什麼這是問題 |
| 建議做法 | 具體如何修改 |
| 優先級 | 高/中/低，表示處理的急迫性 |
| 影響範圍 | 預期對 SEO 的影響程度 |

### 優先級說明

- **🔴 高優先**：嚴重影響 SEO 表現，建議立即處理
  - 例：缺少 Title Tag、重複內容、Canonical 錯誤

- **🟡 中優先**：有明顯改善空間，建議近期處理
  - 例：Meta Description 過短、缺少 Alt 文字、內容深度不足

- **🟢 低優先**：錦上添花，有時間再處理
  - 例：可以新增的 Schema、進階結構優化

## 執行優化建議

部分建議可能會提供一鍵 AI 輔助功能（以實際頁面顯示為準）：

### AI 輔助功能

| 功能 | 說明 |
|-----|------|
| **生成 Meta Description** | AI 根據頁面內容自動撰寫 Meta Description |
| **生成圖片 Alt 文字** | AI 分析圖片內容並生成描述性 Alt 文字 |
| **生成 FAQ Schema** | AI 根據內容生成 FAQ 並輸出 JSON-LD |

若你在建議旁看到「AI 生成」等按鈕，即可直接使用。

### 手動修改

大部分建議需要你到網站後台或 CMS 手動修改。每個建議都提供了明確的修改方向，你可以：

1. 複製建議內容
2. 到網站後台進行修改
3. 修改完成後重新執行審查確認

## 保存與追蹤

### 審查歷史

每次審查結果都會自動保存，你可以：

- 在「AI Search 審查」頁面查看歷史記錄
- 比較不同時間的分數變化
- 追蹤優化進度

### 設定定期審查

對於重要頁面，建議設定定期自動審查：

1. 前往「排程管理（Scheduler）」
2. 點擊「新增排程」，選擇審查類型與頻率（每日/每週/每月）
3. 排程會在背景執行；完成後可回到審查列表/排程執行紀錄查看結果

詳情請參考 [定期審查](/docs/seo-audit/scheduled-audits)。

## 常見問題

### 審查分數會影響實際 Google 排名嗎？

Otlex 分數是內部評估指標，不直接影響 Google 排名。但分數反映的問題確實是影響 SEO 的因素，改善這些問題通常會對排名有正面影響。

### 為什麼同一頁面多次審查分數不同？

可能原因包含：
- 頁面內容有更新
- 競品頁面有變化（影響 Competitor 分數）
- 分析時點差異（例如抓取到的內容版本不同）

### 審查一個頁面要多久？

通常 30 秒到 2 分鐘，取決於：
- 頁面複雜度
- 伺服器回應速度
- 是否需要 JavaScript 渲染

### 可以審查競品頁面嗎？

可以！輸入任何公開網址即可審查，這是分析競品 SEO 策略的好方法。
