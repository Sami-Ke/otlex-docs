## 為什麼需要整合？

Otlex 透過整合外部服務來獲取真實數據、直接發布內容、以及自動化社群媒體排程。

| 整合帶來的效益 | 說明 |
|---------|------|
| **真實搜尋數據** | 取得實際的排名、點擊、曝光，而非估算值 |
| **全面性分析** | 結合多種數據源，提供 360 度視角 |
| **直接發布** | 文章寫完後直接推送到你的 CMS，無需手動複製 |
| **效率提升** | 減少在不同工具間切換的操作成本 |

---

## 整合服務總覽

### 數據整合（分析用）

| 服務 | 功能說明 | 重要性 |
|-----|---------|--------|
| **Google Search Console** | 搜尋查詢/頁面的點擊、曝光、CTR、平均排名 | ⭐⭐⭐ 強烈建議 |
| **Google Analytics 4** | 流量分析、使用者行為、工作階段、轉換追蹤 | ⭐⭐ 建議 |

### 內容發布整合（CMS 直接發布）

| 平台 | 狀態 | 說明 |
|-----|------|------|
| **WordPress** | ✅ 支援 | 可建立草稿或直接發布，包含基本欄位（標題、內容、狀態） |
| **Shopify** | ✅ 支援 | 發布到 Shopify Blog（基本欄位） |
| **Webflow** | 🚧 Beta | 發布到 CMS Collection，需設定 token + collectionId + 欄位對應 |
| **Webhook** | ✅ 支援 | 將內容推送到任意自訂 HTTP 端點，由你的系統處理 |
| **Notion** | 🚧 臨時匯出 | 預設提供 Markdown/HTML 複製；API 模式可建立 Notion Page（純文字段落） |
| **Google Docs** | 🚧 臨時匯出 | 預設提供 Markdown/HTML 複製；API 模式可建立 Google 文件（純文字寫入） |

<Callout type="info">
  WordPress 和 Shopify 的直接發布功能（非複製貼上）需要 Pro 以上方案，且需先在「設定 > 整合」完成設定。
</Callout>

### 社群平台整合（社群排程發布用）

| 平台 | 連接方式 | 支援功能 |
|------|---------|---------|
| **Facebook** | OAuth 授權 | 發布到粉絲專頁 |
| **Instagram** | 透過 Facebook 授權 | 發布貼文（需商業帳號） |
| **LinkedIn** | OAuth 授權 | 發布到個人頁或公司頁 |
| **Twitter/X** | OAuth 授權 | 發布推文 |

社群平台的連接與管理在「社群運營 > 平台帳號」中進行，不在「設定 > 整合」頁面。

## 整合優先順序建議

### 第一步：連接 Google Search Console

GSC 是 **最重要** 的整合，強烈建議在使用 Otlex 前優先完成。連接後可以：

- 取得各頁面的真實搜尋排名
- 看到帶來流量的關鍵字清單
- 識別排名 11-20 名（最有機會衝首頁）的關鍵字
- 在內容地圖中顯示每頁的點擊與曝光數據
- 讓 SEO 審查的「SEO 機會」分析有實際數據支撐

沒有 GSC 整合，排名追蹤、著陸關鍵字分析、內容地圖的流量維度都無法運作。

**設定入口**：「設定」>「整合」>「Google Search Console」

詳細步驟請參考 [連接 Google Search Console](/docs/integrations/google-search-console)。

### 第二步：連接 Google Analytics 4

GA4 整合後可以補充以下數據：

- 實際的網站流量（工作階段數、使用者數）
- 使用者在頁面上的停留時間與互動行為
- 轉換追蹤（購買、表單填寫等）
- 流量來源分析（有機搜尋 vs 社群 vs 直接流量）

**設定入口**：「設定」>「整合」>「Google Analytics 4」

詳細步驟請參考 [連接 Google Analytics 4](/docs/integrations/google-analytics)。

### 第三步：設定內容發布整合（如需使用 AI 文章生成發布功能）

如果你計畫使用 AI 文章生成並直接發布到 CMS，需要先完成對應的整合設定：

| 你使用的 CMS | 需要設定的整合 |
|------------|-------------|
| WordPress | WordPress 網站 URL + Application Password |
| Shopify | Shopify 商店網址 + Admin API Token |
| Webflow | Webflow API Token + Collection ID |
| Webhook | 你的 Webhook 端點 URL |

**設定入口**：「設定」>「整合」>「CMS 整合」

詳細步驟請參考 [CMS 整合設定](/docs/integrations/cms-integrations)。

## 查看與管理整合狀態

### 整合狀態說明

進入「設定」>「整合」，可以看到每個整合的目前狀態：

| 狀態指示 | 說明 | 建議行動 |
|---------|------|---------|
| ✅ **已連接** | 整合正常運作，授權有效 | 無需處理 |
| ⚠️ **需要重新授權** | OAuth Token 已過期 | 點擊「重新連接」，重新完成授權 |
| 🔴 **連接失敗** | 授權被撤銷或服務端錯誤 | 重新連接，或檢查是否在 Google 端撤銷了授權 |
| ⏳ **同步中** | 資料同步任務正在背景執行 | 等待完成（通常幾分鐘到幾十分鐘） |

### 授權過期的常見原因

- Google OAuth Token 有效期到期（通常幾個月）
- 你在 Google 帳戶安全設定中撤銷了 Otlex 的存取權
- 你變更了 Google 帳號密碼
- GSC 資源的存取權限被移除

## 資料同步說明

### 觸發方式

| 整合服務 | 如何同步 | 備註 |
|---------|---------|------|
| **Google Search Console** | 在「設定 > 整合」或功能頁面點擊「同步 GSC」，會排程背景任務 | GSC 本身有 2-3 天資料延遲（Google 的限制） |
| **Google Analytics 4** | 在「設定 > 整合」點擊同步，會排程背景任務 | GA4 資料有 24-48 小時處理延遲 |
| **CMS 發布** | 你在文章頁按下「發佈」時才觸發 | 無延遲 |

<Callout type="info">
  GSC 的 2-3 天資料延遲是 Google 的限制，並非 Otlex 的問題。這表示今天同步到的資料，可能只反映到 2-3 天前的搜尋表現。
</Callout>

### 設定定期自動同步

為了讓排名追蹤和儀表板的數據保持新鮮，建議設定定期的分析同步排程：

1. 進入「排程管理（Scheduler）」
2. 新增「分析同步」任務
3. 選擇每天自動同步（建議設定在非流量高峰時段，如凌晨 2-4 點）

詳細說明請參考 [排程管理](/docs/seo-audit/scheduled-audits)。

## 權限與資料安全

### 最小必要權限原則

Otlex 只申請完成功能所需的最低權限：

| 服務 | 申請的權限 | 不申請的權限 |
|-----|---------|------------|
| GSC | 唯讀：搜尋數據、已驗證網站清單 | 不需要修改 GSC 設定的權限 |
| GA4 | 唯讀：分析報表數據 | 不需要 Google Drive、Gmail 等 |
| Notion | 你授權的頁面/資料庫的讀寫 | 不存取未授權的 Notion 內容 |
| Google Docs | 建立和編輯指定文件的寫入 | 不存取未授權的 Drive 內容 |
| WordPress | 文章發布 API（Application Password） | 不需要後台管理員全部權限 |

### Token 安全存儲

- 所有 OAuth Token 和 API Key 在存入資料庫前會進行加密
- 你的整合憑證只用於 Otlex 平台功能
- 你可以隨時在「整合」頁面中斷連接，並在 Google 帳戶設定中撤銷授權

## 各整合詳細設定指南

### Google Search Console 連接

連接流程：
1. 進入「設定」>「整合」
2. 點擊 Google Search Console 的「連接」按鈕
3. 選擇要授權的 Google 帳號
4. 授予 Otlex 唯讀存取搜尋數據的權限
5. 選擇對應的 GSC 資源（Property）

**重要注意事項：**
- 你需要在 GSC 中具有該資源的「完整」存取權限（Viewer 以上）
- 每個品牌只能連接一個 GSC 資源
- GSC 資源需與你在品牌設定中的網站網址一致

詳細步驟請參考 [連接 Google Search Console](/docs/integrations/google-search-console)。

### WordPress 整合

連接方式：使用 WordPress Application Password（不是你的登入密碼）

**設定步驟：**
1. 在 WordPress 後台，進入「使用者 > 個人資料」
2. 捲動到「應用程式密碼」區塊
3. 輸入名稱（例如「Otlex」）並點擊「新增應用程式密碼」
4. 複製生成的密碼（只顯示一次）
5. 在 Otlex 的「設定 > 整合 > WordPress」頁面填入網站 URL 和 Application Password

**前提條件：**
- WordPress 需為 5.6 以上版本（Application Password 功能）
- 需要 REST API 可以正常訪問（部分安全外掛可能封鎖）
- SSL（HTTPS）強烈建議

## 常見問題

### 必須連接所有整合嗎？

不必。但強烈建議至少連接 Google Search Console，這是獲得最佳 SEO 分析效果的關鍵。其他整合可以依需求選擇性連接。

### 連接 GSC 需要什麼條件？

- 你的網站必須已在 Google Search Console 中完成驗證
- 你需要在 GSC 中具有該資源的存取權限（Owner 或有資料存取權的使用者）
- 你使用 Google 帳號登入 Otlex 會更方便（可直接看到你有存取權的 GSC 資源清單）

### 一個品牌可以連接多個 GSC 資源嗎？

不行。每個品牌只能連接一個 GSC 資源。如果你有多個網站，請建立多個品牌。

### OAuth 授權多久後會過期？

Google OAuth Token 通常在幾個月後過期，或是在你更改 Google 帳戶密碼、撤銷授權時立即失效。Otlex 會在整合頁面顯示需要重新授權的提示。建議每次發現數據停止更新時，先到整合頁面確認授權狀態。

### 整合失效了怎麼辦？

**排查步驟：**
1. 進入「設定 > 整合」確認連接狀態
2. 如果顯示「需要重新授權」，點擊「重新連接」
3. 如果重新連接後仍失敗，嘗試先「中斷連接」再重新連接
4. 確認你的 Google 帳戶中是否撤銷了 Otlex 的存取權（Google 帳戶 > 安全性 > 第三方應用程式）
5. 如問題持續，請聯繫客服並提供錯誤截圖

### CMS 整合要怎麼設定？

詳細的 CMS 整合設定說明（WordPress、Shopify、Webflow、Webhook 等）請參考 [CMS 整合設定](/docs/integrations/cms-integrations)。
