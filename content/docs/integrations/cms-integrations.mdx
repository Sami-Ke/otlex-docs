## 概覽

CMS 整合讓你可以將 Otlex 中生成或編輯的內容，直接發布到你使用的內容管理系統，省去複製貼上的步驟。

### 支援的平台

| 平台 | 狀態 | 主要功能 |
|-----|------|---------|
| **Notion** | ✅ 完整支援 | 發布為頁面、同步到資料庫 |
| **Google Docs** | ✅ 完整支援 | 發布為文件、保留格式 |
| **WordPress** | ✅ 完整支援 | 發布文章、支援草稿 |
| **Shopify** | ✅ 完整支援 | 發布部落格文章 |
| **Webflow** | 🚧 Beta | 基本發布功能 |

### 為什麼需要 CMS 整合？

| 好處 | 說明 |
|-----|------|
| **節省時間** | 省去複製貼上、格式調整的步驟 |
| **保持格式** | 標題層級、列表、連結等格式完整保留 |
| **減少錯誤** | 避免手動複製時遺漏或格式錯誤 |
| **流程自動化** | 可與排程功能結合，自動發布 |

## Notion 整合

### 連接 Notion

1. 進入「設定」>「整合」
2. 找到「Notion」卡片
3. 點擊「連接」
4. 登入 Notion 並授權

### 授權設定

在 Notion 授權畫面：

1. 選擇你要分享給 Otlex 的頁面或資料庫
2. 點擊「允許存取」

<Callout type="info">
  Otlex 只能存取你在授權時明確選擇的頁面和資料庫，無法存取你的其他 Notion 內容。
</Callout>

### 發布到 Notion

1. 在 Otlex 編輯器中完成文章
2. 點擊「發布」>「Notion」
3. 選擇發布方式：

| 方式 | 說明 |
|-----|------|
| **新增頁面** | 在指定位置建立新頁面 |
| **新增資料庫項目** | 在指定資料庫新增一筆記錄 |

4. 選擇目標位置
5. 點擊「發布」

### Notion 資料庫欄位對應

如果發布到資料庫，可以對應欄位：

| Otlex 內容 | 對應的 Notion 欄位類型 |
|-----------|---------------------|
| 標題 | Title |
| 內文 | Page Content |
| 摘要 | Text |
| 標籤 | Multi-select |
| 發布日期 | Date |

### 更新已發布內容

1. 在 Otlex 中找到已發布的文章
2. 進行編輯
3. 點擊「更新」>「Notion」
4. 確認更新

Notion 中的頁面會同步更新。

## Google Docs 整合

### 連接 Google Docs

1. 進入「設定」>「整合」
2. 找到「Google Docs」卡片
3. 點擊「連接」
4. 選擇 Google 帳號並授權

### 權限說明

| 權限 | 用途 |
|-----|------|
| 查看和管理 Google Drive 中的文件 | 建立和編輯 Google 文件 |
| 查看 Google Drive 資料夾 | 選擇發布位置 |

### 發布到 Google Docs

1. 在 Otlex 編輯器中完成文章
2. 點擊「發布」>「Google Docs」
3. 選擇目標資料夾
4. 輸入文件名稱（選用）
5. 點擊「發布」

### 格式保留

發布到 Google Docs 時會保留：

- ✅ 標題層級（H1、H2、H3）
- ✅ 粗體、斜體、底線
- ✅ 有序和無序列表
- ✅ 超連結
- ✅ 圖片
- ✅ 表格（基本格式）
- ❌ 複雜表格樣式
- ❌ 自訂顏色

## WordPress 整合

### 連接 WordPress

1. 進入「設定」>「整合」
2. 找到「WordPress」卡片
3. 輸入你的 WordPress 網站網址
4. 輸入應用程式密碼
5. 點擊「測試連接」
6. 連接成功後儲存

### 建立應用程式密碼

在 WordPress 後台建立應用程式密碼：

1. 登入 WordPress 後台
2. 進入「使用者」>「個人資料」
3. 滾動到「應用程式密碼」區塊
4. 輸入名稱（如「Otlex」）
5. 點擊「新增應用程式密碼」
6. 複製生成的密碼（只會顯示一次！）

<Callout type="warning">
  應用程式密碼只會顯示一次，請立即複製。如果忘記，需要重新建立一個。
</Callout>

### 發布到 WordPress

1. 在 Otlex 編輯器中完成文章
2. 點擊「發布」>「WordPress」
3. 設定發布選項：

| 選項 | 說明 |
|-----|------|
| **發布狀態** | 草稿 / 立即發布 / 排程發布 |
| **分類** | 選擇文章分類 |
| **標籤** | 新增標籤 |
| **特色圖片** | 設定文章封面圖 |
| **作者** | 選擇發布作者 |

4. 點擊「發布」

### 發布狀態選項

| 狀態 | 說明 |
|-----|------|
| **草稿** | 儲存為草稿，不公開 |
| **待審閱** | 提交審閱，等待核准 |
| **立即發布** | 直接發布為公開文章 |
| **排程** | 設定未來發布時間 |

### SEO 外掛整合

如果你的 WordPress 安裝了 SEO 外掛（Yoast、Rank Math 等），發布時可以設定：

- Meta Title
- Meta Description
- Focus Keyword

## Shopify 整合

### 連接 Shopify

1. 進入「設定」>「整合」
2. 找到「Shopify」卡片
3. 輸入你的 Shopify 商店網址
4. 點擊「安裝 App」
5. 在 Shopify 後台授權

### 發布到 Shopify

1. 在 Otlex 編輯器中完成文章
2. 點擊「發布」>「Shopify」
3. 選擇目標部落格
4. 設定發布選項
5. 點擊「發布」

### Shopify 限制

| 功能 | 支援狀態 |
|-----|---------|
| 部落格文章 | ✅ 完整支援 |
| 產品描述 | ✅ 支援 |
| 頁面內容 | ✅ 支援 |
| 複雜格式 | ⚠️ 部分支援 |

## 媒體處理

### 圖片上傳

發布時，文章中的圖片會自動處理：

1. 上傳到目標 CMS
2. 更新圖片連結
3. 保留 Alt 文字
4. 維持原始尺寸

### 圖片限制

| 平台 | 最大檔案大小 | 支援格式 |
|-----|-------------|---------|
| Notion | 5 MB | JPG, PNG, GIF, WebP |
| Google Docs | 50 MB | JPG, PNG, GIF |
| WordPress | 依主機設定 | JPG, PNG, GIF, WebP |
| Shopify | 20 MB | JPG, PNG, GIF |

<Callout type="info">
  大型圖片會自動壓縮以符合平台限制。
</Callout>

## 發布記錄

### 查看發布記錄

每篇文章的發布記錄會保存：

1. 進入「內容管理」>「文章」
2. 找到目標文章
3. 點擊「發布記錄」

### 記錄內容

| 欄位 | 說明 |
|-----|------|
| 發布時間 | 發布的時間點 |
| 目標平台 | 發布到哪個 CMS |
| 發布位置 | 具體的頁面/文章連結 |
| 狀態 | 成功 / 失敗 / 已更新 |

### 重新發布

如果發布失敗或需要更新：

1. 查看失敗原因
2. 修正問題（如連接過期）
3. 點擊「重新發布」

## 疑難排解

### 發布失敗怎麼辦？

常見原因和解決方式：

| 原因 | 解決方式 |
|-----|---------|
| 連接過期 | 重新授權連接 |
| 權限不足 | 確認有發布權限 |
| 網路問題 | 檢查網路後重試 |
| 內容太長 | 部分平台有字數限制 |
| 圖片過大 | 壓縮圖片後重試 |

### WordPress 連接失敗？

檢查項目：

1. 網站網址是否正確（含 https://）
2. 應用程式密碼是否正確
3. 網站是否允許 REST API
4. 是否有安全外掛阻擋

### Notion 找不到頁面？

確認在 Notion 授權時有選擇該頁面。如需新增頁面存取權：

1. 中斷 Notion 連接
2. 重新連接
3. 在授權畫面選擇需要的頁面

## 常見問題

### 可以同時連接多個 CMS 嗎？

可以。每種 CMS 都可以獨立連接，發布時選擇目標平台。

### 發布後在 Otlex 編輯會同步嗎？

不會自動同步。編輯後需要點擊「更新」才會同步到 CMS。

### 可以排程發布嗎？

WordPress 支援排程發布。其他平台目前需要手動發布或使用各自的排程功能。

### CMS 上刪除會影響 Otlex 嗎？

不會。Otlex 的內容和 CMS 是獨立的，在 CMS 上刪除不會影響 Otlex 中的內容。
