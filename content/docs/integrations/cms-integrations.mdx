## 支援的 CMS

Otlex 支援將內容直接發布到以下平台：

| 平台 | 狀態 | 說明 |
|-----|------|------|
| **Notion** | ✅ 完整支援 | 發布為 Notion 頁面 |
| **Google Docs** | ✅ 完整支援 | 發布為 Google 文件 |
| **WordPress** | ✅ 完整支援 | 發布為草稿或文章 |
| **Shopify** | ✅ 完整支援 | 發布為部落格文章 |
| **Webflow** | 🚧 部分支援 | 基本發布功能 |

## Notion 整合

### 連接 Notion

1. 進入「設定」>「整合」>「Notion」
2. 點擊「連接 Notion」
3. 選擇要分享給 Otlex 的頁面或資料庫
4. 確認授權

### 發布內容

1. 在編輯器中完成文章
2. 點擊「發布」>「Notion」
3. 選擇目標頁面或資料庫
4. 點擊「發布」

### Notion 權限說明

- Otlex 只能存取你明確分享的頁面
- 可以建立新頁面
- 可以編輯已發布的內容
- 無法存取其他 Notion 內容

## Google Docs 整合

### 連接 Google Docs

1. 進入「設定」>「整合」>「Google Docs」
2. 點擊「連接」
3. 選擇 Google 帳號並授權

### 發布內容

1. 在編輯器中完成文章
2. 點擊「發布」>「Google Docs」
3. 選擇目標資料夾
4. 點擊「發布」

文章會以 Google 文件格式保存，保留格式和圖片。

## WordPress 整合

### 連接 WordPress

1. 進入「設定」>「整合」>「WordPress」
2. 輸入你的 WordPress 網站網址
3. 輸入應用程式密碼（需在 WordPress 後台建立）
4. 測試連接

### 建立應用程式密碼

1. 登入 WordPress 後台
2. 進入「使用者」>「個人資料」
3. 滾動到「應用程式密碼」區塊
4. 新增一個名為「Otlex」的應用程式密碼
5. 複製密碼（只會顯示一次）

### 發布選項

- **草稿**：發布為草稿，需手動審核後發布
- **立即發布**：直接發布為公開文章
- **排程發布**：設定未來發布時間

## Shopify 整合

### 連接 Shopify

1. 進入「設定」>「整合」>「Shopify」
2. 輸入你的 Shopify 商店網址
3. 安裝 Otlex 應用程式
4. 授權存取

### 發布內容

內容會發布到 Shopify 的部落格功能。

## 媒體處理

發布時，文章中的圖片會：

1. 上傳到目標 CMS
2. 自動調整連結
3. 保留 Alt 文字

**注意**：大型圖片可能需要較長上傳時間。

## 常見問題

### 發布失敗怎麼辦？

1. 確認 CMS 連接仍有效
2. 檢查你的權限是否足夠
3. 確認網路連接正常
4. 嘗試重新發布

### 可以更新已發布的內容嗎？

是的，Otlex 會記錄發布連結。你可以：

1. 在 Otlex 中編輯內容
2. 選擇「更新」而非「發布」
3. 目標 CMS 中的內容會同步更新
