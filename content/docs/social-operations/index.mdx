## 你可以用它做什麼

社群運營是管理社群媒體內容的集中平台：

- **AI 生成貼文**：根據品牌風格自動產生社群內容
- **管理貼文集合**：統一管理草稿、已排程、已發布的貼文
- **排程發布**：設定自動發布時間，一次安排一週內容
- **多平台管理**：連接 Facebook、Instagram、LinkedIn 等平台
- **品牌規則**：建立發文規範讓 AI 更符合調性

<Callout type="info">
  社群運營讓你把 SEO 內容延伸到社群媒體，放大內容的觸及率。
</Callout>

## 快速開始

1. **連接社群帳號**：進入「平台帳號」分頁，連接你的社群平台
2. **建立品牌規則**（選填）：設定品牌語調和發文規範
3. **產生第一篇貼文**：進入「產生貼文」，輸入主題讓 AI 生成
4. **編輯並儲存**：調整內容後儲存到貼文列表
5. **設定排程或發布**：選擇發布時間或立即發布

## 前置條件

| 條件 | 必要性 | 說明 |
|------|--------|------|
| 社群帳號 | **必要** | 需要有要管理的社群帳號 |
| AI 額度（如用 AI 生成） | 建議 | AI 生成貼文會消耗額度 |
| 平台授權 | 視需求 | 自動發布需要 OAuth 授權 |

## 五大功能區塊

### 1. 產生貼文（Composer）

AI 協助撰寫符合品牌風格的社群內容：

**輸入選項**：

| 選項 | 說明 | 範例 |
|------|------|------|
| **目標平台** | 選擇要發布的平台 | Facebook、Instagram |
| **貼文主題** | 告訴 AI 要寫什麼 | 「新功能發布公告」 |
| **來源文章**（選填） | 從文章延伸社群內容 | 選擇已發布的文章 |
| **額外指示**（選填） | 給 AI 的額外要求 | 「用輕鬆的語氣」 |

**AI 會自動處理**：
- 依平台調整字數（Twitter 280 字、FB 較長）
- 調整標籤格式（# 標籤位置和數量）
- 套用品牌語調
- 加入 UTM 追蹤參數

**操作流程**：

1. 選擇目標平台（可多選）
2. 輸入主題或選擇來源文章
3. 點擊「生成」
4. AI 產生草稿（約 10-20 秒）
5. 在編輯器中調整
6. 點擊「儲存」

### 2. 貼文列表（Posts）

管理所有貼文的集中地：

| 狀態 | 說明 | 可執行動作 |
|------|------|-----------|
| **草稿** | 尚未發布或排程 | 編輯、刪除、排程、發布 |
| **已排程** | 等待自動發布 | 編輯、取消排程 |
| **已發布** | 已成功發布 | 查看、複製再用 |
| **發布失敗** | 發布時發生錯誤 | 查看錯誤、重試 |

**功能**：
- 搜尋特定貼文
- 篩選平台和狀態
- 批量操作
- 排序（日期、平台）

### 3. 排程管理（Schedule）

設定貼文的自動發布時間表：

**設定排程**：

1. 選擇要排程的貼文
2. 設定發布日期和時間
3. 選擇要發布的平台（可多選同一貼文發多平台）
4. 確認排程

**行事曆視圖**：
- 月/週/日檢視
- 直觀看到排程分布
- 拖放調整時間

<Callout type="info">
  自動發布需要先連接社群帳號。若尚未連接，可以使用「複製貼文」功能手動發布。
</Callout>

### 4. 平台帳號（Platforms）

連接和管理社群平台帳號：

| 平台 | 連接方式 | 支援功能 |
|------|----------|----------|
| **Facebook** | OAuth 授權 | 發布到粉絲專頁 |
| **Instagram** | 透過 Facebook 授權 | 發布貼文（需商業帳號） |
| **LinkedIn** | OAuth 授權 | 發布到個人或公司頁面 |
| **Twitter/X** | OAuth 授權 | 發布推文 |

**連接流程**：

1. 點擊「連接帳號」
2. 選擇平台
3. 登入並授權 Otlex
4. 選擇要管理的頁面/帳號
5. 確認連接

**管理功能**：
- 查看連接狀態
- 重新授權（過期時）
- 解除綁定

### 5. 品牌社群規則

建立品牌專屬的發文規範：

| 設定項目 | 說明 | 範例 |
|----------|------|------|
| **語調** | 整體風格 | 專業但親切 |
| **用語偏好** | 特定要用的詞彙 | 「用戶」而非「客戶」 |
| **避免用語** | 不要出現的詞 | 避免「最好」、「第一」 |
| **標籤規則** | 標籤使用方式 | 最多 5 個、放在結尾 |
| **Emoji 風格** | Emoji 使用程度 | 適度使用、不要太多 |
| **內容限制** | 不應提及的話題 | 避免政治、競品比較 |

設定好規則後，AI 生成貼文時會自動套用。

## 使用情境

### 情境 1：從文章延伸社群內容

最常見的使用方式：

```
文章發布 → 點擊「貼文助手」→ AI 生成社群貼文 → 編輯 → 排程發布
```

1. 在文章編輯器中完成文章
2. 點擊「貼文助手」或「產生社群貼文」
3. 選擇目標平台
4. AI 自動生成貼文（含 UTM 參數）
5. 編輯調整
6. 儲存並設定排程

### 情境 2：規劃一週社群內容

批量建立一週份的貼文：

1. 先規劃好一週的內容主題
2. 依序建立每天的貼文
3. 使用行事曆視圖安排發布時間
4. 確認排程分布合理
5. 設定好就讓系統自動發布

### 情境 3：多平台同步發布

同一則消息發到多個平台：

1. 建立貼文時選擇多個平台
2. AI 會為每個平台調整格式
3. 可以針對各平台微調
4. 一次排程發布到所有平台

## 常見問題

### 平台授權失敗

**可能原因**：

| 原因 | 解法 |
|------|------|
| 平台服務暫時不可用 | 稍後再試 |
| 授權被拒絕 | 確認有給予所需權限 |
| 帳號問題 | 確認帳號狀態正常 |

### 排程的貼文沒有發布

**排查步驟**：

1. 檢查平台連接狀態是否正常
2. 確認授權沒有過期
3. 查看貼文狀態的錯誤訊息
4. 確認發布時間是否已過

### Instagram 無法自動發布

Instagram 自動發布的限制：

- 必須是 **Instagram 商業帳號**
- 必須透過 **Facebook 粉專** 連接
- 不支援限時動態（只支援一般貼文）
- 圖片需符合尺寸要求

### AI 生成的貼文不符合品牌風格

**改善方式**：

1. 完善「品牌社群規則」設定
2. 在生成時加入「額外指示」
3. 生成後手動調整
4. 多生成幾次選擇最好的

## 最佳實踐

### 內容規劃

| 建議 | 說明 |
|------|------|
| **規律發文** | 維持固定的發文頻率 |
| **內容多樣化** | 不要都是同類型內容 |
| **配合 SEO 內容** | 用社群放大文章觸及 |
| **預留編輯時間** | 排程前預留審核時間 |

### 平台差異化

不同平台需要不同策略：

| 平台 | 建議策略 |
|------|----------|
| **Facebook** | 較長內容、可放連結 |
| **Instagram** | 視覺導向、標籤重要 |
| **LinkedIn** | 專業內容、產業觀點 |
| **Twitter/X** | 簡短有力、及時話題 |

### 發布時間優化

一般建議的發布時段（依你的受眾調整）：

| 平台 | 建議時段 |
|------|----------|
| Facebook | 週二至週四 9:00-12:00 |
| Instagram | 週一至週五 11:00-13:00 |
| LinkedIn | 週二至週四 8:00-10:00 |
| Twitter | 週三至週五 9:00-11:00 |

### 效率技巧

- **批量建立**：一次規劃一週份量
- **範本複用**：好的貼文結構可以重複使用
- **UTM 追蹤**：每篇都加 UTM 追蹤成效
- **定期檢視**：觀察哪類內容效果好

## 注意事項

### 平台 API 限制

| 限制 | 說明 |
|------|------|
| 每日發布次數 | 部分平台有每日上限 |
| 圖片尺寸 | 各平台要求不同 |
| 連結預覽 | 部分平台會自動抓取 |
| 授權有效期 | 需定期重新授權 |

### 時區設定

- 發布時間以伺服器時區為準
- 建議設定時考慮你的目標受眾時區
- 行事曆會顯示本地時間
