## 你可以用它做什麼

儀表板是 Otlex 的數據中樞，幫助你：

- **快速掌握現況**：一眼看出網站流量、搜尋表現是否正常
- **發現異常與機會**：透過洞察告警及早發現問題或新機會
- **做出數據驅動的決策**：用數據支持內容策略和優先順序判斷
- **掌握內容表現**：透過內容洞察與頁面類型分析，了解不同類型內容的 SEO 效果

<Callout type="info">
  儀表板是用來「做決策」，不只是看報表。每次打開儀表板，問自己：「這週有什麼需要我行動的？」
</Callout>

## 快速開始

1. **確認整合狀態**：檢查 GA4 和 GSC 是否已連接（頁面會顯示狀態）
2. **選擇時間範圍**：點擊右上角選擇 14 天 / 30 天 / 自訂
3. **檢視核心指標**：查看頂部 KPI 卡片
4. **切換分頁**：瀏覽「內容洞察」與「頁面類型分析」等 Tab
5. **掃描告警區**：看是否有需要注意的洞察
6. **深入分析**：點擊有興趣的指標或圖表查看詳情

## 數據來源與更新頻率

| 數據類型 | 來源 | 更新頻率 | 延遲說明 |
|----------|------|----------|----------|
| 流量數據 | GA4 | 每次同步 | GA4 資料通常有 24-48 小時延遲 |
| 搜尋數據 | GSC | 每次同步 | GSC 資料通常有 2-3 天延遲 |
| 內容數據 | 內部 | 即時 | 無延遲 |
| 洞察告警 | 分析引擎 | 同步後觸發 | 依賴 GA4/GSC 資料 |

<Callout type="warning">
  **常見誤解**：「為什麼昨天的數據還沒出來？」這是正常的！GA4 和 GSC 都有資料延遲，不是系統故障。
</Callout>

## 功能詳解

### 頂部指標卡

儀表板頂部顯示核心 KPI 卡片：

| 指標 | 定義 | 來源 | 常見問題 |
|------|------|------|----------|
| **Sessions** | 選定時間範圍內的工作階段數。一個使用者可能產生多個 sessions。 | GA4 | 數字比預期低？檢查 GA4 追蹤碼是否正確安裝 |
| **Users** | 不重複訪客數量（依 cookie/device 識別）。 | GA4 | 跨裝置訪客會被算成多個 users |
| **Conversions** | 達成目標事件的次數（需在 GA4 設定轉換事件）。 | GA4 | 顯示 0？檢查是否有設定轉換事件 |
| **待處理主題** | 在「主題」模組中狀態為待處理的主題數。 | 內部 | 這是你的內容待辦清單 |

### 流量趨勢圖

以時間序列呈現網站流量變化：

- **如何解讀**：觀察整體趨勢（上升/下降/持平）比單日數字重要
- **週期性波動**：B2B 網站週末通常流量較低，這是正常的
- **異常尖峰**：突然的流量暴增可能是行銷活動、媒體曝光、或爬蟲流量

**互動操作**：
- 滑鼠懸停顯示當日詳細數據
- 點擊圖表可切換顯示 Sessions / Users / Pageviews

### 內容洞察 Tab

分析網站內容的 SEO 表現：

- **內容表現分析**：識別哪些內容帶來最多搜尋流量
- **快速致勝建議（Quick Wins）**：找出只需小幅優化就能顯著提升排名/流量的頁面
- **關鍵字機會矩陣**：以曝光量與排名的交叉分析，找出高潛力的優化機會

### 頁面類型分析 Tab

依頁面類型分類檢視 SEO 表現：

- 按頁面類型（文章、產品頁、首頁、分類頁等）分組統計
- 了解不同類型頁面的流量貢獻與 SEO 健康度

### 權威性快照

展示網站的整體權威性指標：

- **Authority Snapshot Card**：快速了解網站在搜尋引擎中的整體表現水準

### Schema 涵蓋矩陣

檢視網站的結構化資料涵蓋狀態：

- 各種 Schema 類型的實作比例
- 找出缺少結構化資料的頁面

### 搜尋查詢排行榜

顯示從 Google 搜尋帶來最多流量的關鍵字：

| 欄位 | 定義 | 如何使用 |
|------|------|----------|
| 關鍵字 | 使用者在 Google 搜尋的查詢詞 | 找出表現好的主題 |
| 點擊數 | 從搜尋結果點擊進入你網站的次數 | 越高越好 |
| 曝光數 | 你的網頁在搜尋結果中出現的次數 | 高曝光低點擊 = 標題/描述需優化 |
| CTR | 點擊率 = 點擊 / 曝光 | 平均約 2-5%，品牌詞可達 20%+ |
| 平均排名 | 在搜尋結果中的平均位置 | 排名 1-3 點擊率最高 |

<Callout type="info">
  **行動建議**：找出「高曝光、低 CTR」的關鍵字，優化這些頁面的標題和 Meta Description 可快速提升流量。
</Callout>

### 設備分布

圓餅圖顯示訪客的設備類型分布：

| 設備類型 | 說明 | 行動建議 |
|----------|------|----------|
| Desktop | 桌面電腦 | B2B 產品通常桌面佔比較高 |
| Mobile | 行動裝置 | 若佔比 > 50%，優先優化行動體驗 |
| Tablet | 平板 | 通常佔比最低 |

### Search Console 洞察告警

系統會自動分析 GSC 數據，當發現以下情況時顯示告警：

| 告警類型 | 觸發條件 | 建議處置 |
|----------|----------|----------|
| 排名大幅上升 | 重要關鍵字排名提升 5+ 位 | 持續觀察，考慮加強該主題內容 |
| 排名大幅下降 | 重要關鍵字排名下降 5+ 位 | 檢查頁面是否有變動、競爭對手是否有新內容 |
| 流量異常 | 點擊或曝光變化超過 20% | 確認是否有外部因素（季節、新聞事件） |
| 新機會 | 發現新的高潛力關鍵字 | 評估是否建立專屬內容 |

**處置流程**：
1. 點擊告警查看詳情
2. 確認是資料問題還是真實變化
3. 定位受影響的頁面/關鍵字
4. 前往對應工具（審查/內連/排名追蹤）採取行動

## 匯出月報

儀表板右上角的「**匯出月報**」按鈕，可讓你產生自訂的 HTML 格式報表：

### 操作步驟

1. 點擊頁面右上角「匯出月報」按鈕
2. 在「自訂報表」dialog 中：
   - **選擇報表期間**：本月或上月
   - **勾選要包含的區塊**：
     - GSC 搜尋概覽（曝光、點擊、CTR）
     - 流量趨勢
     - 內容產出統計
3. 點擊「產生報表」，系統會開啟 HTML 預覽後供下載

<Callout type="info">
  需要已連接 GSC 或 GA4 並完成至少一次資料同步，才能產出有數據的報表。
</Callout>

## 收藏日期範圍

將常用的日期範圍設定收藏，下次一鍵切換：

### 如何建立

1. 切換好你想收藏的日期範圍（14 天 / 30 天 / 自訂）
2. 點擊頁面右上角的「收藏此範圍」按鈕
3. 瀏覽器會跳出提示框，輸入這個收藏的名稱（例如「月報期間」）
4. 確認後即儲存

### 如何使用

儲存的收藏會以標籤形式顯示在日期範圍按鈕下方，點擊標籤即可快速套用該範圍。點擊標籤旁的 × 可移除收藏。

**限制**：最多可儲存 6 個收藏。

## 設定與前置條件

### 必要整合

| 整合 | 必要性 | 影響範圍 |
|------|--------|----------|
| Google Search Console | 強烈建議 | 搜尋查詢、排名、CTR、洞察告警 |
| Google Analytics (GA4) | 建議 | Sessions、Users、Conversions、設備分布 |

### 整合狀態檢查

儀表板會顯示各項整合的連線狀態：

- ✅ **已連接**：正常運作
- ⏳ **同步中**：資料正在更新，請稍候
- ❌ **未連接**：點擊「前往設定」完成整合
- ⚠️ **授權過期**：需要重新授權

## FAQ / 故障排除

### 看不到任何數據

**症狀**：所有指標都顯示「--」或「尚未同步」

**可能原因與解法**：

| 原因 | 解法 |
|------|------|
| 未連接 GA4/GSC | 前往「設定 > 整合」完成連接 |
| 連接後尚未同步 | 等待第一次同步完成（通常需要幾分鐘） |
| 授權已過期 | 重新授權整合 |
| 選擇的網站無資料 | 確認 GSC 中選擇的網站有流量 |

### 數字和 GA4/GSC 後台不一致

**症狀**：Otlex 顯示的數字和 Google 後台不同

**可能原因與解法**：

| 原因 | 解法 |
|------|------|
| 時區差異 | 確認 Otlex 和 GA4/GSC 使用相同時區 |
| 日期範圍不同 | 確認選擇的日期範圍完全一致 |
| 同步延遲 | Otlex 的資料可能比 Google 後台晚幾小時更新 |
| 篩選條件不同 | 檢查是否有套用額外的篩選 |

### 洞察告警沒有顯示

**症狀**：告警區一直是空的

**可能原因**：
- GSC 尚未連接或同步
- 網站流量/排名變化不大（這是好事！）
- 資料量不足以觸發分析

### 同步一直顯示進行中

**症狀**：整合狀態一直顯示「同步中」超過 30 分鐘

**解法**：
1. 重新整理頁面
2. 若仍未解決，嘗試重新連接整合
3. 聯絡客服協助檢查

## 最佳實踐

### 每週復盤清單

每週花 15 分鐘完成以下檢查：

- [ ] 流量趨勢是否正常？（與上週比較）
- [ ] 有沒有異常告警需要處理？
- [ ] 搜尋查詢排行榜前 10 名有無變化？
- [ ] 待處理主題是否需要調整優先順序？
- [ ] 是否有「高曝光低 CTR」的關鍵字可優化？

### 把發現轉成任務

當你在儀表板發現問題時，建立對應的行動：

| 發現 | 下一步行動 | 對應工具 |
|------|------------|----------|
| 某關鍵字排名下降 | 檢查頁面 SEO 健康度 | SEO 審查 |
| 高曝光低 CTR | 優化標題和描述 | 文章編輯器 |
| 發現新機會關鍵字 | 建立專屬內容 | 主題 / 文章生成 |
| 頁面沒有內連 | 加入內部連結 | 內部連結工具 |

### 建立固定檢視習慣

- **每日**（5 分鐘）：掃一眼流量趨勢和告警
- **每週**（15 分鐘）：完成復盤清單，調整優先順序
- **每月**（30 分鐘）：用 30 天視圖回顧整體表現，規劃下月重點
